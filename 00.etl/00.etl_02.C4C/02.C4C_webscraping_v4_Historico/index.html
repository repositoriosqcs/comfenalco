    Configuración del Entorno para la Automatización del Proceso de Scraping¶

    Este bloque de código configura el entorno necesario para llevar a cabo un proceso de scraping automatizado utilizando herramientas modulares y un sistema de logging bien estructurado.

    Primero, se importan los módulos básicos de Python como logging, os, sys, time, datetime y dateutil.relativedelta, esenciales para la configuración del entorno. También se agrega al sys.path la ruta de una carpeta superior que contiene el módulo Utils.Funciones. Esto permite organizar el proyecto en módulos reutilizables y facilita la importación de funciones específicas. Entre las funciones importadas destacan setup_logger, para configurar el sistema de logging, log_tiempo, para registrar el tiempo de ejecución, y download_file_between_dates, para manejar la descarga de archivos.

    El sistema de logging se configura para registrar eventos clave del proceso. Un logger identificado por el nombre del módulo (__name__) se establece con un nivel de detalle INFO, suficiente para capturar información relevante y errores críticos. Los mensajes se registran en dos destinos: un archivo llamado scraper.log para el seguimiento a largo plazo y la consola para monitoreo en tiempo real. Ambos manejadores comparten un formato estandarizado que incluye la hora, el nivel de registro y el mensaje, lo que facilita la depuración y el análisis.

    Además, se ignoran advertencias específicas para evitar distracciones innecesarias durante la ejecución del script. Si hay otros loggers configurados, también se ajusta su nivel de detalle a INFO para mantener la coherencia en el registro de eventos.

    1. Configuración del Entorno para la Automatización del Proceso de Scraping: Configura el entorno necesario para llevar a cabo un proceso de scraping automatizado utilizando herramientas modulares y un sistema de logging bien estructurado.
    2. Importación de Librerías: Importa las librerías necesarias como selenium, logging, os, sys, time, datetime, y dateutil.relativedelta. También se agrega la ruta de la carpeta Funciones al sys.path y se importan funciones específicas.
    3. Definición de Selectores XPath: Define un diccionario _dict que contiene los selectores XPath necesarios para interactuar con los elementos de la página web.
    4. Inicio del Proceso ETL: Registra el tiempo de inicio y configura un logger para registrar mensajes en un archivo y en la consola.
    5. Obtención y Formateo de Fechas: Calcula la lista de fechas cada tres meses desde el 06/01/2017 hasta el 06/01/2024 y las formatea en el formato 'mm/dd/yyyy'.
    6. Automatización de la Descarga de Archivos: Utiliza Selenium WebDriver para automatizar la descarga de archivos desde un sitio web específico. Se inicializa el WebDriver, se abre la URL, y se llama a la función download_file_between_dates() para realizar la descarga.
    7. Registro de Información y Cálculo del Tiempo: Registra mensajes informativos sobre el inicio y la finalización de la descarga y calcula el tiempo total del proceso ETL.

    El siguiente código utiliza Selenium WebDriver para automatizar la descarga de archivos desde un sitio web específico. A continuación, se explica cada parte del código en detalle:

    Primero, se define la ruta al ejecutable de ChromeDriver y se establece en la variable de entorno webdriver.chrome.driver. Esto es necesario para que Selenium pueda controlar el navegador Chrome. Luego, se itera sobre los pares de fechas consecutivas generadas previamente en date_list.

    Se crea una instancia de webdriver.Chrome() para iniciar una sesión de navegador Chrome. No es necesario proporcionar el argumento executable_path ya que la ruta se ha configurado previamente en las variables de entorno. A continuación, se maximiza la ventana del navegador utilizando el método maximize_window().

    El navegador se dirige a la URL especificada mediante el método get(). Esta URL parece ser una página de inicio de sesión de un sistema CRM basado en SAP. Se imprime un mensaje indicando el rango de fechas para la descarga.

    La función download_file_between_dates() se llama con las fechas inicial y final, así como el controlador del navegador. Esta función se encarga de conectar al sistema C4C, iterar sobre las opciones en un diccionario _dict, y realizar las acciones necesarias para configurar las fechas y descargar el archivo.

    Finalmente, se imprime un mensaje indicando que la descarga se ha completado. Este script automatiza el proceso de descarga de archivos, lo que puede ser útil para tareas repetitivas y ahorrar tiempo en la gestión de datos.
