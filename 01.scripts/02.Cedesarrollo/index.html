    Documentación del Esquema Cedesarrollo¶

    1. Propósito del Esquema¶

    El esquema Cedesarrollo está diseñado para gestionar datos académicos y operativos de una institución educativa dentro de un Data Warehouse. Su estructura dimensional facilita el análisis de:

    • Rendimiento académico (notas, asistencia, deserción).
    • Gestión docente (ausentismo, evaluación de desempeño).
    • Procesos administrativos (matrículas, facturación, horarios).
    • Planificación curricular (programas, módulos, jornadas).

    2. Estructura del Esquema¶

    2.1. Tablas de Dimensiones (DIM_*)¶

    Tabla Descripción Columnas Clave Relaciones
    DIM_ESTUDIANTES Registra estudiantes y sus vínculos con entidades externas (empresas, afiliados). ID_ESTUDIANTE (PK) DIM_EMPRESAS, DIM_AFILIADOS, DIM_BENEFICIARIOS, DIM_APORTANTE_NOAFILIADO
    DIM_PERIODO_ACADEMICO Define períodos académicos (ej. semestres). ID_PERIODO (PK) DIM_UNIDAD
    DIM_PROGRAMA Cataloga programas académicos ofrecidos. ID_PROGRAMA (PK) -
    DIM_JORNADA Jornadas disponibles (diurna, nocturna). ID_JORNADA (PK) -
    DIM_PLAN_CURRICULAR Detalla módulos/cursos de cada programa. ID_MODULO (PK) DIM_PROGRAMA

    2.2. Tablas de Hechos (FACT_*)¶

    Tabla Descripción Métricas Principales Relaciones
    FACT_NOTAS Calificaciones por módulo y período. PRIMER_CORTE, NOTA_FINAL DIM_ESTUDIANTES, DIM_PERIODO_ACADEMICO, DIM_PLAN_CURRICULAR
    FACT_HORARIO Horarios de clases. HORA_INICIO, HORA_FIN DIM_PERIODO_ACADEMICO, DIM_PLAN_CURRICULAR, DIM_JORNADA
    FACT_AUSENTISMO_DOCENTE Ausencias del personal docente. AUSENCIA_HORAS, TIPO_AUSENCIA DIM_PERSONAL, DIM_PERIODO_ACADEMICO
    FACT_DESERCION Deserción estudiantil. TIPO, CAUSA DIM_ESTUDIANTES, DIM_PERIODO_ACADEMICO
    FACT_FACTURACION Transacciones financieras (matrículas, pagos). VALOR_FACTURADO, ADEUDA DIM_TARIFAS_SERVICIOS
    FACT_DESEMPENHO_DOCENTE_DE Evaluación docente por estudiantes. CALIFICACION DIM_PERSONAL, DIM_PERIODO_ACADEMICO

    4. Buenas Prácticas Implementadas¶

    • Valores por Defecto:
      • -1 para claves foráneas no definidas (ej. ID_EMPRESA en DIM_ESTUDIANTES).
      • Registro ID_ESTUDIANTE = -1 para casos de datos faltantes.
    • Integridad Referencial:
      • Claves foráneas en todas las tablas de hechos.
      • Eliminación segura del esquema previo (DROP de constraints antes de tablas).
    • Normalización:
      • Separación clara entre dimensiones (entidades estáticas) y hechos (eventos dinámicos).

DECLARE @sql NVARCHAR(MAX) = '';
SELECT @sql += 'ALTER TABLE ' + QUOTENAME(s.name) + '.' + QUOTENAME(t.name) 
    + ' DROP CONSTRAINT ' + QUOTENAME(f.name) + '; ' 
FROM sys.foreign_keys f
INNER JOIN sys.tables t ON f.parent_object_id = t.object_id
INNER JOIN sys.schemas s ON t.schema_id = s.schema_id
WHERE s.name = 'Cedesarrollo';
EXEC sp_executesql @sql;
-- Eliminar tablas
SET @sql = '';
SELECT @sql += 'DROP TABLE ' + QUOTENAME(s.name) + '.' + QUOTENAME(t.name) + '; ' 
FROM sys.tables t
INNER JOIN sys.schemas s ON t.schema_id = s.schema_id
WHERE s.name = 'Cedesarrollo';
EXEC sp_executesql @sql;
-- Eliminar el esquema
IF EXISTS (SELECT * FROM sys.schemas WHERE name = 'Cedesarrollo')
BEGIN
    DROP SCHEMA Cedesarrollo;
END
GO
