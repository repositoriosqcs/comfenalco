    Documentación SQL para el Esquema Protección¶

    Introducción¶

    El script SQL establece el esquema Protección en el Data Warehouse DWH_COMFENALCO. Este esquema está diseñado para gestionar información relacionada con población, establecimientos educativos, programas, caracterización y actividades relacionadas con protección y visitas.

    Propósito del Script¶

    1. Limpieza del Esquema Existente:

      • Elimina restricciones de claves foráneas, tablas y el esquema si ya existe, para permitir una implementación limpia.

    2. Creación de Estructuras Dimensionales:

      • Tablas Dimensionales (DIM): Almacenan datos estáticos o descriptivos relacionados con población, programas, y características.
      • Tablas de Hechos (FACT): Contienen registros transaccionales y métricas relacionadas con actividades de protección.

    3. Gestión de Integridad:

      • Claves foráneas y primarias para garantizar relaciones consistentes entre dimensiones y hechos.

    Estructura del Modelo¶

    Tablas Dimensionales¶

    1. DIM_POBLACION¶

    Contiene información básica de la población objeto de análisis.

    Columna Tipo Descripción
    ID_POBLACION int Identificador único de la población.
    TIPO_DOCUMENTO nvarchar Tipo de documento de identidad.
    DOCUMENTO nvarchar Número de documento de identidad.
    ID_EMPRESA int Relación con empresas en otros esquemas.
    ID_AFILIADO int Relación con afiliados en otros esquemas.
    ID_BENEFICIARIO int Relación con beneficiarios en otros esquemas.
    ID_APORTANTE int Relación con aportantes en otros esquemas.

    2. DIM_ESTABLECIMIENTO_EDUCATIVO¶

    Información sobre los establecimientos educativos vinculados al sistema.

    Columna Tipo Descripción
    ID_ESTABLECIMIENTO_EDUCATIVO int Identificador único del establecimiento.
    NOMBRE_ESTABLECIMIENTO nvarchar Nombre del establecimiento.
    REPRESENTANTE_LEGAL nvarchar Representante legal del establecimiento.
    DIRECCION nvarchar Dirección del establecimiento.
    MUNICIPIO nvarchar Municipio donde se encuentra el establecimiento.

    3. DIM_PROGRAMA¶

    Programas asociados a actividades de protección.

    Columna Tipo Descripción
    ID_PROGRAMA int Identificador único del programa.
    PROGRAMA nvarchar Nombre del programa.

    4. DIM_CAMPOS_CARACT¶

    Contiene las preguntas utilizadas en caracterizaciones.

    Columna Tipo Descripción
    ID_PREGUNTA int Identificador único de la pregunta.
    PREGUNTA nvarchar Texto de la pregunta.
    OBSERVACIONES nvarchar Observaciones relacionadas con la pregunta.

    5. DIM_PREGUNTAS_EE_JEC¶

    Preguntas específicas para diagnósticos de establecimientos educativos.

    Columna Tipo Descripción
    ID_PREGUNTA int Identificador único de la pregunta.
    PREGUNTA nvarchar Texto de la pregunta.
    ID_PROGRAMA int Relación con la dimensión de programas.

    6. DIM_RESPUESTAS_EE_JEC¶

    Respuestas asociadas a las preguntas de establecimientos educativos.

    Columna Tipo Descripción
    ID_RESPUESTA int Identificador único de la respuesta.
    RESPUESTA nvarchar Texto de la respuesta.
    ID_PREGUNTA int Relación con la dimensión de preguntas.

    Tablas de Hechos¶

    1. FACT_CARACTERIZACION¶

    Registra las respuestas y observaciones relacionadas con la caracterización de la población.

    Columna Tipo Descripción
    ID_CARACTERIZACION int Identificador único del registro de caracterización.
    ID_FECHA int Relación con la dimensión de tiempo (DIM_TIEMPO).
    ID_POBLACION int Relación con la dimensión de población.
    ID_PROGRAMA int Relación con la dimensión de programas.
    ID_PREGUNTA int Relación con la dimensión de preguntas.
    RESPUESTA nvarchar Respuesta de la caracterización.
    OBSERVACIONES nvarchar Observaciones adicionales.

    2. FACT_VENTA¶

    Registro de ventas asociadas a servicios y población.

    Columna Tipo Descripción
    ID_VENTA int Identificador único de la venta.
    ID_FECHA int Relación con la dimensión de tiempo (DIM_TIEMPO).
    ID_POBLACION int Relación con la dimensión de población.
    ID_TARIFA int Relación con tarifas de servicios.
    SERVICIO nvarchar Nombre del servicio.
    COSTO decimal Costo asociado al servicio.
    SUBSIDIO decimal Subsidio aplicado al servicio.
    VALOR_PAGADO_SIN_IMP decimal Valor pagado sin impuestos.

    3. FACT_DESERCION¶

    Registra casos de deserción en establecimientos educativos.

    Columna Tipo Descripción
    ID_REGISTRO int Identificador único del registro de deserción.
    ID_ESTABLECIMIENTO_EDUCATIVO int Relación con la dimensión de establecimientos educativos.
    ID_POBLACION int Relación con la población.
    ANIO_ACADEMICO nvarchar Año académico de la deserción.
    ID_FECHA int Relación con la dimensión de tiempo.
    CAUSA nvarchar Causa de la deserción.

    Buenas Prácticas Implementadas¶

    1. Modularidad:

      • Separación clara entre datos descriptivos (DIM) y registros transaccionales (FACT).

    2. Integridad Referencial:

      • Uso de claves foráneas para garantizar la consistencia entre las tablas.

    3. Escalabilidad:

      • Permite la adición de nuevas dimensiones y hechos sin impactar la estructura existente.

    4. Eficiencia:

      • Índices primarios y relaciones bien definidas para optimizar las consultas.

SELECT @sql += 'ALTER TABLE ' + QUOTENAME(s.name) + '.' + QUOTENAME(t.name) 
    + ' DROP CONSTRAINT ' + QUOTENAME(f.name) + '; ' 
FROM sys.foreign_keys f
INNER JOIN sys.tables t ON f.parent_object_id = t.object_id
INNER JOIN sys.schemas s ON t.schema_id = s.schema_id
WHERE s.name = 'Proteccion';
EXEC sp_executesql @sql;
-- Eliminar tablas
