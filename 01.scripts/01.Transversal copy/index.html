    Documentación SQL para el Esquema Transversal¶

    Introducción¶

    Este script SQL está diseñado para gestionar el esquema Transversal dentro de un Data Warehouse (DWH). Implementa una arquitectura dimensional compuesta por tablas de dimensiones y tablas de hechos, siguiendo las mejores prácticas de modelado dimensional.

    Propósito del Script¶

    1. Eliminación y Creación del Esquema:
    2. Limpieza del esquema existente Transversal eliminando restricciones de claves foráneas, tablas y el esquema en sí.

    3. Reinstalación del esquema Transversal desde cero.

    4. Creación de Tablas Dimensionales y de Hechos:

    5. Tablas de Dimensiones (DIM): Contienen atributos descriptivos y categóricos relacionados con empresas, afiliados, beneficiarios, aportantes no afiliados, entre otros.

    6. Tablas de Hechos (FACT): Contienen métricas y claves foráneas que conectan con las dimensiones, facilitando análisis transaccionales y operativos.

    7. Relaciones entre Tablas:

    8. Configuración de restricciones FOREIGN KEY para garantizar integridad referencial.

    Detalle de las Tablas y Relación Dimensional¶

    Tablas Dimensionales¶

    1. DIM_EMPRESAS¶

    Contiene información básica sobre empresas, incluyendo tipo de documento, razón social, sector y estado.

    Columna Tipo Descripción
    ID_EMPRESA int Identificador único de la empresa.
    RAZON_SOCIAL nvarchar Nombre de la empresa.
    TIPO_DOCUMENTO nvarchar Tipo de documento (NIT, CC).
    DOCUMENTO nvarchar Número del documento.

    2. DIM_AFILIADOS¶

    Contiene datos de los afiliados como nombre, género, estado civil, y afiliación.

    Columna Tipo Descripción
    ID_AFILIADO int Identificador único del afiliado.
    NOMBRE_COMPLETO nvarchar Nombre completo del afiliado.
    GENERO nvarchar Género del afiliado.
    FECHA_AFILIACION nvarchar Fecha de afiliación.

    3. DIM_BENEFICIARIOS¶

    Contiene información de los beneficiarios relacionados con los afiliados.

    Columna Tipo Descripción
    ID_BENEFICIARIO int Identificador único del beneficiario.
    NOMBRE_COMPLETO nvarchar Nombre completo del beneficiario.
    PARENTESCO nvarchar Relación del beneficiario con el titular.

    Diagrama de las Tablas Dimensionales¶

    erDiagram
    DIM_EMPRESAS {
        int ID_EMPRESA PK
        nvarchar RAZON_SOCIAL
        nvarchar TIPO_DOCUMENTO
        nvarchar DOCUMENTO
    }

    DIM_AFILIADOS {
        int ID_AFILIADO PK
        nvarchar NOMBRE_COMPLETO
        nvarchar GENERO
        nvarchar FECHA_AFILIACION
    }
    DIM_BENEFICIARIOS {
        int ID_BENEFICIARIO PK
        nvarchar NOMBRE_COMPLETO
        nvarchar PARENTESCO
    }

    DIM_EMPRESAS ||--o{ DIM_AFILIADOS : "ID_EMPRESA"
    DIM_AFILIADOS ||--o{ DIM_BENEFICIARIOS : "ID_AFILIADO"

    Tablas de Hechos¶

    1. FACT_APORTES_SHR_DET¶

    Registra detalles financieros de aportes realizados por empresas y afiliados.

    Columna Tipo Descripción
    ID_EMPRESA int Relación con la tabla DIM_EMPRESAS.
    ID_AFILIADO int Relación con la tabla DIM_AFILIADOS.
    PERIODO varchar Período de los aportes.
    APORTE_NUEVO numeric Monto del aporte registrado.

    2. FACT_DETALLE_CONTABLE¶

    Registra operaciones contables como ingresos, gastos y resultados financieros.

    Columna Tipo Descripción
    ID_CEBE bigint Relación con la tabla DIM_UNIDADES_ORGANIZATIVAS.
    ID_CUENTA bigint Relación con la tabla DIM_CUENTA_CONTABLE.
    IMPORTE decimal Importe de la operación contable.

    3. FACT_ENCUESTAS¶

    Registra resultados de encuestas relacionadas con servicios y satisfacción.

    Columna Tipo Descripción
    ID_UNIDAD int Relación con la tabla DIM_UNIDAD.
    SERVICIO nvarchar Servicio evaluado en la encuesta.
    CALIFICACION nvarchar Puntuación otorgada en la encuesta.

    Diagrama de las Tablas de Hechos¶

    erDiagram
    FACT_APORTES_SHR_DET {
        int ID_EMPRESA FK
        int ID_AFILIADO FK
        varchar PERIODO
        numeric APORTE_NUEVO
    }

    FACT_DETALLE_CONTABLE {
        bigint ID_CEBE FK
        bigint ID_CUENTA FK
        decimal IMPORTE
    }

    FACT_ENCUESTAS {
        int ID_UNIDAD FK
        nvarchar SERVICIO
        nvarchar CALIFICACION
    }

    DIM_EMPRESAS ||--o{ FACT_APORTES_SHR_DET : "ID_EMPRESA"
    DIM_AFILIADOS ||--o{ FACT_APORTES_SHR_DET : "ID_AFILIADO"
    DIM_UNIDADES_ORGANIZATIVAS ||--o{ FACT_DETALLE_CONTABLE : "ID_CEBE"
    DIM_CUENTA_CONTABLE ||--o{ FACT_DETALLE_CONTABLE : "ID_CUENTA"
    DIM_UNIDAD ||--o{ FACT_ENCUESTAS : "ID_UNIDAD"

    Buenas Prácticas Implementadas¶

    1. Integridad Referencial:

    2. Uso de claves foráneas para garantizar relaciones consistentes entre dimensiones y hechos.

    3. Arquitectura Modular:

    4. Separación de tablas DIM y FACT para optimizar consultas y mantener un diseño limpio.

    5. Documentación:

    6. Las descripciones de columnas y relaciones facilitan el entendimiento del modelo.

    Conclusión¶

    El script implementa un esquema Transversal eficiente para análisis en un Data Warehouse. Las tablas de dimensiones proporcionan atributos descriptivos, mientras que las tablas de hechos registran transacciones y métricas clave. Este modelo es escalable y adaptable a múltiples casos de uso analítico.
