    Indices Cedesarrollo¶

    DIM_ESTUDIANTES¶

    1. Índice único no clúster en el campo DOCUMENTO:
      Garantiza consultas rápidas y únicas basadas en el número de documento, generalmente usado para identificar estudiantes.

    2. Índice no clúster para el campo ID_EMPRESA:
      Optimiza consultas relacionadas con empresas asociadas a los estudiantes.

    3. Índice no clúster para el campo ID_AFILIADO:
      Mejora la eficiencia de las búsquedas relacionadas con afiliados.

    4. Índice no clúster para el campo ID_BENEFICIARIO:
      Facilita las consultas relacionadas con beneficiarios de estudiantes.

    5. Índice no clúster para el campo ID_APORTANTE:
      Aumenta la velocidad de las consultas que usan el campo de aportantes no afiliados.

    6. Índice compuesto no clúster en TIPO_DOCUMENTO y DOCUMENTO:
      Se orienta a optimizar búsquedas que combinen ambos campos, usados frecuentemente como filtro conjunto.

    Justificación

    • Eficiencia en consultas frecuentes: Los índices en campos individuales (ID_EMPRESA, ID_AFILIADO, etc.) y combinados (TIPO_DOCUMENTO, DOCUMENTO) aceleran las operaciones de búsqueda y filtros.
    • Evitar redundancias: Los índices se diseñan para no replicar información ya cubierta por índices primarios o claves foráneas.
    • Mejorar la unicidad: El índice único en DOCUMENTO asegura que este campo no contenga duplicados, previniendo inconsistencias.

    Código

    -- Índice único no clúster en el campo DOCUMENTO
CREATE UNIQUE NONCLUSTERED INDEX IX_DIM_ESTUDIANTES_DOCUMENTO
ON Cedesarrollo.DIM_ESTUDIANTES (DOCUMENTO);
GO

-- Índice no clúster para el campo ID_EMPRESA
CREATE NONCLUSTERED INDEX IX_DIM_ESTUDIANTES_EMPRESA
ON Cedesarrollo.DIM_ESTUDIANTES (ID_EMPRESA);
GO

-- Índice no clúster para el campo ID_AFILIADO
CREATE NONCLUSTERED INDEX IX_DIM_ESTUDIANTES_AFILIADO
ON Cedesarrollo.DIM_ESTUDIANTES (ID_AFILIADO);
GO

-- Índice no clúster para el campo ID_BENEFICIARIO
CREATE NONCLUSTERED INDEX IX_DIM_ESTUDIANTES_BENEFICIARIO
ON Cedesarrollo.DIM_ESTUDIANTES (ID_BENEFICIARIO);
GO

-- Índice no clúster para el campo ID_APORTANTE
CREATE NONCLUSTERED INDEX IX_DIM_ESTUDIANTES_APORTANTE
ON Cedesarrollo.DIM_ESTUDIANTES (ID_APORTANTE);
GO

-- Índice compuesto no clúster en TIPO_DOCUMENTO y DOCUMENTO
CREATE NONCLUSTERED INDEX IX_DIM_ESTUDIANTES_TIPO_DOCUMENTO_DOCUMENTO
ON Cedesarrollo.DIM_ESTUDIANTES (TIPO_DOCUMENTO, DOCUMENTO);
GO

    DIM_JORNADA¶

    1. Índice no clúster para el campo ID_UNIDAD:
      Optimiza las búsquedas y filtros relacionados con la unidad asociada a la jornada.

    2. Índice no clúster para el campo JORNADA:
      Facilita y acelera las consultas basadas en el nombre de la jornada, como búsquedas por texto.

    **JustificaciónDIM_JORNADA

    • ID_UNIDAD: Este campo es clave para unir esta tabla con otras que dependan de la identificación de la unidad. Las búsquedas frecuentes en relaciones o filtros sobre unidades se beneficiarán de este índice.
    • JORNADA: Usualmente, las consultas por nombre son comunes y al tratarse de un campo nvarchar, un índice específico mejora significativamente el rendimiento en búsquedas textuales.

    Código

    -- Índice no clúster para el campo ID_UNIDAD
CREATE NONCLUSTERED INDEX IX_DIM_JORNADA_ID_UNIDAD
ON Cedesarrollo.DIM_JORNADA (ID_UNIDAD);
GO

-- Índice no clúster para el campo JORNADA
CREATE NONCLUSTERED INDEX IX_DIM_JORNADA_NOMBRE
ON Cedesarrollo.DIM_JORNADA (JORNADA);
GO

    DIM_PERIODO_ACADEMICO¶

    1. Índice no clúster para el campo ID_UNIDAD:
      Mejora el rendimiento de consultas relacionadas con las unidades asociadas a los períodos académicos.

    2. Índice no clúster para el campo PERIODO_ACADEMICO:
      Optimiza búsquedas y filtros basados en el nombre o descripción del período académico.

    3. Índice compuesto no clúster para los campos FECHA_INICIO y FECHA_FIN:
      Diseñado para acelerar consultas relacionadas con rangos de fechas, como períodos activos o históricos.

    Justificación

    • ID_UNIDAD: Este campo es clave en las relaciones con la tabla [DIM_UNIDAD]. Un índice en esta columna agiliza consultas de unión o búsquedas relacionadas.
    • PERIODO_ACADEMICO: Al tratarse de un campo descriptivo, puede usarse comúnmente en filtros por texto, por lo que un índice acelera estas búsquedas.
    • FECHA_INICIO y FECHA_FIN: Al ser campos utilizados frecuentemente para filtrar períodos dentro de un rango de tiempo, el índice compuesto asegura eficiencia en este tipo de consultas.

    Código

    -- Índice no clúster para el campo ID_UNIDAD
CREATE NONCLUSTERED INDEX IX_DIM_PERIODO_ACADEMICO_ID_UNIDAD
ON Cedesarrollo.DIM_PERIODO_ACADEMICO (ID_UNIDAD);
GO

-- Índice no clúster para el campo PERIODO_ACADEMICO
CREATE NONCLUSTERED INDEX IX_DIM_PERIODO_ACADEMICO_NOMBRE
ON Cedesarrollo.DIM_PERIODO_ACADEMICO (PERIODO_ACADEMICO);
GO

-- Índice compuesto no clúster para los campos FECHA_INICIO y FECHA_FIN

ON Cedesarrollo.DIM_PERIODO_ACADEMICO (FECHA_INICIO, FECHA_FIN);
