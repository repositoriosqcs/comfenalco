    Documentación del Script SQL para el Esquema Dimensiones Tiempo¶

    Introducción¶

    Este script SQL pertenece a un sistema de manejo de datos en un Data Warehouse, específicamente en el esquema Dwh. Su propósito es eliminar restricciones de claves foráneas en las tablas del esquema, eliminar las tablas y el esquema, y luego crear el esquema y la tabla DIM_TIEMPO. A continuación, se detallan las secciones del código.

    Descripción del Código¶

    1. Contexto Inicial¶

    El script comienza seleccionando la base de datos DWH_COMFENALCO y establece configuraciones iniciales como ANSI_NULLS y QUOTED_IDENTIFIER para asegurar el comportamiento estándar de SQL Server.

    USE DWH_COMFENALCO;
GO
SET ANSI_NULLS ON;
GO
SET QUOTED_IDENTIFIER ON;
GO
    • USE DWH_COMFENALCO: Cambia el contexto de ejecución a la base de datos DWH_COMFENALCO.
    • SET ANSI_NULLS ON: Habilita el tratamiento estándar de valores NULL en operaciones comparativas.
    • SET QUOTED_IDENTIFIER ON: Permite usar nombres de identificadores entre comillas dobles.

    2. Eliminar Restricciones de Claves Foráneas¶

    Se utiliza un procedimiento dinámico para eliminar todas las restricciones de claves foráneas asociadas al esquema Dwh.

    DECLARE @sql NVARCHAR(MAX) = '';
SELECT @sql += 'ALTER TABLE ' + QUOTENAME(s.name) + '.' + QUOTENAME(t.name) 
    + ' DROP CONSTRAINT ' + QUOTENAME(f.name) + '; ' 
FROM sys.foreign_keys f
INNER JOIN sys.tables t ON f.parent_object_id = t.object_id
INNER JOIN sys.schemas s ON t.schema_id = s.schema_id
WHERE s.name = 'Dwh';
EXEC sp_executesql @sql;

    Desglose:¶

    1. sys.foreign_keys: Obtiene información de todas las claves foráneas en las tablas.
    2. sys.tables y sys.schemas: Se utiliza para vincular las tablas a sus esquemas.
    3. Construcción dinámica: El código genera comandos ALTER TABLE para eliminar cada restricción.
    4. EXEC sp_executesql: Ejecuta los comandos generados dinámicamente.

    Motivación: Esto es útil para realizar cambios en las estructuras de las tablas sin conflictos con dependencias existentes.

    3. Eliminar Tablas y Esquema¶

    El script elimina todas las tablas del esquema Dwh y luego elimina el esquema.

    -- Eliminar tablas
SET @sql = '';
SELECT @sql += 'DROP TABLE ' + QUOTENAME(s.name) + '.' + QUOTENAME(t.name) + '; ' 
FROM sys.tables t
INNER JOIN sys.schemas s ON t.schema_id = s.schema_id
WHERE s.name = 'Dwh';
EXEC sp_executesql @sql;

-- Eliminar el esquema
IF EXISTS (SELECT * FROM sys.schemas WHERE name = 'Dwh')
BEGIN
    DROP SCHEMA Dwh;
END
GO

    Desglose:¶

    1. Eliminar tablas: Genera y ejecuta comandos DROP TABLE para eliminar todas las tablas del esquema Dwh.
    2. Eliminar esquema: Verifica si el esquema Dwh existe y lo elimina si es así.

    Motivación: Esto asegura que el esquema Dwh y sus tablas sean eliminados antes de recrearlos.

    4. Crear Esquema y Tabla DIM_TIEMPO¶

    El script crea el esquema Dwh y la tabla DIM_TIEMPO con sus respectivas columnas.

    -- Crear el esquema Dwh
CREATE SCHEMA Dwh;
GO

-- Crear DIM_TIEMPO
CREATE TABLE [Dwh].[DIM_TIEMPO](
    [ID_FECHA] [int] NOT NULL,
    [FECHA] [datetime] NULL,
    [DESC_FECHA] [varchar](50) NULL,
    [ID_SEMANA] [int] NULL,
    [DESC_SEMANA] [varchar](50) NULL,
    [ID_NO_MES] [int] NULL,
    [DESC_NO_MES] [varchar](50) NULL,
    [ID_MES] [int] NULL,
    [DESC_MES] [varchar](50) NULL,
    [DESC_MES_CORTA] [varchar](50) NULL,
    [ID_BIMESTRE] [int] NULL,
    [DESC_BIMESTRE] [varchar](50) NULL,
    [ID_TRIMESTRE] [int] NULL,
    [DESC_TRIMESTRE] [varchar](50) NULL,
    [ID_CUATRIMESTRE] [int] NULL,
    [DESC_CUATRIMESTRE] [varchar](50) NULL
);
GO

    Desglose:¶

    1. Crear esquema: Crea el esquema Dwh.
    2. Crear tabla DIM_TIEMPO: Define la tabla DIM_TIEMPO con sus columnas y tipos de datos.

    Motivación: Esto establece la estructura necesaria para almacenar datos de tiempo en el esquema Dwh.

    Buenas Prácticas Implementadas¶

    1. Uso de Restricciones de Integridad:

      • La eliminación y recreación de restricciones asegura la integridad de los datos.

    2. Automatización en la Eliminación de Restricciones:

      • Permite realizar modificaciones estructurales masivas sin errores manuales.

    3. Configuraciones Iniciales:

      • Se asegura un comportamiento estándar al trabajar con valores NULL y nombres de columnas.

    4. Modularidad y Dinamismo:

      • El script está diseñado para ser modular y dinámico, facilitando modificaciones estructurales complejas.

    5. Uso de Procedimientos Dinámicos:

      • La construcción dinámica de comandos SQL permite una mayor flexibilidad y adaptabilidad.

    6. Definición Clara de Esquemas:

      • Las tablas y sus relaciones están claramente definidas, lo que mejora la comprensión y mantenimiento del código.

    Conclusión¶

    Este script SQL está diseñado para gestionar de manera eficiente la eliminación y creación de estructuras de datos en el esquema Dwh. La automatización y modularidad implementadas aseguran que las operaciones se realicen de manera consistente y sin errores, facilitando el mantenimiento y la evolución del Data Warehouse.
