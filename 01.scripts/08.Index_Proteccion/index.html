    Indices Protección¶

    [DIM_CAMPOS_CARACT]¶

    1. Índice clúster existente para ID_PREGUNTA (PRIMARY KEY):
      Asegura acceso rápido y ordenado mediante el identificador único de cada registro.

    2. Índice no clúster para PREGUNTA:
      Optimiza búsquedas y filtros basados en el texto de la pregunta.

    3. Índice no clúster para OBSERVACIONES:
      Facilita búsquedas y consultas relacionadas con observaciones específicas.

    Justificación

    • ID_PREGUNTA: Clave primaria que asegura un acceso eficiente al registro por su identificador único.
    • PREGUNTA: Campo clave para búsquedas o filtros basados en el texto de la pregunta.
    • OBSERVACIONES: Mejora la velocidad de análisis y búsquedas cuando se necesitan filtros o análisis basados en observaciones específicas.

    Código

    -- Índice clúster ya existente para la clave primaria ID_PREGUNTA
-- (No requiere creación adicional)

-- Índice no clúster para el campo PREGUNTA
CREATE NONCLUSTERED INDEX IX_DIM_CAMPOS_CARACT_PREGUNTA
ON [Proteccion].[DIM_CAMPOS_CARACT] (PREGUNTA);
GO

-- Índice no clúster para el campo OBSERVACIONES
CREATE NONCLUSTERED INDEX IX_DIM_CAMPOS_CARACT_OBSERVACIONES
ON [Proteccion].[DIM_CAMPOS_CARACT] (OBSERVACIONES);
GO

    [DIM_ESTABLECIMIENTO_EDUCATIVO]¶

    1. Índice clúster existente para ID_ESTABLECIMIENTO_EDUCATIVO (PRIMARY KEY):
      Garantiza acceso rápido y ordenado mediante el identificador único de cada registro.

    2. Índice no clúster para COD_ESTABLECIMIENTO_EDUCATIVO:
      Optimiza búsquedas relacionadas con el código del establecimiento educativo.

    3. Índice no clúster para NOMBRE_ESTABLECIMIENTO:
      Facilita búsquedas y filtros por el nombre del establecimiento.

    4. Índice no clúster para DOCUMENTO:
      Mejora consultas relacionadas con documentos específicos de los establecimientos.

    5. Índice no clúster para REPRESENTANTE_LEGAL:
      Acelera búsquedas relacionadas con los representantes legales de los establecimientos.

    6. Índice no clúster para COD_CIUDAD:
      Optimiza análisis y filtros basados en la ciudad de ubicación del establecimiento.

    Justificación

    • ID_ESTABLECIMIENTO_EDUCATIVO: Clave primaria para garantizar un acceso rápido y ordenado.
    • COD_ESTABLECIMIENTO_EDUCATIVO: Es un identificador alternativo usado frecuentemente en consultas.
    • NOMBRE_ESTABLECIMIENTO: Se utiliza para búsquedas textuales en análisis y reportes.
    • DOCUMENTO: Permite búsquedas específicas para identificar establecimientos.
    • REPRESENTANTE_LEGAL: Optimiza consultas basadas en responsables legales.
    • COD_CIUDAD: Mejora análisis agrupados por ubicación geográfica.

    Código

    -- Índice clúster ya existente para la clave primaria ID_ESTABLECIMIENTO_EDUCATIVO
-- (No requiere creación adicional)

-- Índice no clúster para el campo COD_ESTABLECIMIENTO_EDUCATIVO
CREATE NONCLUSTERED INDEX IX_DIM_ESTABLECIMIENTO_EDUCATIVO_COD_ESTABLECIMIENTO
ON [Proteccion].[DIM_ESTABLECIMIENTO_EDUCATIVO] (COD_ESTABLECIMIENTO_EDUCATIVO);

-- Índice no clúster para el campo NOMBRE_ESTABLECIMIENTO

GO

-- Índice no clúster para el campo DOCUMENTO

ON [Proteccion].[DIM_ESTABLECIMIENTO_EDUCATIVO] (DOCUMENTO);
GO

-- Índice no clúster para el campo REPRESENTANTE_LEGAL

    [DIM_POBLACION]¶

    1. Índice clúster existente para ID_POBLACION (PRIMARY KEY):
      Garantiza acceso rápido y ordenado mediante el identificador único de cada registro.

    2. Índice no clúster para TIPO_DOCUMENTO y DOCUMENTO:
      Optimiza consultas basadas en la combinación del tipo de documento y su número.

    3. Índice no clúster para ID_EMPRESA:
      Mejora búsquedas relacionadas con la empresa asociada.

    4. Índice no clúster para ID_AFILIADO:
      Facilita consultas relacionadas con los afiliados.

    5. Índice no clúster para ID_BENEFICIARIO:
      Acelera análisis relacionados con los beneficiarios.

    6. Índice no clúster para ID_APORTANTE:
      Optimiza búsquedas relacionadas con aportantes específicos.

    Justificación

    • ID_POBLACION: Clave primaria que asegura un acceso eficiente al registro por su identificador único.
    • TIPO_DOCUMENTO y DOCUMENTO: La combinación de estos campos permite búsquedas rápidas de individuos específicos.
    • ID_EMPRESA: Facilita el análisis de datos a nivel empresarial.
    • ID_AFILIADO: Optimiza consultas relacionadas con individuos afiliados.
    • ID_BENEFICIARIO: Mejora el acceso a datos de beneficiarios en reportes y análisis.
    • ID_APORTANTE: Permite búsquedas rápidas relacionadas con quienes realizan aportes.

    Código

    -- Índice clúster ya existente para la clave primaria ID_POBLACION
-- (No requiere creación adicional)

-- Índice compuesto no clúster para los campos TIPO_DOCUMENTO y DOCUMENTO
CREATE NONCLUSTERED INDEX IX_DIM_POBLACION_TIPO_DOCUMENTO_DOCUMENTO
ON [Proteccion].[DIM_POBLACION] (TIPO_DOCUMENTO, DOCUMENTO);
GO

ON [Proteccion].[DIM_POBLACION] (ID_EMPRESA);
GO

-- Índice no clúster para el campo ID_AFILIADO

ON [Proteccion].[DIM_POBLACION] (ID_AFILIADO);

-- Índice no clúster para el campo ID_BENEFICIARIO

    [FACT_CARACTERIZACION]¶

    1. Índice clúster existente para ID_CARACTERIZACION (PRIMARY KEY):
      Garantiza un acceso rápido y ordenado mediante el identificador único de cada registro.

    2. Índice no clúster para ID_FECHA:
      Mejora el rendimiento de consultas que utilizan la fecha para análisis o relaciones con otras tablas.

    3. Índice no clúster para ID_POBLACION:
      Optimiza consultas relacionadas con la población asociada a cada caracterización.

    4. Índice no clúster para ID_PROGRAMA:
      Acelera búsquedas y relaciones con programas específicos.

    5. Índice no clúster para ID_PREGUNTA:
      Mejora el rendimiento de uniones o filtros relacionados con preguntas específicas.

    6. Índice no clúster para el campo RESPUESTA:
      Optimiza búsquedas textuales relacionadas con las respuestas.

    7. Índice compuesto no clúster para ID_FECHA y ID_PROGRAMA:
      Diseñado para consultas frecuentes que combinen fechas y programas en filtros o análisis.

    Justificación

    • ID_CARACTERIZACION: Clave primaria que asegura un acceso eficiente y ordenado a los registros.
    • ID_FECHA: Frecuentemente utilizado para análisis basados en tiempo o relaciones con la tabla de tiempo.
    • ID_POBLACION: Como clave foránea, se utiliza para uniones y análisis relacionados con datos de población.
    • ID_PROGRAMA: Fundamental para consultas que filtran o agrupan por programas asociados.
    • ID_PREGUNTA: Importante para relacionar y analizar preguntas específicas.
    • RESPUESTA: Este campo puede ser utilizado en búsquedas textuales o análisis relacionados con respuestas específicas.
    • Índice compuesto para ID_FECHA y ID_PROGRAMA: Facilita análisis multidimensionales en escenarios con múltiples dimensiones (fecha y programa).

    Código

    -- Índice clúster ya existente para la clave primaria ID_CARACTERIZACION
-- (No requiere creación adicional)

-- Índice no clúster para el campo ID_FECHA

    2. Índice no clúster para ID_ESTABLECIMIENTO_EDUCATIVO:
      Optimiza las consultas relacionadas con establecimientos educativos.

    3. Índice no clúster para ID_POBLACION:
      Mejora el rendimiento de búsquedas o uniones relacionadas con población.

    4. Índice no clúster para ID_PROGRAMA:
      Acelera las consultas relacionadas con programas.

    5. Índice no clúster para ID_FECHA:
      Mejora el rendimiento en análisis basados en tiempo y uniones con la tabla de tiempo.

    6. Índice no clúster para el campo ANIO_ACADEMICO:
      Optimiza consultas que analizan registros por año académico.

    7. Índice no clúster para el campo CAUSA:
      Acelera búsquedas y agrupaciones relacionadas con las causas de deserción.

    8. Índice compuesto no clúster para ID_FECHA y ID_ESTABLECIMIENTO_EDUCATIVO:
      Diseñado para consultas que filtran por tiempo y establecimiento educativo.

    • ID_PROGRAMA: Usado en consultas para agrupar o filtrar datos por programa.
    • ID_FECHA: Clave foránea que permite uniones rápidas con la dimensión de tiempo para análisis temporales.
    • ANIO_ACADEMICO y CAUSA: Campos comunes en análisis y agrupaciones específicas sobre deserción.
    • Índice compuesto para ID_FECHA y ID_ESTABLECIMIENTO_EDUCATIVO: Acelera consultas multidimensionales que incluyen filtros por tiempo y establecimiento.
