    DIM_CURSO¶

    1. Índice clúster existente para ID_CURSO (PRIMARY KEY):
      Asegura acceso rápido y ordenado a través del identificador único de cada curso.

    2. Índice no clúster para DESC_CURSO:
      Facilita búsquedas relacionadas con descripciones de cursos.

    3. Índice compuesto no clúster para ESTADO_REGISTRO y FECHA_CREACION:
      Optimiza consultas relacionadas con el estado de los cursos y su fecha de creación.

    Justificación

    • ID_CURSO: Clave primaria, asegura un acceso rápido.
    • DESC_CURSO: Mejora el rendimiento en búsquedas textuales.
    • ESTADO_REGISTRO y FECHA_CREACION: Mejora análisis combinados por estado y fecha.

    Código

    -- Índice clúster ya existente para la clave primaria ID_CURSO
-- (No requiere creación adicional)

-- Índice no clúster para el campo DESC_CURSO
CREATE NONCLUSTERED INDEX IX_DIM_CURSO_DESC_CURSO
ON [Colegio].[DIM_CURSO] (DESC_CURSO);
GO

-- Índice compuesto no clúster para ESTADO_REGISTRO y FECHA_CREACION
CREATE NONCLUSTERED INDEX IX_DIM_CURSO_ESTADO_FECHA
ON [Colegio].[DIM_CURSO] (ESTADO_REGISTRO, FECHA_CREACION);
