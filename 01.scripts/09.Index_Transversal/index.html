    Indices Transversal¶

    [DIM_AFILIADOS]¶

    Justificación

    1. Índice clúster existente para ID_AFILIADO

      • La clave primaria garantiza un acceso eficiente y ordenado a los registros.

    2. Índice no clúster para NUMERO_DOCUMENTO

      • Este índice es esencial para las búsquedas frecuentes por el número de documento, ya que suele ser un identificador único en sistemas de afiliados.

    3. Índice compuesto no clúster para TIPO_DOCUMENTO y NUMERO_DOCUMENTO

      • Optimiza las consultas que combinan tipo y número de documento para validar registros.

    4. Índice no clúster para ID_EMPRESA

      • Facilita las búsquedas y uniones con la dimensión de empresas.

    5. Índice no clúster para CORREO

      • Mejora las búsquedas por correo electrónico, a menudo usado como identificador único o para comunicación.

    6. Índice compuesto no clúster para PRIMER_APELLIDO y SEGUNDO_APELLIDO

      • Optimiza las consultas de búsqueda y agrupación por apellidos, comunes en análisis demográficos.

    7. Índice no clúster para GENERO

      • Mejora el rendimiento en análisis de género, común en reportes de afiliados.

    8. Índice compuesto no clúster para ESTADO_AFILIACION y FECHA_AFILIACION

      • Facilita el análisis y la segmentación por estado y fecha de afiliación.

    9. Índice compuesto no clúster para FEC_DESDE y FEC_HASTA

      • Optimiza consultas de rangos temporales, como la validez de afiliación.

    10. Índice no clúster para ESTADOREGISTRO

      • Mejora el rendimiento de consultas que filtran registros activos o inactivos.

    Código

    -- Índice clúster ya existente para la clave primaria ID_AFILIADO
-- (No requiere creación adicional)

-- Índice no clúster para el campo NUMERO_DOCUMENTO
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_NUMERO_DOCUMENTO
ON [Transversal].[DIM_AFILIADOS] (NUMERO_DOCUMENTO);
GO

-- Índice compuesto no clúster para los campos TIPO_DOCUMENTO y NUMERO_DOCUMENTO
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_TIPO_Y_DOCUMENTO
ON [Transversal].[DIM_AFILIADOS] (TIPO_DOCUMENTO, NUMERO_DOCUMENTO);
GO

-- Índice no clúster para el campo ID_EMPRESA
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_ID_EMPRESA
ON [Transversal].[DIM_AFILIADOS] (ID_EMPRESA);
GO

-- Índice no clúster para el campo CORREO
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_CORREO
ON [Transversal].[DIM_AFILIADOS] (CORREO);
GO

-- Índice compuesto no clúster para los campos PRIMER_APELLIDO y SEGUNDO_APELLIDO
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_APELLIDOS
ON [Transversal].[DIM_AFILIADOS] (PRIMER_APELLIDO, SEGUNDO_APELLIDO);
GO

-- Índice no clúster para el campo GENERO
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_GENERO
ON [Transversal].[DIM_AFILIADOS] (GENERO);
GO

-- Índice compuesto no clúster para los campos ESTADO_AFILIACION y FECHA_AFILIACION
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_ESTADO_FECHA
ON [Transversal].[DIM_AFILIADOS] (ESTADO_AFILIACION, FECHA_AFILIACION);
GO

-- Índice compuesto no clúster para los campos FEC_DESDE y FEC_HASTA
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_FECHAS
ON [Transversal].[DIM_AFILIADOS] (FEC_DESDE, FEC_HASTA);
GO

-- Índice no clúster para el campo ESTADOREGISTRO
CREATE NONCLUSTERED INDEX IX_DIM_AFILIADOS_ESTADOREGISTRO
ON [Transversal].[DIM_AFILIADOS] (ESTADOREGISTRO);

    [FACT_PRESUPUESTO]¶

    Justificación

    1. Índice no clúster para ID_FECHA

      • Mejora la eficiencia en consultas que analizan presupuestos basados en periodos de tiempo.

    2. Índice no clúster para ID_CUENTA

      • Optimiza la búsqueda y análisis de presupuestos asociados a cuentas contables específicas.

    3. Índice no clúster para ID_CEBE

      • Aumenta el rendimiento en consultas relacionadas con unidades organizativas.

    4. Índice compuesto no clúster para ID_TIPO_PRESUPUESTO y SEGMENT

      • Facilita el análisis de presupuestos según el tipo y segmento.

    5. Índice no clúster para campos relacionados con valores (INGRESOS, GASTOS, COSTOS)

      • Mejora el rendimiento de cálculos financieros basados en estos campos.
