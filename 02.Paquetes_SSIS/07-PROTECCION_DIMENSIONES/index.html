    PROTECCION_DIMENSIONES¶

    El paquete SSIS "07-PROTECCION_DIMENSIONES" está diseñado para gestionar flujos ETL que procesan datos críticos relacionados con dimensiones de protección, como preguntas y respuestas educativas, características de campos y establecimientos educativos. Este paquete asegura un flujo de trabajo eficiente, consolidando datos desde múltiples fuentes y garantizando su integridad en el Data Warehouse DWH_COMFENALCO.

    Propósito del Paquete¶

    El objetivo principal del paquete es transformar y cargar datos clave para dimensiones educativas y de protección social. Esto incluye validar información de campos, programas y encuestas, asegurando que los datos consolidados sean precisos, consistentes y preparados para análisis estratégicos.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes utilizadas:
        • Archivos Excel: Información sobre preguntas, respuestas, programas y características de campos.
        • Bases de Datos: Validación de registros en tablas maestras como DIM_PROGRAMA, DIM_PREGUNTAS_EE_JEC y DIM_ESTABLECIMIENTO_EDUCATIVO.
      • Conexiones configuradas:
        • Administradores OLE DB y ADO.NET.

    2. Transformación de Datos:

      • Conversión de Datos (Data Conversion):
        • Asegura compatibilidad de tipos entre columnas de entrada y destino.
      • Validaciones (Lookup):
        • Garantiza consistencia mediante búsquedas en tablas maestras.
      • Clasificación (Conditional Split):
        • Filtra registros válidos y redirige no válidos.

    3. Carga de Datos en el Data Warehouse:

      • Tablas destino:
        • DIM_CAMPOS_CARACT
        • DIM_PROGRAMA
        • DIM_PREGUNTAS_EE_JEC
        • DIM_RESPUESTAS_EE_JEC
        • DIM_ESTABLECIMIENTO_EDUCATIVO
      • Configuración:
        • Inserciones masivas habilitadas para optimizar la carga.

    4. Automatización y Scripts:

      • Integración de scripts Python para automatizar descargas desde SharePoint y procesar datos.

    Tablas y Columnas Clave¶

    1. DIM_CAMPOS_CARACT:

      • ID_CARACTERISTICA: Identificador único.
      • PREGUNTA: Texto de la pregunta.
      • OBSERVACIONES: Información adicional.

    2. DIM_PROGRAMA:

      • ID_PROGRAMA: Identificador del programa.
      • NOMBRE_PROGRAMA: Descripción del programa.

    3. DIM_PREGUNTAS_EE_JEC:

      • ID_PREGUNTA: Identificador único.
      • PREGUNTA: Texto de la pregunta.
      • ID_PROGRAMA: Relación con el programa.

    4. DIM_RESPUESTAS_EE_JEC:

      • ID_RESPUESTA: Identificador único.
      • RESPUESTA: Texto de la respuesta.
      • ID_PREGUNTA: Relación con la pregunta.

    5. DIM_ESTABLECIMIENTO_EDUCATIVO:

      • ID_ESTABLECIMIENTO: Identificador único.
      • NOMBRE_ESTABLECIMIENTO: Nombre del establecimiento.
      • REPRESENTANTE_LEGAL: Persona responsable.

    Componentes¶

    Componente Ejecución ETLs¶

    Descripción General¶

    Este componente ejecuta un proceso Python encargado de la conexión y extracción de datos desde SharePoint para la carpeta específica Protección, ubicada dentro de la estructura de archivos del proyecto. Es parte del flujo de automatización de ETL en un paquete de SSIS.

    Componentes del Flujo de Datos¶

    • Descripción: Este componente utiliza una tarea de tipo Execute Process Task en SSIS para invocar un script Python que se encuentra dentro del entorno virtual configurado para el proyecto.

    • Propiedades:

      • Executable: 

        @[$Project::Working_Directory]+ "COMFENALCO_EDUCACION\\01.Scripts\\Utils\\env\\Scripts\\python.exe"
        Especifica la ruta al ejecutable de Python dentro del entorno virtual del proyecto.

      • WorkingDirectory: 

        @[$Project::Working_Directory] +"COMFENALCO_EDUCACION\\01.Scripts\\03.Archivos_Manuales\\03.Proteccion"
        Define el directorio de trabajo donde se encuentra el script Python.

      • Arguments: 

        SharePoint_Connection_Proteccion.py
        Indica el script que será ejecutado para conectar y procesar los datos de SharePoint.

      • TimeOut: 

        @[$Project::Tiempo_Espera_Segundos]
        Configura el tiempo máximo de espera en segundos para que el proceso se complete, definido en 600 segundos (10 minutos).

    Diagrama de Flujo del Proceso¶

    El siguiente diagrama ilustra el flujo de ejecución para el componente Ejecución ETLs:

    A[Inicio del proceso] --> B[Obtener ruta del ejecutable Python]
    B --> C[Definir directorio de trabajo]

    Componente Data DIM_CAMPOS_CARACT¶

    Descripción General¶

    El componente Data DIM_CAMPOS_CARACT forma parte de un flujo ETL desarrollado en SSIS para cargar y transformar datos relacionados con las características de campos en la tabla DIM_CAMPOS_CARACT dentro del esquema Protección. Este proceso incluye la lectura de datos desde un archivo Excel, conversión de datos, validación mediante un componente de tipo Lookup, y finalmente, la carga en la base de datos de destino.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos (Excel Source):

      • Descripción: Conecta y extrae datos desde una hoja llamada Sheet1$ en un archivo Excel, con columnas PREGUNTA y OBSERVACIONES.
      • Propiedades:
        • AccessMode: 0 (por tabla u objeto).
        • Hoja seleccionada: Sheet1$.

    2. Conversión de Datos (Data Conversion):

      • Descripción: Convierte los datos extraídos a un formato compatible con los siguientes componentes del flujo.
      • Columnas Convertidas:
        • PREGUNTA → Copy of PREGUNTA.
        • OBSERVACIONES → Copy of OBSERVACIONES.

    3. Validación (Lookup):

      • Descripción: Valida y enriquece los datos mediante la búsqueda de correspondencias en la tabla destino DIM_CAMPOS_CARACT.
      • SQL de Consulta: 
        SELECT * 
FROM [Proteccion].[DIM_CAMPOS_CARACT]
WHERE [PREGUNTA] = ?
      • Columnas Salida:
        • PREGUNTA_.
        • OBSERVACIONES_.
        • ID_PREGUNTA.

    4. Destino (ADO NET Destination):

      • Descripción: Carga los datos procesados y validados en la tabla DIM_CAMPOS_CARACT dentro del esquema Protección.
      • Propiedades:
        • BatchSize: 0 (tamaño predeterminado).
        • CommandTimeout: 30 segundos.
        • UseBulkInsertWhenPossible: true (usa inserción masiva si es posible).

    Diagrama de Flujo¶

    El siguiente diagrama ilustra el flujo completo de datos del componente Data DIM_CAMPOS_CARACT:

    graph TD
    A[Inicio] --> B[Lectura desde Excel Source]
    B --> C[Conversión de datos]
    C --> D[Validación mediante Lookup]
    D --> E{¿Coincidencia encontrada?}
    E -- Sí --> F[Cargar datos en ADO NET Destination]
    E -- No --> G[Registrar datos no coincidentes]
    F --> H[Fin]
    G --> H

    Componente Data DIM_PREGUNTAS_EE_JEC¶

    Descripción General¶

    Este componente gestiona un flujo de datos ETL que extrae preguntas relacionadas con ejecución de eventos educativos desde un archivo Excel y las procesa para ser almacenadas en la tabla DIM_PREGUNTAS_EE_JEC del esquema Protección. Incluye validaciones, transformaciones, y manejo de datos no coincidentes.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos (Excel Source):

      • Descripción: Extrae los datos desde un archivo Excel (Sheet1$).
      • Columnas de Entrada:
        • PROGRAMA
        • PREGUNTA
      • Propiedades:
        • AccessMode: 0 (lectura directa desde una hoja).
        • Conexión: Administrador de conexiones Excel_Connection_Dim_Preguntas_JEC.

    2. Validación de Programas (Lookup 1):

      • Descripción: Valida los programas en los datos de entrada con los registros existentes en DIM_PROGRAMA.
      • SQL de Consulta: 
        SELECT * 
FROM [Proteccion].[DIM_PROGRAMA]
WHERE [PROGRAMA] = ?
      • Salida:
        • Coincidencia encontrada: Asocia el ID_PROGRAMA.
        • Sin coincidencia: Redirige a la etapa de conversión de datos.

    3. Conversión de Datos (Data Conversion):

      • Descripción: Transforma las columnas extraídas (PREGUNTA) a un formato compatible con la base de datos destino.
      • Columnas Convertidas:
        • PREGUNTA → Copy of PREGUNTA.

    4. Validación de Preguntas (Lookup):

      • Descripción: Verifica si las preguntas existen en la tabla destino DIM_PREGUNTAS_EE_JEC.
      • SQL de Consulta: 
        SELECT * 
FROM [Proteccion].[DIM_PREGUNTAS_EE_JEC]
WHERE [PREGUNTA] = ?

    5. Destino de Datos (ADO NET Destination):

      • Descripción: Carga los datos procesados en la tabla DIM_PREGUNTAS_EE_JEC.
      • Propiedades:
        • BatchSize: 0 (valor predeterminado).
        • CommandTimeout: 30 segundos.
        • UseBulkInsertWhenPossible: true (usa inserción masiva).

    Componente Cargar_DIM_SEDES¶

    Descripción General¶

    El componente Cargar_DIM_SEDES es una tarea de tipo Execute SQL Task en SSIS. Su función principal es limpiar (truncar) la tabla de staging que contiene información de población de protección antes de realizar nuevas cargas de datos. Esto asegura que la tabla esté vacía y preparada para la inserción de datos actualizados.

    Detalles Técnicos¶

    • Tipo de Tarea: Execute SQL Task
    • Nombre del Componente: Cargar_DIM_SEDES

    Componente Data DIM_POBLACION TO_STAGE_AREA¶

    Descripción General¶

    El componente Data DIM_POBLACION TO_STAGE_AREA es un flujo de datos (Data Flow Task) dentro de un paquete SSIS que se encarga de extraer información de población desde un archivo Excel, convertir el formato de algunas columnas y posteriormente cargar los datos transformados en una tabla de staging ubicada en el esquema Transversal. En este caso, la tabla destino es [STAGE_AREA].[Transversal].[DIM_POBLACION_PROTECCION]. Este proceso forma parte de la preparación de los datos antes de su integración final en el Data Warehouse.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos (Excel Source 1)

      • Descripción: Conecta y extrae datos desde un archivo Excel.
      • Propiedades:
        • Hoja seleccionada: Sheet1$
        • Se extraen las columnas: TIPO_DOCUMENTO y DOCUMENTO.

    2. Conversión de Datos (Data Conversion 1)

      • Descripción: Transforma los datos extraídos para ajustarse a los requerimientos del destino.
      • Columnas Convertidas:
        • La columna DOCUMENTO se convierte a un nuevo campo denominado Copy of DOCUMENTO con un tamaño de 40 caracteres.
        • La columna TIPO_DOCUMENTO se convierte a Copy of TIPO_DOCUMENTO con un tamaño de 20 caracteres.
      • Propiedades adicionales:
        • Se activa la opción de conversión sin uso de FastParse (FastParse = false) para asegurar la correcta transformación de los datos.

    3. Destino de Datos (ADO NET Destination: Destino de ADO NET 1)

      • Descripción: Carga los datos procesados en la tabla de staging [STAGE_AREA].[Transversal].[DIM_POBLACION_PROTECCION].
      • Propiedades:
        • TableOrViewName: "Transversal"."DIM_POBLACION_PROTECCION"
        • BatchSize: 0 (usa el tamaño predeterminado del búfer interno de SSIS).
        • CommandTimeout: 30 segundos.
        • UseBulkInsertWhenPossible: true para mejorar el rendimiento mediante inserciones masivas.

    Componente Data DIM_POBLACION¶

    Descripción General¶

    Este componente procesa y carga información sobre la población en la tabla DIM_POBLACION del esquema Proteccion. El flujo de datos extrae registros desde un archivo Excel, realiza conversiones de datos y luego inserta la información en el destino mediante un ADO NET Destination. Además, se incorpora un componente que obtiene datos desde la fuente mediante una consulta SQL que utiliza una expresión CTE (Common Table Expression) para transformar y validar los datos de población provenientes de la etapa (STAGE_AREA).

    Componentes del Flujo de Datos¶

    1. Fuente de Datos (Excel Source 1)

      • Descripción: Extrae datos desde un archivo Excel, leyendo la hoja Sheet1$ y capturando las columnas TIPO_DOCUMENTO y DOCUMENTO.
      • Propiedades:
        • OpenRowset: Sheet1$
        • AccessMode: 0 (lectura directa desde la hoja)
        • Conexión: Excel_Connection_Dim_Poblacion

    2. Conversión de Datos (Data Conversion 1)

      • Descripción: Convierte las columnas extraídas para ajustarlas a los requerimientos del destino.
      • Columnas Convertidas:
        • DOCUMENTO → Copy of DOCUMENTO (convertido a wstr con longitud 40)
        • TIPO_DOCUMENTO → Copy of TIPO_DOCUMENTO (convertido a wstr con longitud 20)

    3. Fuente de Datos SQL (poblacion)

      • Descripción: Utiliza una consulta SQL que define la CTE DIM_POBLACION_PROTECCION para transformar y validar los registros provenientes de la etapa STAGE_AREA. Se aplican múltiples LEFT JOIN a tablas de referencia (DIM_EMPRESAS, DIM_AFILIADOS, DIM_BENEFICIARIOS y DIM_APORTANTE_NOAFILIADO) para obtener los identificadores correspondientes. Luego, se filtran los registros que aún no han sido cargados en la tabla destino DIM_POBLACION (de modo que los registros duplicados se descarten).
      • SQL: 
        WITH DIM_POBLACION_PROTECCION as (
    SELECT 
        pp.TIPO_DOCUMENTO, 
        pp.DOCUMENTO,
        CASE WHEN e.ID_EMPRESA IS NULL THEN -1 ELSE e.ID_EMPRESA END AS ID_EMPRESA,
        CASE WHEN a.ID_AFILIADO IS NULL THEN -1 ELSE a.ID_AFILIADO END AS ID_AFILIADO,
        CASE WHEN b.ID_BENEFICIARIO IS NULL THEN -1 ELSE b.ID_BENEFICIARIO END AS ID_BENEFICIARIO,
        CASE WHEN an.ID_APORTANTE IS NULL THEN -1 ELSE an.ID_APORTANTE END AS ID_APORTANTE
    FROM [STAGE_AREA].[Transversal].[DIM_POBLACION_PROTECCION] pp
    LEFT JOIN [DWH_COMFENALCO].[Transversal].[DIM_EMPRESAS] e
        ON pp.TIPO_DOCUMENTO = e.COD_TIPO_DOCUMENTO AND pp.DOCUMENTO = e.DOCUMENTO
    LEFT JOIN [DWH_COMFENALCO].[Transversal].[DIM_AFILIADOS] a
        ON pp.TIPO_DOCUMENTO = a.COD_TIPO_DOCUMENTO AND pp.DOCUMENTO = a.NUMERO_DOCUMENTO
    LEFT JOIN [DWH_COMFENALCO].[Transversal].[DIM_BENEFICIARIOS] b
        ON pp.TIPO_DOCUMENTO = b.COD_TIPO_DOCUMENTO AND pp.DOCUMENTO = b.NUMERO_DOCUMENTO
    LEFT JOIN [DWH_COMFENALCO].[Transversal].[DIM_APORTANTE_NOAFILIADO] an
        ON pp.TIPO_DOCUMENTO = an.COD_TIPO_DOCUMENTO AND pp.DOCUMENTO = an.DOCUMENTO
)
SELECT pp.* 
FROM DIM_POBLACION_PROTECCION pp
LEFT JOIN [DWH_COMFENALCO].[Proteccion].[DIM_POBLACION] ppp
    ON pp.TIPO_DOCUMENTO = ppp.TIPO_DOCUMENTO
    AND pp.DOCUMENTO = ppp.DOCUMENTO
    AND pp.ID_EMPRESA = ppp.ID_EMPRESA
    AND pp.ID_AFILIADO = ppp.ID_AFILIADO
    AND pp.ID_BENEFICIARIO = ppp.ID_BENEFICIARIO
    AND pp.ID_APORTANTE = ppp.ID_APORTANTE
WHERE ppp.TIPO_DOCUMENTO IS NULL
      • Conexión: Utiliza el administrador de conexiones DWH_COMFENALCO.

    4. Destino de Datos (ADO NET Destination)

      • Descripción: Inserta los datos transformados y validados en la tabla DIM_POBLACION del esquema Proteccion.
      • Propiedades:
        • TableOrViewName: "Proteccion"."DIM_POBLACION"
        • BatchSize: 0
        • CommandTimeout: 30 segundos
        • UseBulkInsertWhenPossible: true
      • Conexión: DWH_COMFENALCO_Destino

        • DIRECCION
        • MUNICIPIO
      • Propiedades:
        • Conexión: Excel_Connection_Dim_EE.

        • REPRESENTANTE_LEGAL
