    CEDESARROLLO_DIMENSIONES¶

    El paquete SSIS "05-CEDESARROLLO_DIMENSIONES" está diseñado para procesar, transformar y cargar datos esenciales relacionados con dimensiones clave como períodos académicos, programas educativos, jornadas, y población matriculada. Este paquete asegura la calidad e integridad de los datos consolidados en el Data Warehouse DWH_COMFENALCO, proporcionando una base sólida para análisis estratégicos y toma de decisiones.

    Propósito del Paquete¶

    El propósito principal es gestionar, transformar y cargar datos relacionados con dimensiones transversales del ámbito educativo y administrativo, garantizando su consistencia y disponibilidad para análisis detallados en plataformas de inteligencia de negocios.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes principales:
        • Bases de Datos:
        • DIM_PERIODO_ACADEMICO, DIM_PROGRAMA, DIM_JORNADA, DIM_POBLACION_MATRICULA.
        • Archivos Excel:
        • Información de jornadas y programas.
      • Herramientas utilizadas:
        • Conexiones ADO.NET y OLE DB.

    2. Transformación de Datos:

      • Validaciones (Lookup):
        • Asegura consistencia mediante búsquedas en tablas maestras como DIM_ESTUDIANTES y DIM_TIEMPO.
      • Conversión de Tipos (Data Conversion):
        • Garantiza compatibilidad con las tablas destino.
      • Columnas Derivadas (Derived Column):
        • Genera identificadores y valores predeterminados.

    3. Carga de Datos en el Data Warehouse:

      • Tablas procesadas:
        • DIM_PERIODO_ACADEMICO
        • DIM_PROGRAMA
        • DIM_JORNADA
        • DIM_POBLACION_MATRICULA
      • Configuración:
        • Inserciones masivas habilitadas para maximizar el rendimiento.

    4. Automatización y Scripts:

      • Scripts Python para automatizar tareas de descarga y validación.

    Tablas y Columnas Clave¶

    1. DIM_PERIODO_ACADEMICO:

      • ID_PERIODO, ID_UNIDAD, PERIODO_ACADEMICO, FECHA_INICIO, FECHA_FIN.

    2. DIM_PROGRAMA:

      • ID_PROGRAMA, NOMBRE_PROGRAMA, DESCRIPCION.

    3. DIM_JORNADA:

      • ID_JORNADA, NOMBRE_JORNADA, HORARIO.

    4. DIM_POBLACION_MATRICULA:

      • ID_POBLACION, NOMBRE, DOCUMENTO, GENERO, FECHA_NACIMIENTO.

    SQL: Agregar -1 a Periodo_Academico¶

    Descripción General¶

    Esta tarea de tipo "Ejecutar SQL" realiza la limpieza y la inserción de un registro especial en la tabla [DIM_PERIODO_ACADEMICO] en la base de datos [Cedesarrollo]. Este registro representa un valor por defecto (-1) para períodos académicos sin datos.

    Detalles de la Configuración

    • Nombre del Componente: Agregar -1 a Periodo_Academico
    • Descripción: Tarea Ejecutar SQL
    • Conexión: {C2A27DDB-56C2-4889-8A4B-7AA7124DFFD7}
    • Sentencia SQL:
    TRUNCATE TABLE [DWH_COMFENALCO].[Cedesarrollo].[DIM_PERIODO_ACADEMICO];

-- Insertar un registro para el periodo sin datos
SET IDENTITY_INSERT [Cedesarrollo].[DIM_PERIODO_ACADEMICO] ON;

INSERT INTO [Cedesarrollo].[DIM_PERIODO_ACADEMICO] (
    [ID_PERIODO], 
    [ID_UNIDAD], 
    [PERIODO_ACADEMICO], 
    [FECHA_INICIO], 
    [FECHA_FIN]
) 
VALUES (
    -1, 
    5, 
    'Sin datos', 
    NULL, 
    NULL
);

SET IDENTITY_INSERT [Cedesarrollo].[DIM_PERIODO_ACADEMICO] OFF;
GO

    Propósito

    1. Limpieza: Se utiliza la instrucción TRUNCATE TABLE para vaciar la tabla [DIM_PERIODO_ACADEMICO] y garantizar que esté lista para nuevos datos.
    2. Inserción de Registro Especial:
      • ID_PERIODO: -1, para representar períodos sin datos.
      • ID_UNIDAD: 5, asociado a la unidad correspondiente.
      • PERIODO_ACADEMICO: 'Sin datos', indicando ausencia de datos.
      • FECHA_INICIO y FECHA_FIN: NULL, ya que no son aplicables para este registro.

    Notas Técnicas

    1. IDENTITY_INSERT: La opción se activa para permitir la inserción manual de un valor en la columna ID_PERIODO, que es de tipo IDENTITY.
    2. Compatibilidad: Se asegura que la tabla esté vacía antes de insertar el nuevo registro.
    3. Ejecución Segura:
    4. No se utiliza DELETE sino TRUNCATE, optimizando el rendimiento y liberando espacio utilizado.

    Impacto Este registro asegura que cualquier operación dependiente de [DIM_PERIODO_ACADEMICO] pueda manejar escenarios donde no existan datos reales, evitando errores o inconsistencias en el modelo de datos.

    Componente DIM_JORNADA¶

    Descripción General¶

    El componente DIM_JORNADA gestiona el flujo de datos necesario para procesar, limpiar y almacenar información relacionada con jornadas en la tabla DIM_JORNADA dentro de una base de datos. Este proceso incluye la lectura de datos desde un archivo Excel, la transformación de los campos, la validación de duplicados y la carga final en la base de datos destino.

    Componentes del Flujo de Datos¶

    1. Excel Source

      • Descripción: Lectura de datos desde una hoja de Excel llamada Sheet1$.
      • Propiedades:
        • Modo de acceso: 0 (Abrir como tabla).
        • Timeout: 0 (sin límite de espera).
        • Conexión: Administrador de conexiones OleDbConnection.

    2. Sort JORNADA

      • Descripción: Ordena los datos en el flujo según la columna JORNADA, eliminando duplicados.
      • Propiedades:
        • Eliminar duplicados: true.
        • Número máximo de hilos: -1 (utiliza el máximo disponible).

    3. ELIMINAR VACIOS

      • Descripción: Filtra registros donde el campo JORNADA está vacío o nulo, enrutando solo los registros válidos.
      • Condición: 
        !ISNULL(JORNADA) && LEN(JORNADA) > 0

    4. AJUSTAR CAMPO JORNADA

      • Descripción: Transforma la columna JORNADA ajustando el formato de los datos a un tipo de dato compatible.
      • Propiedades:
        • Tipo de salida: Cadena (wstr) de longitud máxima 40.

    5. CREAR ID_UNIDAD

      • Descripción: Asigna valores únicos al campo ID_UNIDAD en el flujo.
      • Código principal: (Simplificado en Visual Basic) 
        Public Overrides Sub Entrada0_ProcessInputRow(ByVal Row As Entrada0Buffer)
        Row.IDUNIDAD = 2
End Sub

    6. Lookup

      • Descripción: Busca registros existentes en la tabla DIM_JORNADA basados en las columnas JORNADA y ID_UNIDAD.
      • Comportamiento sin coincidencias: Enviar los registros sin coincidencia para ser almacenados en la tabla destino.
      • Cache: Completo (CacheType = 0).

    7. Guardar en DIM_JORNADA

      • Descripción: Inserta los datos procesados y transformados en la tabla DIM_JORNADA.
      • Propiedades:
        • Nombre de la tabla: "Cedesarrollo"."DIM_JORNADA".
        • Tamaño de lote: 0 (por defecto, igual al tamaño del buffer).
        • Tiempo de espera: 30 segundos.
        • Inserción masiva: true.

    Componente DIM_JORNADA emp¶

    Descripción General¶

    El paquete DIM_JORNADA emp realiza la extracción, transformación y carga de datos relacionados con las jornadas, utilizando diversas tareas y componentes. Este flujo procesa datos desde una fuente de Excel y realiza validaciones, conversiones y operaciones de búsqueda antes de cargar los datos en una tabla de destino ADO.NET.

    Componentes del Flujo de Datos¶

    1. Excel Source

      • Descripción: Extrae datos de una hoja de Excel (Sheet1$).
      • Propiedades:
        • CommandTimeout: 0 (sin límite de tiempo).
        • OpenRowset: Sheet1$.
      • Columnas de salida:
        • JORNADA (wstr, longitud 255)
        • Sede - jornada - Programa (wstr, longitud 255)

    2. Sort

      • Descripción: Ordena los datos por la columna JORNADA.
      • Propiedades:
        • EliminateDuplicates: true.
        • MaximumThreads: -1.
      • Salida Ordenada: Datos ordenados por JORNADA.

    3. Conditional Split

      • Descripción: Divide los datos en múltiples salidas según la condición especificada.
      • Condición Principal: Filtra filas donde JORNADA no es nulo y su longitud es mayor a 0.
      • Salida: Filas que cumplen con las condiciones.

    4. Data Conversion (Verificar Tipos de Datos)

      • Descripción: Convierte el tipo de datos de JORNADA a longitud 40.
      • Columnas de salida:
        • JORNADA_Convertida (wstr, longitud 40).

    5. Script Component

      • Descripción: Personaliza el procesamiento de datos, asignando valores incrementales y campos específicos.
      • Operaciones principales:
        • Asigna valores al campo ID_UNIDAD.

    6. Lookup

      • Descripción: Realiza una búsqueda en la tabla de DIM_JORNADA para obtener valores relacionados.
      • Propiedades:
        • SQLCommand: 
          SELECT * FROM [Cedesarrollo].[DIM_JORNADA]
        • NoMatchBehavior: Envía filas no coincidentes a la salida predeterminada.
      • Columnas de búsqueda:
        • JORNADA, ID_UNIDAD.

    7. ADO.NET Destination

      • Descripción: Carga los datos procesados en la tabla de destino "Cedesarrollo"."DIM_JORNADA".
      • Propiedades:
        • BatchSize: 0.
        • CommandTimeout: 30.
        • UseBulkInsertWhenPossible: true.

    Componente DIM_PERIODO_ACADEMICO¶

    Descripción General¶

    Este flujo de datos en SSIS realiza la extracción, transformación y carga (ETL) para la dimensión DIM_PERIODO_ACADEMICO. Utiliza datos de una fuente de Excel, realiza conversiones de tipo y valida los registros con una tabla de referencia antes de cargarlos en la base de datos.

    Componentes del Flujo de Datos¶

    1. Excel Source

      • Descripción: Extrae los datos desde una hoja de Excel llamada Sheet1$.
      • Conexión: Administrador de conexiones con Excel 8.
      • Columnas Extraídas:
        • PERIODO (Texto, longitud 255)
        • ID_UNIDAD (Real)
        • FECHA_INICIO (Fecha)
        • FECHA_FIN (Fecha)

    2. Data Conversion

      • Descripción: Convierte los datos de entrada en tipos y formatos específicos.
      • Columnas Convertidas:
      • _PERIODO (Texto, longitud 40)
      • _ID_UNIDAD (Entero)
      • _FECHA_INICIO (Fecha)
      • _FECHA_FIN (Fecha)

    3. Lookup

      • Descripción: Compara los registros con la tabla DIM_PERIODO_ACADEMICO en la base de datos para identificar coincidencias.
      • Conexión: DWH_COMFENALCO_Destino_OLEDB.
      • Consulta de Validación: 
        SELECT * 
FROM [Cedesarrollo].[DIM_PERIODO_ACADEMICO]
WHERE [PERIODO_ACADEMICO] = ? 
    AND [ID_UNIDAD] = ?
      • Columnas Validadas: PERIODO_ACADEMICO, ID_UNIDAD
      • Salidas:
        • Match Output: Registros coincidentes.
        • No Match Output: Registros sin coincidencias, enviados para carga.

    4. Guardar DIM_PERIODO_ACADEMICO

      • Descripción: Inserta los registros no coincidentes en la tabla de destino.

    Componente DIM_ESTUDIANTES¶

    Descripción General¶

    El componente DIM_ESTUDIANTES es una tarea de flujo de datos en SSIS diseñada para procesar, transformar y cargar información sobre estudiantes en una tabla de destino denominada DIM_ESTUDIANTES. Incluye fuentes de datos Excel, conversiones de datos, transformaciones derivadas y múltiples búsquedas para enriquecer la información de los estudiantes con datos relacionados.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos: Excel Source

    • Descripción: Obtiene datos desde una hoja de Excel, Sheet1$, utilizando un administrador de conexión OLE DB.
    • Propiedades:
    • Timeout: 0 segundos (sin límite de espera).
    • Modo de Acceso: Directo a una tabla o vista.

    2. Transformación: Data Conversion

    • Descripción: Convierte columnas clave (TIPO_DOCUMENTO, DOCUMENTO_ESTUDIANTE) en tipos de datos y longitudes compatibles con el destino.
    • Propiedades de Salida:
    • _TIPO_DOCUMENTO → Tipo wstr, longitud 40.
    • _DOCUMENTO → Tipo wstr, longitud 20.

    3. Transformación: Derived Column

    • Descripción: Genera nuevas columnas basadas en expresiones derivadas, como valores predeterminados si las columnas originales son nulas o están vacías.
    • Propiedades:
    • _DOCUMENTO → ISNULL(_DOCUMENTO) || TRIM(_DOCUMENTO) == "" ? "-1" : _DOCUMENTO.

    4. Transformación: Lookup

    • Descripción: Realiza búsquedas en tablas relacionadas para enriquecer datos:
      • Lookup Documento: Valida documentos en DIM_ESTUDIANTES.
      • Lookup 1 1: Consulta DIM_EMPRESAS para obtener ID_EMPRESA.
      • Lookup 1 1 1: Consulta DIM_AFILIADOS para obtener ID_AFILIADO.
      • Lookup 1 1 1 1: Consulta DIM_BENEFICIARIOS para obtener ID_BENEFICIARIO.
      • Lookup 1 1 1 1 1: Consulta DIM_APORTANTE_NOAFILIADO para obtener ID_APORTANTE.

    5. Transformación: Aggregate

    • Descripción: Agrupa y realiza operaciones como conteo o consolidación de documentos y tipos de documentos.

    6. Destino: Flat File Destination

    • Descripción: Registra errores de procesamiento en un archivo plano.
    • Propiedades:
      • Sobrescribir archivo: Activado.
      • Conexión: Flat File Connection Manager.

    7. Destino: ADO.NET Destination

    • Descripción: Carga los datos procesados en la tabla DIM_ESTUDIANTES.
    • Propiedades:
      • Tabla o Vista Destino: "Cedesarrollo"."DIM_ESTUDIANTES".
      • Inserción Masiva: Activado.

    Componente Data DIM_PROGRAMA¶

    Descripción General¶

    El componente Data DIM_PROGRAMA implementa un flujo de datos para extraer información de un archivo Excel, procesarla mediante operaciones de clasificación, agregación y búsquedas, y finalmente cargar los datos en la tabla DIM_PROGRAMA de la base de datos. Este proceso se utiliza para estructurar y consolidar datos relacionados con programas académicos.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos: Excel Source

      • Propósito: Leer datos desde un archivo Excel ubicado en la ruta configurada en el administrador de conexiones OleDbConnection.
      • Columnas Procesadas:
        • PROGRAMA
        • COD_MODULO
        • MODULO
        • SEMESTRE, entre otras.
      • Propiedades Principales:
        • Tabla de Origen: Sheet1$
        • Acceso: Modo directo (AccessMode = 0).

    2. Transformación: Sort PROGRAMA

      • Propósito: Ordenar los datos por la columna PROGRAMA y eliminar duplicados.
      • Propiedades Principales:
        • Columna de Ordenación: PROGRAMA.
        • Eliminación de Duplicados: Activada.
      • Salida:
        • Datos ordenados para el componente de búsqueda.

    3. Transformación: Lookup PROGRAMA

      • Propósito: Realizar una búsqueda en la tabla DIM_PROGRAMA para identificar programas existentes.
      • Propiedades Principales:
        • SQL: 
          SELECT * FROM [Cedesarrollo].[DIM_PROGRAMA] WHERE [PROGRAMA] = ?
        • Comportamiento de No Coincidencia: Las filas sin coincidencias se envían al componente de agregación.
      • Salida:
        • Coincidencias: Filas que ya existen en la tabla DIM_PROGRAMA.
        • No Coincidencias: Filas nuevas que requieren agregación.

    4. Transformación: Aggregate 1

      • Propósito: Consolidar datos agrupados por la columna PROGRAMA.
      • Propiedades Principales:
        • Columna de Agrupación: PROGRAMA.
        • Escala de Claves: Baja (500,000 valores aproximados).
      • Salida:
        • Datos agrupados enviados al destino.

    5. Destino: Guardar DIM_PROGRAMA

      • Propósito: Cargar los datos procesados en la tabla DIM_PROGRAMA de la base de datos.
      • Propiedades Principales:
        • Tabla de Destino: "Cedesarrollo"."DIM_PROGRAMA".
        • Inserción Masiva: Activada.
      • Columnas Procesadas:
        • PROGRAMA.

    Componente Data DIM_PLAN_CURRICULAR¶

    Descripción General¶

    El componente Data DIM_PLAN_CURRICULAR es un flujo de datos que extrae información desde una fuente Excel, realiza transformaciones derivadas y operaciones de búsqueda, y finalmente inserta los datos procesados en la tabla DIM_PLAN_CURRICULAR de la base de datos. Este flujo asegura la consolidación y estructuración de la información del plan curricular.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos: PLAN_CURRICULAR

      • Propósito: Leer datos desde un archivo Excel utilizando el administrador de conexiones OleDbConnection.
      • Columnas Procesadas:
        • PROGRAMA, COD_MODULO, MODULO, SEMESTRE, INTENSIDAD_HORARIA, entre otras.
      • Propiedades Principales:
        • Tabla de Origen: Sheet1$
        • Modo de Acceso: Directo (AccessMode = 0).

    2. Transformación: Tranformar Columnas

      • Propósito: Crear nuevas columnas derivadas a partir de columnas existentes.
      • Columnas Derivadas:
        • _SEMESTRE: Convertido de SEMESTRE a formato DT_WSTR, 40.
        • _MODULO: Convertido de MODULO a formato DT_WSTR, 200.
        • _INTENSIDAD_HORARIA, _INTENSIDAD_HORARIA_SEMANAL, _NO_CREDITOS: Convertidas a formato DT_WSTR, 40.

    3. Transformación: Lookup ID_PROGRAMA

      • Propósito: Verificar y asociar la clave primaria ID_PROGRAMA de la tabla DIM_PROGRAMA con las filas procesadas.
      • Propiedades Principales:
        • SQL: 
          SELECT * FROM [Cedesarrollo].[DIM_PROGRAMA] WHERE [PROGRAMA] = ?
        • Comportamiento de No Coincidencia: Ignorar fallas (IgnoreFailure).
      • Salida:
        • Coincidencias: Filas existentes en DIM_PROGRAMA.
        • No Coincidencias: Se procesan en el siguiente paso.

    4. Transformación: Lookup DIM_PLAN_CURRICULAR

      • Propósito: Identificar filas ya existentes en la tabla DIM_PLAN_CURRICULAR.
      • Propiedades Principales:
        • SQL: 
          SELECT * FROM [Cedesarrollo].[DIM_PLAN_CURRICULAR] WHERE [MODULO] = ?
        • Comportamiento de No Coincidencia: Filas nuevas se envían al destino.

    5. Destino: Guardar DIM_PLAN_CURRICULAR

      • Propósito: Insertar los datos procesados en la tabla DIM_PLAN_CURRICULAR de la base de datos.
      • Propiedades Principales:
        • Tabla de Destino: "Cedesarrollo"."DIM_PLAN_CURRICULAR".
        • Inserción Masiva: Activada (UseBulkInsertWhenPossible = true).
      • Columnas Procesadas: _MODULO, _INTENSIDAD_HORARIA, _INTENSIDAD_HORARIA_SEMANAL, _NO_CREDITOS, _SEMESTRE, ID_PROGRAMA.
