    CEDESARROLLO_DIMENSIONES¶

    El paquete SSIS "05-CEDESARROLLO_DIMENSIONES" está diseñado para procesar, transformar y cargar datos esenciales relacionados con dimensiones clave como períodos académicos, programas educativos, jornadas, y población matriculada. Este paquete asegura la calidad e integridad de los datos consolidados en el Data Warehouse DWH_COMFENALCO, proporcionando una base sólida para análisis estratégicos y toma de decisiones.

    Propósito del Paquete¶

    El propósito principal es gestionar, transformar y cargar datos relacionados con dimensiones transversales del ámbito educativo y administrativo, garantizando su consistencia y disponibilidad para análisis detallados en plataformas de inteligencia de negocios.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes principales:
        • Bases de Datos:
        • DIM_PERIODO_ACADEMICO, DIM_PROGRAMA, DIM_JORNADA, DIM_POBLACION_MATRICULA.
        • Archivos Excel:
        • Información de jornadas y programas.
      • Herramientas utilizadas:
        • Conexiones ADO.NET y OLE DB.

    2. Transformación de Datos:

      • Validaciones (Lookup):
        • Asegura consistencia mediante búsquedas en tablas maestras como DIM_ESTUDIANTES y DIM_TIEMPO.
      • Conversión de Tipos (Data Conversion):
        • Garantiza compatibilidad con las tablas destino.
      • Columnas Derivadas (Derived Column):
        • Genera identificadores y valores predeterminados.

    3. Carga de Datos en el Data Warehouse:

      • Tablas procesadas:
        • DIM_PERIODO_ACADEMICO
        • DIM_PROGRAMA
        • DIM_JORNADA
        • DIM_POBLACION_MATRICULA
      • Configuración:
        • Inserciones masivas habilitadas para maximizar el rendimiento.

    4. Automatización y Scripts:

      • Scripts Python para automatizar tareas de descarga y validación.

    Tablas y Columnas Clave¶

    1. DIM_PERIODO_ACADEMICO:

      • ID_PERIODO, ID_UNIDAD, PERIODO_ACADEMICO, FECHA_INICIO, FECHA_FIN.

    2. DIM_PROGRAMA:

      • ID_PROGRAMA, NOMBRE_PROGRAMA, DESCRIPCION.

    3. DIM_JORNADA:

      • ID_JORNADA, NOMBRE_JORNADA, HORARIO.

    4. DIM_POBLACION_MATRICULA:

      • ID_POBLACION, NOMBRE, DOCUMENTO, GENERO, FECHA_NACIMIENTO.

    Componente Actualizar Dim Estudiantes¶

    Este componente es una tarea de flujo de datos (Data Flow Task) dentro del paquete SSIS diseñado para actualizar la dimensión de estudiantes en la base de datos. La tarea toma como entrada los datos extraídos y transformados (provenientes del componente "Consulta Motor") y los carga en la tabla "Cedesarrollo"."DIM_ESTUDIANTES" utilizando una conexión ADO.NET. Esto garantiza que la información de los estudiantes se mantenga actualizada y refleje los cambios que provienen de las fuentes de datos integradas.

    Propósito del Componente¶

    • Actualización de Datos de Estudiantes:
      Integrar y actualizar la información de la dimensión de estudiantes, asegurando que los registros contengan los datos más recientes provenientes de diversas fuentes.

    • Consolidación de Información:
      Actualizar campos clave como el tipo de documento, número de documento y los identificadores asociados (ID_EMPRESA, ID_AFILIADO, ID_BENEFICIARIO, ID_APORTANTE) en la tabla de estudiantes, permitiendo análisis precisos en el Data Warehouse.

    Descripción Técnica¶

    • Tipo de Tarea:
      Data Flow Task (Microsoft.Pipeline).

    • Origen de Datos:
      La información se recibe de la salida del componente "Consulta Motor", que extrae y transforma los datos desde el área de staging.

    • Transformaciones y Mapeos:
      Los datos provistos incluyen, entre otros, las siguientes columnas:

      • TIPO_DOCUMENTO (cadena de caracteres, 40 caracteres).
      • DOCUMENTO (cadena de caracteres, 255 caracteres).
      • ID_EMPRESA, ID_AFILIADO, ID_BENEFICIARIO, ID_APORTANTE (números enteros).

    • Destino de Datos:
      Los registros se cargan en la tabla "Cedesarrollo"."DIM_ESTUDIANTES" mediante un destino ADO.NET configurado para:

      • Utilizar inserciones masivas (Bulk Insert) para optimizar el rendimiento.
      • Parámetros configurados: BatchSize = 0 y CommandTimeout = 30 segundos.

    Flujo de Ejecución¶

    1. Extracción y Transformación:
      La salida del componente "Consulta Motor" suministra los datos de estudiantes, los cuales incluyen la información necesaria para la actualización de la dimensión.

    2. Carga de Datos:
      Los datos transformados se cargan en la tabla "Cedesarrollo"."DIM_ESTUDIANTES" actualizando o insertando nuevos registros según corresponda, asegurando que la dimensión refleje la información más reciente.

    Diagrama de Secuencia

    ADO_Dest ->> DB: Actualiza la tabla "Cedesarrollo"."DIM_ESTUDIANTES"
    DB -->> ADO_Dest: Confirmación de actualización

    Componente DIM_JORNADA¶

    El contenedor DIM_JORNADA es un contenedor de secuencias que agrupa dos flujos de datos relacionados con la actualización de la dimensión de jornadas. Este contenedor integra dos procesos principales: uno denominado DIM_JORNADA cede y otro DIM_JORNADA emp. Cada uno se encarga de procesar, transformar y cargar datos de jornadas desde fuentes Excel hacia la tabla de destino, aplicando distintas validaciones y transformaciones específicas para cada flujo.

    1. Proceso: DIM_JORNADA cede¶

    Objetivo:
    Procesar y transformar datos relacionados con jornadas para la dimensión “cede”. Se realiza la lectura de datos desde un archivo Excel, se aplican transformaciones como la conversión y derivación de columnas, se filtran registros vacíos, y se enriquecen los datos mediante un componente Lookup que compara con la tabla de destino para identificar registros nuevos.

    Componentes del Flujo:

    • Excel Source:
      Extrae datos de la hoja Sheet1$ utilizando un administrador de conexión OLE DB.

    • Sort JORNADA:
      Ordena los registros por el campo JORNADA y elimina duplicados, garantizando un flujo de datos limpio y ordenado.

    • ELIMINAR VACIOS (Conditional Split):
      Filtra filas que no sean nulas y cuyo largo sea mayor a 0 en la columna JORNADA utilizando la condición:

      !ISNULL(JORNADA) && LEN(JORNADA) > 0

    • Derived Column:
      Transforma el campo JORNADA para ajustarlo a un formato de cadena de longitud máxima 40 y asigna valores constantes para campos como ID_UNIDAD (valor fijo 2) y ID_JORNADA (inicialmente 0).

    • Lookup:
      Realiza una búsqueda en la tabla de destino [Cedesarrollo].[DIM_JORNADA] utilizando los campos JORNADA (convertida) e ID_UNIDAD para determinar si el registro ya existe.

      • No coincidencias: Los registros que no se encuentran se envían a la siguiente etapa para ser insertados.

    • Guardar en DIM_JORNADA:
      Inserta los registros procesados y validados en la tabla destino "Cedesarrollo"."DIM_JORNADA".

      • Propiedades clave:
      • BatchSize = 0
      • CommandTimeout = 30 segundos
      • Inserción masiva activada (UseBulkInsertWhenPossible = true)

    Diagrama de Secuencia (DIM_JORNADA cede):

    sequenceDiagram
    participant ExcelSource as Excel Source
    participant Sort as Sort JORNADA
    participant EliminarVacios as ELIMINAR VACIOS
    participant DerivedColumn as Derived Column
    participant Lookup as Lookup
    participant Guardar as Guardar en DIM_JORNADA

    ExcelSource ->> Sort: Extrae datos de la hoja Sheet1$
    Sort ->> EliminarVacios: Ordena y elimina duplicados
    EliminarVacios ->> DerivedColumn: Filtra registros válidos
    DerivedColumn ->> Lookup: Transforma y asigna ID_UNIDAD
    Lookup ->> Guardar: Envía registros no coincidentes

    2. Proceso: DIM_JORNADA emp¶

    Objetivo:
    Procesar y transformar datos relacionados con jornadas para la dimensión “emp”. Este flujo se orienta a la extracción de información de otra fuente Excel, con sus propias transformaciones y validaciones para asegurar que los datos sean consistentes antes de ser cargados en la misma tabla de destino "Cedesarrollo"."DIM_JORNADA".

    Componentes del Flujo:

    • Excel Source 1:
      Lee los datos de la hoja Sheet1$ desde un archivo Excel utilizando un administrador de conexión específico para esta fuente.

    • Sort:
      Ordena los registros por la columna JORNADA y elimina duplicados, preparándolos para el procesamiento posterior.

    • Conditional Split:
      Filtra las filas verificando que el campo JORNADA no sea nulo y tenga contenido (condición: !ISNULL(JORNADA) && LEN(JORNADA) > 0).

    • Derived Column:
      Genera la columna JORNADA_Convertida, convirtiendo el valor del campo JORNADA a una cadena de longitud 40. Además, asigna valores fijos a ID_UNIDAD (valor 3) y a ID_JORNADA (inicialmente 0).

    • Lookup:
      Compara los registros procesados con la tabla de destino [Cedesarrollo].[DIM_JORNADA] mediante un Lookup que utiliza los campos JORNADA_Convertida e ID_UNIDAD para identificar registros existentes.

      • Registros sin coincidencia: Se envían a la salida de “No Match” para ser insertados.

    • Destino de ADO.NET:
      Inserta los datos transformados y filtrados en la tabla "Cedesarrollo"."DIM_JORNADA".

      • Propiedades clave:
      • BatchSize = 0
      • CommandTimeout = 30 segundos
      • Bulk Insert activado

    Diagrama de Secuencia (DIM_JORNADA emp):

    sequenceDiagram
    participant ExcelSource as Excel Source 1
    participant Sort as Sort
    participant ConditionalSplit as Conditional Split
    participant DerivedColumn as Derived Column
    participant Lookup as Lookup
    participant Destino as Destino de ADO.NET

    ExcelSource ->> Sort: Extrae datos de Excel
    Sort ->> ConditionalSplit: Ordena y filtra registros válidos
    ConditionalSplit ->> DerivedColumn: Pasa registros válidos
    DerivedColumn ->> Lookup: Aplica transformaciones y asigna ID_UNIDAD
    Lookup ->> Destino: Inserta registros nuevos en DIM_JORNADA

    Componente DIM_PERIODO_ACADEMICO¶

    Descripción General¶

    Este flujo de datos en SSIS realiza la extracción, transformación y carga (ETL) para la dimensión DIM_PERIODO_ACADEMICO. Utiliza datos de una fuente de Excel, realiza conversiones de tipo y valida los registros con una tabla de referencia antes de cargarlos en la base de datos.

    Componentes del Flujo de Datos¶

    1. Excel Source

      • Descripción: Extrae los datos desde una hoja de Excel llamada Sheet1$.
      • Conexión: Administrador de conexiones con Excel 8.
      • Columnas Extraídas:
        • PERIODO (Texto, longitud 255)
        • ID_UNIDAD (Real)
        • FECHA_INICIO (Fecha)
        • FECHA_FIN (Fecha)

    2. Data Conversion

      • Descripción: Convierte los datos de entrada en tipos y formatos específicos.
      • Columnas Convertidas:
      • _PERIODO (Texto, longitud 40)
      • _ID_UNIDAD (Entero)
      • _FECHA_INICIO (Fecha)
      • _FECHA_FIN (Fecha)

    3. Lookup

      • Descripción: Compara los registros con la tabla DIM_PERIODO_ACADEMICO en la base de datos para identificar coincidencias.
      • Conexión: DWH_COMFENALCO_Destino_OLEDB.
      • Consulta de Validación: 
        SELECT * 
FROM [Cedesarrollo].[DIM_PERIODO_ACADEMICO]
WHERE [PERIODO_ACADEMICO] = ? 
    AND [ID_UNIDAD] = ?
      • Columnas Validadas: PERIODO_ACADEMICO, ID_UNIDAD
      • Salidas:
        • Match Output: Registros coincidentes.
        • No Match Output: Registros sin coincidencias, enviados para carga.

    4. Guardar DIM_PERIODO_ACADEMICO

      • Descripción: Inserta los registros no coincidentes en la tabla de destino.
      • Tabla de Destino: "Cedesarrollo"."DIM_PERIODO_ACADEMICO".
      • Propiedades:
      • Inserción Masiva: Activada.
      • Batch Size: 0 (Utiliza el tamaño predeterminado del búfer de SSIS).
      • Timeout de Comando: 30 segundos.
      • Columnas Insertadas: PERIODO_ACADEMICO, ID_UNIDAD, FECHA_INICIO, FECHA_FIN

    Diagrama de Secuencia

    sequenceDiagram
    participant Excel as Fuente de Excel
    participant DataConversion as Conversión de Datos
    participant Lookup as Validación de Datos
    participant Destino as Base de Datos

    Excel->>DataConversion: Extraer datos
    DataConversion->>Lookup: Validar con tabla de referencia
    Lookup->>Destino: Insertar registros no coincidentes

    Componente Data DIM_PROGRAMA¶

    Descripción General¶

    El componente Data DIM_PROGRAMA implementa un flujo de datos para extraer información de un archivo Excel, procesarla mediante operaciones de clasificación, agregación y búsquedas, y finalmente cargar los datos en la tabla DIM_PROGRAMA de la base de datos. Este proceso se utiliza para estructurar y consolidar datos relacionados con programas académicos.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos: Excel Source

      • Propósito: Leer datos desde un archivo Excel ubicado en la ruta configurada en el administrador de conexiones OleDbConnection.
      • Columnas Procesadas:
        • PROGRAMA
        • COD_MODULO
        • MODULO
        • SEMESTRE, entre otras.
      • Propiedades Principales:
        • Tabla de Origen: Sheet1$
        • Acceso: Modo directo (AccessMode = 0).

    2. Transformación: Sort PROGRAMA

      • Propósito: Ordenar los datos por la columna PROGRAMA y eliminar duplicados.
      • Propiedades Principales:
        • Columna de Ordenación: PROGRAMA.
        • Eliminación de Duplicados: Activada.
      • Salida:
        • Datos ordenados para el componente de búsqueda.

    3. Transformación: Lookup PROGRAMA

      • Propósito: Realizar una búsqueda en la tabla DIM_PROGRAMA para identificar programas existentes.
      • Propiedades Principales:
        • SQL: 
          SELECT * FROM [Cedesarrollo].[DIM_PROGRAMA] WHERE [PROGRAMA] = ?
        • Comportamiento de No Coincidencia: Las filas sin coincidencias se envían al componente de agregación.
      • Salida:
        • Coincidencias: Filas que ya existen en la tabla DIM_PROGRAMA.
        • No Coincidencias: Filas nuevas que requieren agregación.

    4. Transformación: Aggregate 1

      • Propósito: Consolidar datos agrupados por la columna PROGRAMA.
      • Propiedades Principales:
        • Columna de Agrupación: PROGRAMA.
        • Escala de Claves: Baja (500,000 valores aproximados).
      • Salida:

        • Tabla de Destino: "Cedesarrollo"."DIM_PROGRAMA".
        • Inserción Masiva: Activada.
      • Columnas Procesadas:
        • PROGRAMA.

    Componente Data DIM_PLAN_CURRICULAR¶

    Descripción General¶

    El componente Data DIM_PLAN_CURRICULAR es un flujo de datos que extrae información desde una fuente Excel, realiza transformaciones derivadas y operaciones de búsqueda, y finalmente inserta los datos procesados en la tabla DIM_PLAN_CURRICULAR de la base de datos. Este flujo asegura la consolidación y estructuración de la información del plan curricular.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos: PLAN_CURRICULAR

      • Propósito: Leer datos desde un archivo Excel utilizando el administrador de conexiones OleDbConnection.
      • Columnas Procesadas:
        • PROGRAMA, COD_MODULO, MODULO, SEMESTRE, INTENSIDAD_HORARIA, entre otras.
      • Propiedades Principales:
        • Tabla de Origen: Sheet1$
        • Modo de Acceso: Directo (AccessMode = 0).

    2. Transformación: Tranformar Columnas

      • Propósito: Crear nuevas columnas derivadas a partir de columnas existentes.
      • Columnas Derivadas:
        • _SEMESTRE: Convertido de SEMESTRE a formato DT_WSTR, 40.
        • _MODULO: Convertido de MODULO a formato DT_WSTR, 200.
        • _INTENSIDAD_HORARIA, _INTENSIDAD_HORARIA_SEMANAL, _NO_CREDITOS: Convertidas a formato DT_WSTR, 40.

    3. Transformación: Lookup ID_PROGRAMA

      • Propósito: Verificar y asociar la clave primaria ID_PROGRAMA de la tabla DIM_PROGRAMA con las filas procesadas.
      • Propiedades Principales:
        • SQL: 
          SELECT * FROM [Cedesarrollo].[DIM_PROGRAMA] WHERE [PROGRAMA] = ?
        • Comportamiento de No Coincidencia: Ignorar fallas (IgnoreFailure).
      • Salida:
        • Coincidencias: Filas existentes en DIM_PROGRAMA.
        • No Coincidencias: Se procesan en el siguiente paso.

    4. Transformación: Lookup DIM_PLAN_CURRICULAR

      • Propósito: Identificar filas ya existentes en la tabla DIM_PLAN_CURRICULAR.
      • Propiedades Principales:
        • SQL: 
          SELECT * FROM [Cedesarrollo].[DIM_PLAN_CURRICULAR] WHERE [MODULO] = ?
        • Comportamiento de No Coincidencia: Filas nuevas se envían al destino.

    5. Destino: Guardar DIM_PLAN_CURRICULAR

      • Propósito: Insertar los datos procesados en la tabla DIM_PLAN_CURRICULAR de la base de datos.
      • Propiedades Principales:
        • Tabla de Destino: "Cedesarrollo"."DIM_PLAN_CURRICULAR".
        • Inserción Masiva: Activada (UseBulkInsertWhenPossible = true).
      • Columnas Procesadas: _MODULO, _INTENSIDAD_HORARIA, _INTENSIDAD_HORARIA_SEMANAL, _NO_CREDITOS, _SEMESTRE, ID_PROGRAMA.

    Componente Data DIM_PROGRAMA¶

    Descripción General¶

    El componente Data DIM_PROGRAMA es un Data Flow Task que se encarga de extraer, transformar y cargar datos provenientes de un archivo Excel, para finalmente insertarlos en la tabla de destino "Cedesarrollo"."DIM_PROGRAMA" de la base de datos. Este flujo de datos consolida información sobre programas académicos, garantizando la integridad y calidad de los datos antes de su almacenamiento.

    Funcionalidad y Proceso¶

    El proceso se divide en varias etapas, las cuales se describen a continuación:

    1. Extracción de Datos (Excel Source)

      • Fuente: Se utiliza un origen de datos Excel para leer información de la hoja Sheet1$.
      • Columnas Procesadas: Entre las columnas se encuentra la columna PROGRAMA, que es el dato clave a trabajar.
      • Conexión: Se emplea un administrador de conexión OLE DB (definido en el Connection Manager Excel_Connection_Dim_Programa_DE).

    2. Transformación de Datos

      • Aggregate 1:
        Se utiliza una transformación de tipo Aggregate para agrupar los registros por la columna PROGRAMA.
        • Función: Reúne las filas para obtener un conjunto único de valores de PROGRAMA.
        • Parámetros:
        • KeyScale: Se especifica como 0 (valor bajo, es decir, para aproximadamente 500,000 claves).
      • Sort PROGRAMA:
        Antes del agrupamiento o para garantizar un orden correcto, se aplica una transformación Sort que ordena los datos en base a la columna PROGRAMA (eliminando duplicados).
      • Lookup ID_PROGRAMA:
        Se realiza un Lookup para obtener la clave primaria ID_PROGRAMA de la tabla [Cedesarrollo].[DIM_PROGRAMA].
        • SQL del Lookup:
          select * from [Cedesarrollo].[DIM_PROGRAMA] where [PROGRAMA] = ?
        • Comportamiento:
          Si se encuentra una coincidencia, se recupera el ID_PROGRAMA correspondiente; de lo contrario, se considerará que la fila es nueva.
      • Lookup PROGRAMA:
        Otra transformación Lookup se utiliza para validar la existencia del valor de PROGRAMA en la dimensión, usando la misma columna como parámetro de búsqueda.
        • SQL del Lookup:
          select * from (select * from [Cedesarrollo].[DIM_PROGRAMA]) [refTable]
where [refTable].[PROGRAMA] = ?
        • NoMatchBehavior:
          Se configura para enviar a la salida “no match” aquellas filas que no tienen coincidencia, permitiendo la posterior inserción de nuevos registros.

    3. Componente de Script

      • Componente de Script Personalizado:
        Se incluye un Script Component (implementado en C#) que recibe la columna PROGRAMA y aplica una transformación para capitalizar el valor (por ejemplo, convertirlo a "Title Case").
        • Función:
        • Lee el valor original de PROGRAMA.
        • Utiliza la cultura “es-ES” para convertir el texto a formato Title Case.
        • Almacena el resultado en una columna de salida denominada PROGRAMACAPITALIZADO.
      • Propósito:
        Este paso permite unificar el formato de los nombres de programa antes de su inserción en la base de datos.

    4. Carga de Datos (Destino: ADO.NET Destination)

      • Destino:
        La transformación final consiste en el componente ADO.NET Destination que inserta los datos transformados en la tabla "Cedesarrollo"."DIM_PROGRAMA".
      • Propiedades Clave:
        • BatchSize: 0 (se utiliza el tamaño predeterminado del búfer de SSIS).
        • CommandTimeout: 30 segundos.
        • UseBulkInsertWhenPossible: Activado, para mejorar el rendimiento de la inserción masiva.

    Componente: Data DIM_PLAN_CURRICULAR¶

    Descripción General¶

    El flujo de datos Data DIM_PLAN_CURRICULAR es un Data Flow Task que procesa la información de planes curriculares proveniente de un archivo Excel. Su objetivo es transformar y validar los datos antes de insertarlos en la tabla de destino en la base de datos, asegurando la calidad e integridad de la información.

    Proceso ETL¶

    El flujo de datos se compone de los siguientes pasos:

    1. Extracción de Datos (Excel Source)

      • Origen: Se extraen datos desde la hoja Sheet1$ de un archivo Excel.
      • Columnas Clave:
        • REGISTRO
        • PROGRAMA
        • COD_MODULO
        • MODULO
        • ABREVIACIÓN
        • SEMESTRE
        • INTENSIDAD_HORARIA
        • INTENSIDAD_HORARIA_SEMANAL
        • NO_CREDITOS
      • Conexión: Utiliza un administrador de conexión OLE DB definido en Excel_Connection_Dim_Programa_DE.

    2. Transformación de Datos – Tranformar Columnas (Derived Column)

      • Función:
        Se crean columnas derivadas para formatear y convertir los valores extraídos:
        • _SEMESTRE: Convierte el valor de SEMESTRE a una cadena de 40 caracteres.
        • _INTENSIDAD_HORARIA: Convierte el valor de INTENSIDAD_HORARIA a una cadena de 40 caracteres.
        • _INTENSIDAD_HORARIA_SEMANAL: Convierte el valor de INTENSIDAD_HORARIA_SEMANAL a una cadena de 40 caracteres.
        • _NO_CREDITOS: Convierte el valor de NO_CREDITOS a una cadena de 40 caracteres.
        • _MODULO: Convierte el valor de MODULO a una cadena de 200 caracteres.
      • Objetivo: Normalizar y preparar los datos para la validación y la carga.

    3. Ordenación – Sort PROGRAMA (Sort)

      • Función:
        Ordena los datos en función de la columna PROGRAMA, eliminando duplicados para facilitar el procesamiento posterior.
      • Propiedades Destacadas:
        • EliminateDuplicates: Activado.
        • MaximumThreads: -1 (utiliza el máximo disponible).

    4. Validación y Enriquecimiento – Lookup Transformations

      • Lookup DIM_PLAN_CURRICULAR:

        • Objetivo:
          Verificar si el valor de la columna derivada _MODULO ya existe en la tabla de destino.
        • Consulta:
          select * from (select * from [Cedesarrollo].[DIM_PLAN_CURRICULAR]) [refTable]
where [refTable].[MODULO] = ?
        • Comportamiento en No Coincidencia:
          Los registros sin coincidencia se envían a la salida “Lookup No Match Output”, lo que indica que son nuevos y deben insertarse.

      • Lookup ID_PROGRAMA:

        • Objetivo:
          Obtener el ID_PROGRAMA correspondiente desde la tabla [Cedesarrollo].[DIM_PROGRAMA] utilizando el valor de PROGRAMA extraído del archivo Excel.
        • Consulta:
          select * from [Cedesarrollo].[DIM_PROGRAMA] where [PROGRAMA] = ?
        • Uso:
          La salida de este Lookup se utiliza en la siguiente transformación para separar registros existentes de aquellos nuevos.

    5. División de Datos – Conditional Split

      • Función:
        Evalúa la existencia del valor ID_PROGRAMA (resultado del Lookup ID_PROGRAMA).
      • Condición:
        • Case 1: Registros donde ID_PROGRAMA no es NULL (lo que indica que ya existe un registro relacionado).
        • Default: Registros donde ID_PROGRAMA es NULL (nuevos registros que deben ser insertados).

    6. Carga de Datos – Guardar DIM_PLAN_CURRICULAR (ADO.NET Destination)

      • Función:
        Inserta los registros nuevos (por ejemplo, los provenientes del “Case 1” del Conditional Split) en la tabla "Cedesarrollo"."DIM_PLAN_CURRICULAR".
      • Propiedades Clave:
        • BatchSize: 0 (utiliza el tamaño del buffer interno).
        • CommandTimeout: 30 segundos.
        • UseBulkInsertWhenPossible: True, para optimizar la inserción masiva.
