    TRANSVERSAL_FACT¶

    El paquete SSIS "02-TRANSVERSAL_FACT" está diseñado para procesar y consolidar datos financieros, educativos y operativos relacionados con diversas áreas estratégicas. Este paquete facilita la extracción, transformación y carga (ETL) de datos, asegurando que se integren de manera efectiva en el Data Warehouse DWH_COMFENALCO.

    Propósito del Paquete¶

    El objetivo principal es gestionar datos transversales críticos, desde detalles contables y presupuestarios hasta encuestas y convenios. Este paquete asegura que los datos cargados en el Data Warehouse estén preparados para análisis estratégicos, garantizando calidad, consistencia y precisión.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes principales:
        • Bases de Datos:
        • FACT_DETALLE_CONTABLE
        • FACT_PRESUPUESTO
        • FACT_ENCUESTAS_PSR
        • FACT_CONVENIOS
        • FACT_INICIATIVAS
        • Archivos Excel y CSV:
        • Encuestas, PQRS, y otros registros de entrada.
      • Herramientas utilizadas:
        • Conexiones ADO.NET y OLE DB para acceso eficiente.

    2. Transformación de Datos:

      • Validaciones (Lookup):
        • Garantiza la consistencia comparando con tablas maestras (DIM_TIEMPO, DIM_AFILIADOS, DIM_EMPRESAS).
      • División Condicional (Conditional Split):
        • Clasifica registros en válidos y no válidos según criterios específicos.
      • Conversión de Tipos (Data Conversion):
        • Asegura compatibilidad con el destino.
      • Columnas Derivadas (Derived Column):
        • Genera claves auxiliares y valores predeterminados.

    3. Carga de Datos en el Data Warehouse:

      • Tablas destino:
        • FACT_DETALLE_CONTABLE
        • FACT_PRESUPUESTO
        • FACT_ENCUESTAS_PSR
        • FACT_CONVENIOS
        • FACT_INICIATIVAS
      • Configuración:
        • Inserciones masivas habilitadas para maximizar el rendimiento.

    4. Automatización y Scripts:

      • Scripts Python para automatizar tareas relacionadas con conexiones a SharePoint y procesamiento dinámico de datos.

    5. Mantenimiento de Restricciones:

      • Restauración dinámica de llaves foráneas utilizando SQL dinámico para mantener integridad referencial.

    Tablas y Columnas Clave¶

    1. FACT_DETALLE_CONTABLE:

      • ID_CEBE, ID_CUENTA, ID_FECHA, IMPORTE, GASTOS, ACTIVO, PASIVO, entre otros.

    2. FACT_PRESUPUESTO:

      • ID_CEBE, ID_FECHA, ID_TIPO_PRESUPUESTO, SEGMENT, VALOR, GASTOS, COSTOS.

    3. FACT_ENCUESTAS_PSR:

      • ID_FECHA, ID_EMPRESA, ID_BENEFICIARIO, DOCUMENTO, CALIFICACION.

    4. FACT_CONVENIOS:

      • ID_FECHA, NOMBRE_CONVENIO, VALOR_CONVENIO, ID_PROGRAMA, ID_UNIDAD.

    5. FACT_INICIATIVAS:

      • ID_INICIATIVA, ID_FECHA, NOMBRE_INICIATIVA, DESCRIPCION_INICIATIVA, OBSERVACIONES.

    Componente FACT_PQRS¶

    Descripción General¶

    El componente FACT_PQRS es una tarea de flujo de datos en un paquete SSIS que realiza procesos de extracción, transformación y carga (ETL). Este flujo de datos incluye varias transformaciones, búsquedas y un destino de datos ADO.NET para manejar información de peticiones, quejas, reclamos y sugerencias (PQRS).

    Componentes del Flujo de Datos¶

    1. Fuente de Datos Excel (Excel Source)

      • Descripción: Extrae datos desde un archivo de Excel que contiene registros PQRS.
      • Propiedades:
        • Nombre del objeto: Sheet1$.
        • Tiempo de espera: 0 (infinito).
      • Conexión: Excel_Connection_Fact_PQRS.
      • Columnas de Salida:
        • ID_PQR, ASUNTO, CREADO_POR, ESTADO, TIPO_DOCUMENTO, DOCUMENTO, NOMBRE, entre otras.

    2. Conversión de Datos (Data Conversion)

      • Descripción: Convierte tipos de datos para garantizar compatibilidad con los componentes posteriores.

    3. División Condicional (Conditional Split)

    Restaurar llaves foráneas¶

    Descripción General¶

    El componente Restaurar llaves foráneas es una tarea de ejecución de SQL en un paquete SSIS que se encarga de restaurar las restricciones de clave foránea en el esquema Transversal. Esta tarea utiliza un script dinámico para agregar nuevamente las restricciones desde una tabla persistente, seguida de la limpieza de dicha tabla.

    Componentes del Flujo de Datos¶

    1. Ejecución de SQL (Restaurar llaves foráneas)
    2. Descripción: Restaura las restricciones de clave foránea a partir de una tabla persistente (ForeignKeys_Transversal) en el esquema Transversal.
    3. Propiedades:
      • Consulta SQL: 
        -- Restaurar restricciones de clave foránea
DECLARE @restoreSql NVARCHAR(MAX) = '';

SELECT @restoreSql += 'ALTER TABLE ' + QUOTENAME('Transversal') + '.' + QUOTENAME(fk.TableName) 
    + ' ADD CONSTRAINT ' + QUOTENAME(fk.ConstraintName) 
    + ' FOREIGN KEY (' + QUOTENAME(fk.ColumnName) + ') REFERENCES ' 
    + QUOTENAME(CASE WHEN fk.ReferencedTableName = 'DIM_TIEMPO' THEN 'Dwh' ELSE 'Transversal' END) + '.' + QUOTENAME(fk.ReferencedTableName) 
    + '(' + QUOTENAME(fk.ReferencedColumnName) + '); '
FROM dbo.ForeignKeys_Transversal fk;

-- Ejecutamos el SQL para restaurar las llaves foráneas
EXEC sp_executesql @restoreSql;

-- Limpiar la tabla persistente
DROP TABLE dbo.ForeignKeys_Transversal;
      • Tiempo de espera: 30 segundos.
    4. Conexión: Project.ConnectionManagers[DWH_COMFENALCO].
    5. Descripción Adicional: Esta tarea realiza una restauración dinámica basada en las definiciones de restricciones almacenadas previamente en ForeignKeys_Transversal.
