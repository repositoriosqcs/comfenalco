    COLEGIO_DIMENSIONES_AUXILIAR¶

    El paquete SSIS "03-COLEGIO_DIMENSIONES_AUXILIAR" está diseñado para gestionar la transferencia, transformación y carga (ETL) de datos educativos relacionados con cursos, grados, poblaciones matriculadas y dimensiones de tiempo. Este paquete asegura que los datos provenientes de ambientes remotos se integren eficientemente en el Data Warehouse DWH_COMFENALCO, garantizando su calidad y disponibilidad para análisis.

    Propósito del Paquete¶

    El objetivo principal es sincronizar y centralizar datos de dimensiones educativas clave, asegurando la consistencia y calidad de la información en el Data Warehouse. Esto facilita la toma de decisiones basada en datos para análisis operativos y estratégicos.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes principales:
        • Bases de Datos Remotas:
        • DIM_CURSO, DIM_GRADO, DIM_POBLACION_MATRICULA, DIM_TIEMPO.
      • Herramientas utilizadas:
        • Conexiones ADO.NET para lectura de datos remotos.

    2. Transformación de Datos:

      • Validación de Registros:
        • Se asegura que los datos estén completos antes de cargarlos.
      • Conversión de Tipos (Data Conversion):
        • Asegura compatibilidad con las tablas destino.
      • Inserciones Masivas (Bulk Insert):
        • Habilitadas para maximizar el rendimiento.

    3. Carga de Datos en el Data Warehouse:

      • Tablas procesadas:
        • DIM_CURSO
        • DIM_GRADO
        • DIM_POBLACION_MATRICULA
        • DIM_TIEMPO

    Tablas y Columnas Clave¶

    1. DIM_CURSO:

      • ID_CURSO, DESC_CURSO, FECHA_CREACION, ESTADO_REGISTRO.

    2. DIM_GRADO:

      • ID_GRADO, DESC_GRADO, FECHA_CREACION, ESTADO_REGISTRO.

    3. DIM_POBLACION_MATRICULA:

      • ID_POBLACION_MATRICULA, PARTNER, TIPO_DOCUMENTO, DOCUMENTO, NOMBRE_COMPLETO, GENERO, DIRECCION, TELEFONO, CORREO, FECHA_NACIMIENTO.

    4. DIM_TIEMPO:

      • ID_FECHA, FECHA, DESC_FECHA, ID_SEMANA, DESC_SEMANA, ID_MES, DESC_MES, ID_ANIO, FESTIVO.

    Componentes¶

    Componente Traer a local tablas disponibles en ambiente Comfenalco¶

    Descripción General¶

    El componente Traer a local tablas disponibles en ambiente Comfenalco es una tarea de flujo de datos en un paquete SSIS que transfiere datos desde un origen en un ambiente remoto hacia tablas de destino en un ambiente local. Está diseñado para extraer, transformar y cargar (ETL) datos de múltiples tablas relacionadas con cursos, grados y poblaciones matriculadas.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos ADO.NET (DIM_CURSO_ORIG)

      • Descripción: Consume datos de la tabla [Colegio].[DIM_CURSO] en un origen remoto utilizando ADO.NET.
      • Propiedades:
        • Consulta SQL: No especificada directamente.
        • Tiempo de Espera: 30 segundos.
        • Modo de Acceso: Lectura de la tabla [Colegio].[DIM_CURSO].
      • Conexiones:
        • Connection Manager: Project.ConnectionManagers[DWH_COMFENALCO].
      • Columnas de Salida:
        • ID_CURSO
        • DESC_CURSO
        • FECHA_CREACION
        • ESTADO_REGISTRO

    2. Destino de Datos ADO.NET (DIM_CURSO_DEST)

      • Descripción: Carga datos transformados en la tabla [Colegio].[DIM_CURSO] del ambiente local.
      • Propiedades:
        • Nombre de la Tabla de Destino: "Colegio"."DIM_CURSO".
        • Tamaño del Lote: 0 (predeterminado).
        • Tiempo de Espera: 30 segundos.
        • Inserción Masiva: Activada (UseBulkInsertWhenPossible).
      • Conexiones:
        • Connection Manager: Project.ConnectionManagers[DWH_COMFENALCO_Destino].
      • Columnas de Entrada:

    Componente Traer Dim Tiempo a Local¶

    Descripción General¶

    El componente Traer Dim Tiempo a Local es un flujo de datos en SSIS diseñado para transferir y cargar datos de la dimensión DIM_TIEMPO desde un ambiente remoto hacia un ambiente local. Este flujo está deshabilitado por defecto (Disabled="True"), y su objetivo principal es mantener sincronizados los datos de tiempo.

    Componentes del Flujo de Datos¶

    1. Fuente de Datos ADO.NET (DIM_TIEMPO_ORIG)

      • Descripción: Extrae datos de la tabla [Dwh].[DIM_TIEMPO] en un origen remoto utilizando una conexión ADO.NET.
      • Propiedades:
        • Consulta SQL: No especificada directamente.
        • Tiempo de Espera: 30 segundos.
        • Modo de Acceso: Lectura de la tabla [Dwh].[DIM_TIEMPO].
      • Conexiones:
        • Connection Manager: Project.ConnectionManagers[DWH_COMFENALCO].
      • Columnas de Salida: ID_FECHA, FECHA, DESC_FECHA, ID_SEMANA, DESC_SEMANA, ID_NO_MES, DESC_NO_MES, ID_MES, DESC_MES, DESC_MES_CORTA, ID_BIMESTRE, DESC_BIMESTRE, ID_TRIMESTRE, DESC_TRIMESTRE, ID_CUATRIMESTRE, DESC_CUATRIMESTRE, ID_SEMESTRE, DESC_SEMESTRE, ID_ANIO, ID_ANIO_ANT, NUM_DIA_SEMANA, FESTIVO, FECHA_CORTA

    2. Destino de Datos ADO.NET (DIM_TIEMPO_DEST)

      • Descripción: Carga los datos extraídos y transformados en la tabla [Dwh].[DIM_TIEMPO] en el ambiente local.
      • Propiedades:
        • Nombre de la Tabla de Destino: "Dwh"."DIM_TIEMPO".
        • Tamaño del Lote: 0 (predeterminado).
        • Tiempo de Espera: 30 segundos.
        • Inserción Masiva: Activada (UseBulkInsertWhenPossible).
      • Conexiones:
        • Connection Manager: Project.ConnectionManagers[DWH_COMFENALCO_Destino].
      • Columnas de Entrada: ID_FECHA, FECHA, DESC_FECHA, ID_SEMANA, DESC_SEMANA, ID_NO_MES, DESC_NO_MES, ID_MES, DESC_MES, DESC_MES_CORTA, ID_BIMESTRE, DESC_BIMESTRE, ID_TRIMESTRE, DESC_TRIMESTRE, ID_CUATRIMESTRE, DESC_CUATRIMESTRE, ID_SEMESTRE, DESC_SEMESTRE, ID_ANIO, ID_ANIO_ANT, NUM_DIA_SEMANA, FESTIVO

    Conexiones entre Componentes¶

    1. DIM_TIEMPO_ORIG → DIM_TIEMPO_DEST
