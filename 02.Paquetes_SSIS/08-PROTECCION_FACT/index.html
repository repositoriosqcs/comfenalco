    PROTECCION_FACT¶

    El paquete SSIS "08-PROTECCION_FACT" está diseñado para procesar, transformar y cargar información crítica relacionada con operaciones de protección social, como caracterización de beneficiarios, diagnósticos, entregas de materiales, y visitas de seguimiento. Este paquete consolida datos de múltiples fuentes en el Data Warehouse DWH_COMFENALCO, asegurando su calidad, consistencia y disponibilidad para análisis y reportes estratégicos.

    Propósito del Paquete¶

    El objetivo principal es integrar datos operativos de protección, asegurando su integridad y estructuración para apoyar análisis avanzados y toma de decisiones en el ámbito social y educativo.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes principales:
        • Archivos Planos:
        • FACT_CARACTERIZACION, FACT_DIAGNOSTICOS_EE_JEC, FACT_ENTREGA_MATERIAL.
        • Bases de Datos Relacionales:
        • DIM_TIEMPO, DIM_POBLACION, DIM_PROGRAMA, entre otros.
      • Herramientas utilizadas:
        • Conexiones OLE DB y ADO.NET para integración.

    2. Transformación de Datos:

      • Validaciones (Lookup):
        • Comparación de datos con tablas maestras para enriquecer información, como DIM_ESTABLECIMIENTO_EDUCATIVO y DIM_PREGUNTAS_EE_JEC.
      • Conversión de Tipos (Data Conversion):
        • Garantiza compatibilidad de datos con tablas destino.
      • Columnas Derivadas (Derived Column):
        • Generación de claves únicas y valores predefinidos.

    3. Carga de Datos en el Data Warehouse:

      • Tablas procesadas:
        • FACT_CARACTERIZACION
        • FACT_DIAGNOSTICOS_EE_JEC
        • FACT_ENTREGA_MATERIAL
        • FACT_VISITAS
        • FACT_DESERCION
      • Configuración:
        • Inserciones masivas habilitadas para mejorar el rendimiento.

    4. Automatización y Scripts:

      • Uso de scripts SQL dinámicos para restaurar claves foráneas y garantizar integridad referencial.

    Tablas y Columnas Clave¶

    1. FACT_CARACTERIZACION:

      • ID_POBLACION, ID_PROGRAMA, ID_FECHA, TIPO_DOCUMENTO, DOCUMENTO.

    2. FACT_DIAGNOSTICOS_EE_JEC:

      • ID_FECHA, ID_PREGUNTA, ID_RESPUESTA, ID_REGISTRO.

    3. FACT_ENTREGA_MATERIAL:

      • ID_FECHA_ENTREGA, ID_MATERIAL, ID_PERSONAL, CANTIDAD_MATERIAL.

    4. FACT_VISITAS:

      • ID_FECHA, ID_PROGRAMA, MUNICIPIO, FECHA_PLANEADA, FECHA_EJECUTADA.

    5. FACT_DESERCION:

      • ID_ESTABLECIMIENTO_EDUCATIVO, ID_POBLACION, ID_FECHA_REGISTRO, PROGRAMA, CAUSA.

    Componentes¶

    Componente Procesar FACT_CARACTERIZACION¶

    Descripción General¶

    El componente FACT_CARACTERIZACION realiza un flujo de procesamiento ETL que carga datos desde un archivo plano, realiza conversiones, aplica transformaciones Lookup para enriquecer los datos con información adicional desde tablas externas y finalmente almacena los resultados en una base de datos relacional. Este proceso asegura la integración de múltiples fuentes y formatos de datos en una estructura unificada.

    Componentes del Flujo de Datos¶

    1. Flat File Source
      Descripción:
      Carga datos desde un archivo plano con columnas como FECHA, TIPO_DOCUMENTO, DOCUMENTO, entre otros.

      Propiedades:
      - RetainNulls: false - FileNameColumnName: No especificado.

    2. Data Conversion
      Descripción:
      Convierte los datos de entrada a tipos requeridos para el procesamiento posterior.

      Propiedades:
      - Conversión de columnas como FECHA (a dbTimeStamp) y otras cadenas (wstr) con especificación de longitud.

    3. Lookup (DIM_TIEMPO)
      Descripción:
      Realiza un cruce con la dimensión de tiempo para obtener ID_FECHA.

      Propiedades:
      - SQL Command: select * from [Dwh].[DIM_TIEMPO].

    4. Lookup (DIM_POBLACION)
      Descripción:
      Asocia TIPO_DOCUMENTO y DOCUMENTO con el ID_POBLACION.

      Propiedades:
      - SQL Command: select * from [Proteccion].[DIM_POBLACION].

    5. Lookup (DIM_PROGRAMA)
      Descripción:
      Relaciona los datos de PROGRAMA con ID_PROGRAMA.

      Propiedades:
      - SQL Command: select * from [Proteccion].[DIM_PROGRAMA].

    6. Destino ADO.NET
      Descripción:
      Inserta los datos finales en la tabla FACT_CARACTERIZACION de la base de datos de destino.

      Propiedades:
      - Tabla: "Proteccion"."FACT_CARACTERIZACION".

    Diagrama de Secuencia¶

    Componente Procesar FACT_DIAGNOSTICOS_EE_JEC¶

    Descripción General¶

    El componente FACT_DIAGNOSTICOS_EE_JEC es una tarea de flujo de datos en SSIS diseñada para cargar datos desde un archivo plano en la tabla de hechos FACT_DIAGNOSTICOS_EE_JEC. Incluye conversiones de datos, transformaciones de búsqueda y validaciones previas al almacenamiento en el destino final.

    Componentes del Flujo de Datos¶

    1. Flat File Source

      • Descripción: Lee datos de un archivo plano configurado con el administrador de conexiones Csv_Connection_Fact_Diagnosticos_EE_JEC.
      • Propiedades:
        • RetainNulls: false
        • FileNameColumnName: No especificado.
      • Columnas de salida:
        • FECHA, PROGRAMA, PREGUNTA, RESPUESTA, ID_REGISTRO, entre otros.
      • Errores:
        • Columnas de error incluyen ErrorCode y ErrorColumn.

    2. Data Conversion

      • Descripción: Convierte los datos de entrada a los formatos requeridos para el procesamiento y almacenamiento.
      • Columnas convertidas:
        • Copy of ID_REGISTRO, Copy of FECHA_HORA_INICIO, Copy of FECHA_HORA_FIN, entre otras.

    3. Lookup (DIM_TIEMPO)

      • Descripción: Realiza búsquedas en la dimensión DIM_TIEMPO para recuperar el ID_FECHA basado en la columna FECHA.
      • SQL de referencia: SELECT * FROM [Dwh].[DIM_TIEMPO]
      • Propiedades:
        • CacheType: Completo.
        • NoMatchBehavior: Fallar en ausencia de coincidencias.

    4. Lookup (DIM_PREGUNTAS_EE_JEC)

      • Descripción: Asocia cada pregunta con su identificador único en la tabla DIM_PREGUNTAS_EE_JEC.
      • SQL de referencia: SELECT * FROM [Proteccion].[DIM_PREGUNTAS_EE_JEC]
      • Columnas clave:
        • Copy of PREGUNTA.

    5. Lookup (DIM_RESPUESTAS_EE_JEC)

      • Descripción: Identifica las respuestas mediante las columnas RESPUESTA y ID_PREGUNTA.
      • SQL de referencia: SELECT * FROM [Proteccion].[DIM_RESPUESTAS_EE_JEC]

    6. Destino de ADO NET

      • Descripción: Carga los datos transformados y validados en la tabla FACT_DIAGNOSTICOS_EE_JEC en el servidor de base de datos.
      • Propiedades:
        • TableOrViewName: "Proteccion"."FACT_DIAGNOSTICOS_EE_JEC"
        • UseBulkInsertWhenPossible: true.

    Diagrama de Secuencia¶

    C --> D[Lookup DIM_PREGUNTAS_EE_JEC]

    Componente Procesar FACT_DIAGNOSTICOS_EE_AIPI¶

    Descripción General¶

    Este flujo de datos extrae información desde un archivo plano, realiza transformaciones para cumplir con los requerimientos de integridad y formato, realiza búsquedas en tablas de referencia para enriquecer los datos y finalmente los carga en una tabla destino en SQL Server.

    Componentes del Flujo de Datos¶

    1. Flat File Source

      • Descripción: Este componente carga datos desde un archivo plano (texto) y los envía al flujo de datos para transformarlos y procesarlos.
      • Propiedades:
        • RetainNulls: False (No retiene valores nulos de origen).
        • Columns: Define las columnas a extraer, incluyendo:
          • "ESTABLECIMIENTO EDUCATIVO" (wstr, 255)
          • "NOMBRE_SEDE" (wstr, 255)
          • "MUNICIPIO" (wstr, 255)
          • "ENTIDAD_ADMINISTRADORA" (wstr, 255)
          • "DIRECCION" (wstr, 300)
          • "FECHA" (dbTimestamp).
        • Formato de archivo: Delimitado por comas o tabulaciones (dependiendo de la configuración específica del archivo).

    2. Data Conversion

      • Descripción: Realiza conversiones de datos para asegurar que las columnas tengan el tipo y formato adecuado.
      • Propiedades:
        • Convierte las siguientes columnas:
          • "ESTABLECIMIENTO EDUCATIVO" a "Copy of ESTABLECIMIENTO EDUCATIVO" (wstr, 255).
          • "NOMBRE_SEDE" a "Copy of NOMBRE_SEDE" (wstr, 255).
          • "MUNICIPIO" a "Copy of MUNICIPIO" (wstr, 255).
          • "ENTIDAD_ADMINISTRADORA" a "Copy of ENTIDAD_ADMINISTRADORA" (wstr, 255).
          • "DIRECCION" a "Copy of DIRECCION" (wstr, 300).
          • "FECHA" a "Copy of FECHA" (dbTimestamp).
        • Error Row Disposition: Fail Component (El proceso falla si hay un error en la conversión).

    3. Lookup Transformations

      • Descripción: Realiza búsquedas en tablas de referencia para enriquecer los datos o identificar duplicados.
      • Componentes:
        • Lookup DIM_ESTABLECIMIENTO_EDUCATIVO:
          • Tabla de referencia: DIM_ESTABLECIMIENTO_EDUCATIVO.
          • Columnas para unir:
            • "Copy of ESTABLECIMIENTO EDUCATIVO" con "NOMBRE_ESTABLECIMIENTO".
            • "Copy of MUNICIPIO" con "MUNICIPIO".
          • Columnas devueltas: "ID_ESTABLECIMIENTO_EDUCATIVO".
          • No Match Behavior: Redirect rows to No Match Output.
        • Lookup DIM_PREGUNTAS_EE_AIPI:
          • Tabla de referencia: DIM_PREGUNTAS_EE_AIPI.
          • Columnas para unir:
            • "Copy of PREGUNTA" con "PREGUNTA".
          • Columnas devueltas: "ID_PREGUNTA".
          • No Match Behavior: Redirect rows to No Match Output.
        • Lookup DIM_RESPUESTAS_EE_AIPI:
          • Tabla de referencia: DIM_RESPUESTAS_EE_AIPI.
          • Columnas para unir:
            • "Copy of RESPUESTA" con "RESPUESTA".
          • Columnas devueltas: "ID_RESPUESTA".
          • No Match Behavior: Redirect rows to No Match Output.
        • Lookup FACT_DIAGNOSTICOS_EE_AIPI:
          • Tabla de referencia: FACT_DIAGNOSTICOS_EE_AIPI.
          • Columnas para unir: Todas las columnas clave para verificar duplicados.
          • No Match Behavior: Redirect rows to No Match Output.

    4. ADO.NET Destination

      • Descripción: Inserta los datos procesados en la tabla destino FACT_DIAGNOSTICOS_EE_AIPI.
      • Propiedades: - Tabla o vista: "Proteccion"."FACT_DIAGNOSTICOS_EE_AIPI". - Batch Size: 0 (usa el tamaño predeterminado del búfer). - Command Timeout: 30 segundos. - UseBulkInsertWhenPossible: True (habilita el uso de inserciones masivas para mejorar el rendimiento).
      • Columnas cargadas: - "ID_ESTABLECIMIENTO_EDUCATIVO". - "ID_PREGUNTA". - "ID_RESPUESTA". - "Copy of FECHA".

    Componente Procesar FACT_VENTA¶

    Descripción General¶

    Este flujo de datos se encarga de procesar información sobre ventas a través de varias transformaciones, búsquedas en tablas de referencia y la inserción final en una tabla destino FACT_VENTA. El objetivo principal es enriquecer los datos, realizar validaciones y optimizar el almacenamiento en la base de datos.

    Componentes del Flujo de Datos¶

    1. Flat File Source

      • Descripción: Lee datos desde un archivo plano delimitado.
      • Propiedades:
        • Formato: Delimitado (ejemplo: por comas).
        • Columnas cargadas:
        • FECHA
        • DOCUMENTO
        • NOMBRE_USUARIO
        • CATEGORIA_VENTA
        • SERVICIO
        • TIPO_DOCUMENTO
        • Otros campos como VALOR_PAGADO_SIN_IMP, COSTO, SUBSIDIO.

    2. Data Conversion

      • Descripción: Convierte tipos de datos de entrada para cumplir con los requerimientos del flujo.
      • Propiedades:
        • Convierte las columnas:
        • FECHA a tipo dbTimeStamp.
        • DOCUMENTO, NOMBRE_USUARIO, CATEGORIA_VENTA, SERVICIO, y TIPO_DOCUMENTO a tipo wstr con longitudes específicas.
        • Error Row Disposition: FailComponent.

    3. Lookup Transformations

      • Descripción: Realiza búsquedas en tablas de referencia para enriquecer los datos con identificadores clave.
      • Componentes:
        • Lookup DIM_TIEMPO:
        • Tabla: DIM_TIEMPO.
        • Uniones:
          • Copy of FECHA -> FECHA.
        • Resultado: ID_FECHA.
        • No Match Behavior: Redirigir filas sin coincidencias.
        • Lookup DIM_POBLACION:
        • Tabla: DIM_POBLACION.
        • Uniones:
          • Copy of TIPO_DOCUMENTO -> TIPO_DOCUMENTO.
          • Copy of DOCUMENTO -> DOCUMENTO.
        • Resultado: ID_POBLACION.
        • No Match Behavior: Redirigir filas sin coincidencias.
        • Lookup DIM_TARIFAS_SERVICIOS:
        • Tabla: DIM_TARIFAS_SERVICIOS.
        • Unión:
          • Copy of SERVICIO -> CON_OBJETO_TARIFA.
        • Resultado: Información relacionada con tarifas.
        • Lookup FACT_VENTA:
        • Tabla: FACT_VENTA.
        • Uniones:
          • Clave compuesta: FECHA, DOCUMENTO, CATEGORIA_VENTA, SERVICIO, entre otros.
        • No Match Behavior: Redirigir filas sin coincidencias.

    4. ADO.NET Destination

      • Descripción: Inserta datos en la tabla destino FACT_VENTA.
      • Propiedades:
        • Tabla destino: "Proteccion"."FACT_VENTA".
        • Batch Size: 0 (tamaño predeterminado).
        • Timeout: 30 segundos.
      • Columnas cargadas:
        • ID_FECHA, ID_POBLACION, VALOR_PAGADO_SIN_IMP, COSTO, SUBSIDIO, entre otros.

    Diagrama de Secuencia¶

    graph TD
    A(Flat File Source) --> B(Data Conversion)
    B --> C(Lookup DIM_TIEMPO)
    C --> D(Lookup DIM_POBLACION)
    D --> E(Lookup DIM_TARIFAS_SERVICIOS)
    E --> F(Lookup FACT_VENTA)
    F --> G(ADO.NET Destination)

    Restaurar llaves foráneas esquema Proteccion¶

    Descripción General¶

    Este componente ejecuta un script SQL para restaurar las claves foráneas en el esquema Proteccion. Se asegura de que todas las restricciones de clave foránea estén definidas y actualizadas en las tablas correspondientes. También limpia la tabla persistente ForeignKeys_Proteccion al finalizar.

    Detalles del Componente¶

    1. Tarea Ejecutar SQL (Execute SQL Task):
      • Descripción: Restaura las claves foráneas y limpia la tabla temporal utilizada para su almacenamiento.
      • Propiedades:
        • Conexión: Conexión DWH_COMFENALCO_Destino_OLEDB.
        • Consulta SQL: 
          -- Restaurar restricciones de clave foránea para el esquema Proteccion
DECLARE @restoreSql_Proteccion NVARCHAR(MAX) = '';

SELECT @restoreSql_Proteccion += 'ALTER TABLE ' + QUOTENAME('Proteccion') + '.' + QUOTENAME(fk.TableName) 
    + ' ADD CONSTRAINT ' + QUOTENAME(fk.ConstraintName) 
    + ' FOREIGN KEY (' + QUOTENAME(fk.ColumnName) + ') REFERENCES ' 
    + QUOTENAME(
        CASE 
            WHEN fk.ReferencedTableName IN ('DIM_AFILIADOS', 'DIM_APORTANTE_NOAFILIADO', 'DIM_BENEFICIARIOS', 'DIM_CAPACIDAD_FISICA', 'DIM_CATEGORIA', 'DIM_CUENTA_CONTABLE', 'DIM_EMPRESAS', 'DIM_INFRAESTRUCTURA_CCF', 'DIM_PERSONAL', 'DIM_SEDES', 'DIM_TARIFAS_SERVICIOS', 'DIM_UNIDAD', 'DIM_UNIDADES_ORGANIZATIVAS')
            THEN 'Transversal'
            WHEN fk.ReferencedTableName = 'DIM_TIEMPO' 
            THEN 'Dwh' 
            ELSE 'Proteccion'
        END
    ) + '.' + QUOTENAME(fk.ReferencedTableName) 
    + '(' + QUOTENAME(fk.ReferencedColumnName) + '); ' 
FROM dbo.ForeignKeys_Proteccion fk;

-- Ejecutamos el SQL para restaurar las llaves foráneas del esquema Proteccion
EXEC sp_executesql @restoreSql_Proteccion;

-- Limpiar la tabla persistente
DROP TABLE dbo.ForeignKeys_Proteccion;

    Secuencia de Operaciones¶

    1. Construye dinámicamente los comandos ALTER TABLE para cada clave foránea registrada en la tabla ForeignKeys_Proteccion.
    2. Define las claves foráneas asegurándose de que las tablas referenciadas pertenezcan al esquema correcto (Proteccion, Transversal o Dwh).
    3. Ejecuta el script generado con sp_executesql.
    4. Elimina la tabla temporal ForeignKeys_Proteccion.
