    CEDESARROLLO_FACT¶

    El paquete SSIS "06-CEDESARROLLO_FACT" es un sistema robusto diseñado para realizar procesos ETL (Extracción, Transformación y Carga) relacionados con datos operativos educativos y administrativos. Este paquete se enfoca en integrar información sobre matrículas, jornadas, programas, períodos académicos y evaluaciones, consolidando datos desde múltiples fuentes en el Data Warehouse DWH_COMFENALCO. El objetivo principal es asegurar la disponibilidad de datos precisos y consistentes para análisis estratégico y generación de reportes.

    Propósito del Paquete¶

    El paquete tiene como propósito principal gestionar y transformar datos críticos de varias fuentes. A través de un flujo de trabajo modular y escalable, asegura la integración de los datos, preparándolos para análisis detallados y toma de decisiones estratégicas.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes principales:
        • Bases de Datos:
        • DIM_ESTUDIANTES: Información básica de estudiantes.
        • DIM_JORNADA: Detalles de las jornadas académicas.
        • DIM_PROGRAMA: Datos de programas educativos.
        • FACT_NOTAS: Registros de notas finales de los estudiantes.
        • Archivos Excel y CSV:
        • Planes curriculares.
        • Programas educativos y datos de matrículas.
      • Herramientas utilizadas:
        • Conexión ADO.NET para acceso eficiente a bases de datos.
        • Conexión OLE DB para lectura de archivos Excel y CSV.

    2. Transformación de Datos:

      • Columnas Derivadas (Derived Column):
        • Cálculo de campos adicionales como ID_PROGRAMA y NOTA_PROMEDIO.
      • Validaciones (Lookup):
        • Comparación de datos con tablas maestras como DIM_PERIODO_ACADEMICO y DIM_JORNADA para garantizar consistencia.
      • Clasificación (Conditional Split):
        • Filtrado de datos basándose en condiciones como registros válidos e inválidos.
      • Conversión de Tipos (Data Conversion):
        • Asegura compatibilidad entre columnas de entrada y destino.

    3. Carga de Datos en el Data Warehouse:

      • Tablas utilizadas:
        • DIM_PERIODO_ACADEMICO: Detalles de los períodos académicos.
        • DIM_JORNADA: Información sobre jornadas.
        • DIM_PROGRAMA: Datos de programas educativos.
        • FACT_NOTAS: Registros consolidados de evaluaciones.
      • Configuración:
        • Inserciones Masivas (Bulk Insert): Activada para maximizar el rendimiento en la carga de datos.

    4. Automatización y Scripts:

      • Scripts Python:
        • Automatizan tareas como descargas desde rutas predefinidas y validación de datos en tiempo real.
      • Integración:
        • Uso de variables dinámicas para ajustar rutas de trabajo y parámetros específicos.

    Tablas y Columnas Clave¶

    1. DIM_ESTUDIANTES:

      • ID_ESTUDIANTE: Identificador único.
      • NOMBRE: Nombre del estudiante.
      • DOCUMENTO: Documento de identidad.

    2. DIM_JORNADA:

      • ID_JORNADA: Identificador de la jornada.
      • JORNADA: Descripción de la jornada.

    3. DIM_PROGRAMA:

      • ID_PROGRAMA: Identificador único.
      • NOMBRE_PROGRAMA: Nombre del programa académico.

    4. FACT_NOTAS:

      • ID_NOTA: Identificador único de la nota.
      • ID_ESTUDIANTE: Relación con el estudiante.
      • ID_JORNADA: Relación con la jornada.
      • NOTA_FINAL: Evaluación final.

    Componente Data DIM_ESTUDIANTE 1 1¶

    Descripción General¶

    El flujo de datos Data DIM_ESTUDIANTE 1 1 realiza operaciones de extracción, transformación y carga (ETL) relacionadas con los datos de estudiantes. Este flujo procesa información desde una fuente de Excel, realiza transformaciones de columnas, verifica registros duplicados y vacíos, y finalmente carga los datos en una tabla de destino.

    Componentes del Flujo de Datos¶

    1. Excel Source

      • Descripción: Este componente extrae datos desde una hoja de Excel.
      • Propiedades:
      • Hoja: Sheet1$
      • Columnas extraídas:
        • TIPO_DOCUMENTO
        • REFERENCIA
        • INSCRITO
        • FECHA
        • DOCUMENTO_ESTUDIANTE
        • PROGRAMA
        • NOMBRE
        • SEDE
        • JORNADA

    2. Sort DOCUMENTO

      • Descripción: Ordena los datos en función de las columnas DOCUMENTO_ESTUDIANTE y TIPO_DOCUMENTO.
      • Propiedades:
      • Elimina duplicados: true
      • Columnas utilizadas para ordenar:
        • DOCUMENTO_ESTUDIANTE: Ascendente.
        • TIPO_DOCUMENTO: Ningún orden.

    3. Transformar Columnas

      • Descripción: Transforma las columnas existentes y genera nuevas:
      • _DOCUMENTO: Transformación de DOCUMENTO_ESTUDIANTE.
      • _TIPO_DOCUMENTO: Transformación de TIPO_DOCUMENTO.

    4. Filtrar registros vacíos

      • Descripción: Filtra los registros donde _DOCUMENTO no sea nulo ni vacío.
      • Expresión de filtro:
        !ISNULL(_DOCUMENTO) && LEN(TRIM(_DOCUMENTO)) > 0

    5. Lookup Documento

      • Descripción: Realiza un cruce de información con la tabla DIM_ESTUDIANTES para enriquecer los datos.
      • Propiedades:
        • Consulta SQL: SELECT * FROM [Cedesarrollo].[DIM_ESTUDIANTES]
        • Comportamiento ante no coincidencias: Genera un flujo para registros sin coincidencia.

    6. Aggregate

      • Descripción: Realiza operaciones de agregación sobre las columnas:
        • _DOCUMENTO
        • _TIPO_DOCUMENTO
      • Propiedades:
        • Factor de extensión de memoria: 25%

    7. Guardar DIM_ESTUDIANTES

      • Descripción: Carga los datos procesados en la tabla de destino DIM_ESTUDIANTES.
      • Propiedades:
        • Tabla destino: "Cedesarrollo"."DIM_ESTUDIANTES"
        • Inserción masiva habilitada: true

    Documentación XML del Componente "Restaurar llaves foráneas esquema Cedesarrollo"¶

    Descripción General¶

    El componente Restaurar llaves foráneas esquema Cedesarrollo es una tarea de ejecución de SQL en SSIS que se utiliza para restaurar las restricciones de clave foránea dentro del esquema Cedesarrollo. Además, este proceso limpia la tabla temporal ForeignKeys_Cedesarrollo al final de la ejecución.

    Detalles del Componente¶

    1. Tarea Ejecutar SQL

      • Descripción: Ejecuta un script SQL que reconstruye las claves foráneas en el esquema Cedesarrollo.
      • Conexión:
      • ConnectionManagerID: {57F24C4D-9B0A-4EBB-AE46-43F9B341582B} (Conexión al servidor SQL donde se encuentra el esquema Cedesarrollo).
      • Propiedades:
      • SqlStatementSource: Contiene el siguiente script SQL:
      -- Restaurar restricciones de clave foránea para el esquema Cedesarrollo
DECLARE @restoreSql_Cedesarrollo NVARCHAR(MAX) = '';

SELECT @restoreSql_Cedesarrollo += 'ALTER TABLE ' + QUOTENAME('Cedesarrollo') + '.' + QUOTENAME(fk.TableName) 
    + ' ADD CONSTRAINT ' + QUOTENAME(fk.ConstraintName) 
    + ' FOREIGN KEY (' + QUOTENAME(fk.ColumnName) + ') REFERENCES ' 
    + QUOTENAME(
        CASE 
            WHEN fk.ReferencedTableName IN ('DIM_AFILIADOS', 'DIM_APORTANTE_NOAFILIADO', 'DIM_BENEFICIARIOS', 'DIM_CAPACIDAD_FISICA', 'DIM_CATEGORIA', 'DIM_CUENTA_CONTABLE', 'DIM_EMPRESAS', 'DIM_INFRAESTRUCTURA_CCF', 'DIM_PERSONAL', 'DIM_SEDES', 'DIM_TARIFAS_SERVICIOS', 'DIM_UNIDAD', 'DIM_UNIDADES_ORGANIZATIVAS')
            THEN 'Transversal'
            WHEN fk.ReferencedTableName = 'DIM_TIEMPO' 
            THEN 'Dwh' 
            ELSE 'Cedesarrollo'
        END
    ) + '.' + QUOTENAME(fk.ReferencedTableName) 
    + '(' + QUOTENAME(fk.ReferencedColumnName) + '); '
FROM dbo.ForeignKeys_Cedesarrollo;

-- Ejecutamos el SQL para restaurar las llaves foráneas del esquema Cedesarrollo
EXEC sp_executesql @restoreSql_Cedesarrollo;

-- Limpiar la tabla persistente
DROP TABLE dbo.ForeignKeys_Cedesarrollo;
