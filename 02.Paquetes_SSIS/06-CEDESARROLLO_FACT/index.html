    CEDESARROLLO_FACT¶

    El paquete SSIS "06-CEDESARROLLO_FACT" es un sistema robusto diseñado para realizar procesos ETL (Extracción, Transformación y Carga) relacionados con datos operativos educativos y administrativos. Este paquete se enfoca en integrar información sobre matrículas, jornadas, programas, períodos académicos y evaluaciones, consolidando datos desde múltiples fuentes en el Data Warehouse DWH_COMFENALCO. El objetivo principal es asegurar la disponibilidad de datos precisos y consistentes para análisis estratégico y generación de reportes.

    Propósito del Paquete¶

    El paquete tiene como propósito principal gestionar y transformar datos críticos de varias fuentes. A través de un flujo de trabajo modular y escalable, asegura la integración de los datos, preparándolos para análisis detallados y toma de decisiones estratégicas.

    Descripción del Paquete¶

    1. Extracción de Datos:

      • Fuentes principales:
        • Bases de Datos:
        • DIM_ESTUDIANTES: Información básica de estudiantes.
        • DIM_JORNADA: Detalles de las jornadas académicas.
        • DIM_PROGRAMA: Datos de programas educativos.
        • FACT_NOTAS: Registros de notas finales de los estudiantes.
        • Archivos Excel y CSV:
        • Planes curriculares.
        • Programas educativos y datos de matrículas.
      • Herramientas utilizadas:
        • Conexión ADO.NET para acceso eficiente a bases de datos.
        • Conexión OLE DB para lectura de archivos Excel y CSV.

    2. Transformación de Datos:

      • Columnas Derivadas (Derived Column):
        • Cálculo de campos adicionales como ID_PROGRAMA y NOTA_PROMEDIO.
      • Validaciones (Lookup):
        • Comparación de datos con tablas maestras como DIM_PERIODO_ACADEMICO y DIM_JORNADA para garantizar consistencia.
      • Clasificación (Conditional Split):
        • Filtrado de datos basándose en condiciones como registros válidos e inválidos.
      • Conversión de Tipos (Data Conversion):
        • Asegura compatibilidad entre columnas de entrada y destino.

    3. Carga de Datos en el Data Warehouse:

      • Tablas utilizadas:
        • DIM_PERIODO_ACADEMICO: Detalles de los períodos académicos.
        • DIM_JORNADA: Información sobre jornadas.
        • DIM_PROGRAMA: Datos de programas educativos.
        • FACT_NOTAS: Registros consolidados de evaluaciones.
      • Configuración:
        • Inserciones Masivas (Bulk Insert): Activada para maximizar el rendimiento en la carga de datos.

    4. Automatización y Scripts:

      • Scripts Python:
        • Automatizan tareas como descargas desde rutas predefinidas y validación de datos en tiempo real.
      • Integración:
        • Uso de variables dinámicas para ajustar rutas de trabajo y parámetros específicos.

    Tablas y Columnas Clave¶

    1. DIM_ESTUDIANTES:

      • ID_ESTUDIANTE: Identificador único.
      • NOMBRE: Nombre del estudiante.
      • DOCUMENTO: Documento de identidad.

    2. DIM_JORNADA:

      • ID_JORNADA: Identificador de la jornada.
      • JORNADA: Descripción de la jornada.

    3. DIM_PROGRAMA:

      • ID_PROGRAMA: Identificador único.
      • NOMBRE_PROGRAMA: Nombre del programa académico.

    4. FACT_NOTAS:

      • ID_NOTA: Identificador único de la nota.
      • ID_ESTUDIANTE: Relación con el estudiante.
      • ID_JORNADA: Relación con la jornada.
      • NOTA_FINAL: Evaluación final.

    El componente Truncar tabla Temporal es una tarea de tipo Execute SQL Task que se utiliza para limpiar la tabla temporal utilizada en el proceso ETL de FACT_DESERTORES. Concretamente, se encarga de truncar la tabla [STAGE_AREA].[Transversal].[TMP_DESERTORES_CEDESARROLLO], eliminando todos sus registros antes de cargar nuevos datos. Esto garantiza que el área de staging se encuentre limpia y preparada para el proceso de integración.

    Propiedades del Componente¶

    • Conexión: Utiliza la conexión identificada por {5972EC76-2533-4771-BB68-A92E1CDD645D} para conectarse al servidor SQL donde reside la tabla.
    • Sentencia SQL:
      TRUNCATE TABLE [STAGE_AREA].[Transversal].[TMP_DESERTORES_CEDESARROLLO]
    • Descripción: Ejecuta la sentencia SQL para truncar la tabla temporal, eliminando todos los registros existentes y dejando la tabla lista para la carga de nuevos datos.

    Componente Actualizar Tabla Temporal¶

    Descripción General¶

    El componente Actualizar Tabla Temporal es un Data Flow Task que se encarga de actualizar la tabla temporal utilizada para almacenar información sobre desertores en el proceso ETL. Este flujo extrae datos desde una fuente Excel, aplicando las transformaciones necesarias y finalmente cargando la información en la tabla [Transversal].[TMP_DESERTORES_CEDESARROLLO]. La tarea es fundamental para garantizar que los datos temporales estén actualizados y listos para su posterior procesamiento.

    Componentes del Flujo de Datos¶

    1. Excel Source: cede_desertores

      • Descripción: Extrae datos de un archivo Excel (hoja Sheet1$) que contiene información relevante de desertores.
      • Columnas Extraídas:
        • PROGRAMA
        • NOMBRE_ESTUDIANTE
        • FECHA
        • TIPO
        • CAUSA
        • OBSERVACIONES
        • SEDE
        • JORNADA
      • Conexión: Utiliza el Connection Manager configurado para Excel (por ejemplo, Excel_Connection_Fact_Desertores_CE).

    2. Destino ADO.NET: TMP_DESERTORES_CEDESARROLLO

      • Descripción: Inserta los datos procesados en la tabla temporal [Transversal].[TMP_DESERTORES_CEDESARROLLO] del área de staging.
      • Propiedades:
        • Tabla Destino: "Transversal"."TMP_DESERTORES_CEDESARROLLO"
        • Batch Size: 0 (usa el tamaño predeterminado del buffer)
        • Command Timeout: 30 segundos
        • Uso de Bulk Insert: Activado para mejorar el rendimiento.
      • Conexión: Utiliza el Connection Manager asignado a la base de datos de staging (por ejemplo, STAGE_AREA).

    Flujo del Proceso¶

    1. Extracción: El componente Excel Source extrae la información de la hoja Sheet1$ del archivo configurado.
    2. (Opcional) Procesamiento/Transformación: Si bien en este caso no se indican transformaciones adicionales, en muchos escenarios se pueden incluir pasos intermedios para validar o transformar los datos antes de la carga.
    3. Carga: Los datos extraídos se envían directamente al destino ADO.NET, donde se insertan en la tabla temporal.

    Diagrama de Flujo¶

    Componente Actualizar DIM_PROGRAMA¶

    Descripción General¶

    El componente Actualizar DIM_PROGRAMA es un Data Flow Task que forma parte del proceso ETL del paquete FACT_DESERTORES. Su función es identificar los programas que aún no han sido registrados en la tabla de dimensión DIM_PROGRAMA y posteriormente insertar dichos registros en la base de datos. Para ello, se consume información de la tabla temporal (almacenada en el área de staging) y se realiza un proceso de comparación (lookup) con la tabla de destino para detectar los registros faltantes. Finalmente, los programas identificados se insertan en la tabla [Cedesarrollo].[DIM_PROGRAMA].

    Detalle de Componentes y Flujo¶

    1. Identificar programas

      • Tipo de Componente: Data Reader Source (adaptado mediante un Managed Component Host)
      • Función: Ejecuta una instrucción SQL para extraer la lista de programas distintos desde la tabla temporal [STAGE_AREA].[Transversal].[TMP_DESERTORES_CEDESARROLLO] que aún no se han registrado en la tabla de dimensión.
      • SQL Utilizado: 
        SELECT DISTINCT d.[PROGRAMA]
FROM [STAGE_AREA].[Transversal].[TMP_DESERTORES_CEDESARROLLO] d
LEFT JOIN [DWH_COMFENALCO].[Cedesarrollo].[DIM_PROGRAMA] p ON d.[PROGRAMA] = p.[PROGRAMA]
WHERE p.ID_PROGRAMA IS NULL
      • Objetivo: Extraer los registros de programas nuevos que deben ser insertados.

    2. Guardar DIM_PROGRAMA

      • Tipo de Componente: ADO.NET Destination
      • Función: Inserta en la tabla [Cedesarrollo].[DIM_PROGRAMA] los registros identificados en el paso anterior.
      • Propiedades Clave:
        • Tabla Destino: "Cedesarrollo"."DIM_PROGRAMA"
        • Batch Size: 0 (usa el tamaño predeterminado del buffer)
        • Command Timeout: 30 segundos
        • Bulk Insert: Activado (para optimizar el rendimiento)
      • Conexión: Utiliza el Connection Manager asignado a la base de datos destino (por ejemplo, DWH_COMFENALCO_Destino).

    Flujo del Proceso¶

    1. Extracción de Datos:
      El componente Identificar programas se conecta a la base de datos y ejecuta la consulta SQL para obtener la lista de programas presentes en la tabla temporal que no tienen un registro en DIM_PROGRAMA.

    Componente Data FACT_FACTURACION¶

    Descripción General¶

    El flujo de datos Data FACT_FACTURACION se encarga de extraer, transformar y cargar información relacionada con la facturación. Este componente integra datos provenientes de un archivo Excel (EP-EPT-06) y realiza las siguientes operaciones:

    1. Data Conversion: Convierte el tipo de datos de la columna FUENTE_RECURSOS del flujo de entrada (proveniente del Excel) a un formato de cadena de longitud 40.
    2. Derived Column Transformation: Crea columnas derivadas mediante expresiones. Entre las columnas generadas se encuentran:
      • _DOCUMENTO_PAGO: Convierte el valor de DOCUMENTO_PAGO a una cadena de 20 caracteres.
      • ID_FECHA: Calcula un identificador numérico basado en la fecha contable, combinando año, mes y día.
      • _CONCEPTO: Genera una versión en cadena (longitud 255) del campo CONCEPTO.
      • _ID_CONCEPTO: Convierte el campo ID_CONCEPTO a entero.
      • _TIPO_DOCUMENTO: Convierte TIPO_DOCUMENTO_PAGO a una cadena de 40 caracteres.
      • _NO_RECIBO: Convierte NO_RECIBO a una cadena de 40 caracteres.
      • _ANIO_CONTABLE: Deriva el año de FECHA_CONTABLE en formato cadena (40 caracteres).
    3. Lookup Transformations:
      • Se ejecuta un primer Lookup que, mediante la combinación de los campos NO_RECIBO y la versión derivada de DOCUMENTO_PAGO, consulta la tabla FACT_FACTURACION para determinar si existen registros previos (para evitar duplicados).
      • Se utilizan dos Lookups consecutivos: el primero consulta la tabla destino mediante una expresión SQL sin parámetros y el segundo utiliza parámetros para comparar NO_RECIBO y DOCUMENTO_PAGO.
    4. Destino ADO.NET: Finalmente, el flujo carga los datos transformados y validados en la tabla de destino FACT_FACTURACION. Entre las columnas de destino se encuentran:
      • FECHA_CONTABLE, VALOR_FACTURADO, VALOR_PAGADO, ID_FECHA, TIPO_DOCUMENTO_PAGO, DOCUMENTO_PAGO, CAJERO, ID_CONCEPTO, CONCEPTO, NO_RECIBO, ID_TARIFA, ID_CATEGORIA y FUENTE_RECURSOS.

    Documentación XML del Componente "Restaurar llaves foráneas esquema Cedesarrollo"¶

    Descripción General¶

    El componente Restaurar llaves foráneas esquema Cedesarrollo es una tarea de ejecución de SQL en SSIS que se utiliza para restaurar las restricciones de clave foránea dentro del esquema Cedesarrollo. Además, este proceso limpia la tabla temporal ForeignKeys_Cedesarrollo al final de la ejecución.

    Detalles del Componente¶

    1. Tarea Ejecutar SQL

      • Descripción: Ejecuta un script SQL que reconstruye las claves foráneas en el esquema Cedesarrollo.
      • Conexión:
      • ConnectionManagerID: {57F24C4D-9B0A-4EBB-AE46-43F9B341582B} (Conexión al servidor SQL donde se encuentra el esquema Cedesarrollo).
      • Propiedades:
      • SqlStatementSource: Contiene el siguiente script SQL:
      -- Restaurar restricciones de clave foránea para el esquema Cedesarrollo
DECLARE @restoreSql_Cedesarrollo NVARCHAR(MAX) = '';

SELECT @restoreSql_Cedesarrollo += 'ALTER TABLE ' + QUOTENAME('Cedesarrollo') + '.' + QUOTENAME(fk.TableName) 
    + ' ADD CONSTRAINT ' + QUOTENAME(fk.ConstraintName) 
    + ' FOREIGN KEY (' + QUOTENAME(fk.ColumnName) + ') REFERENCES ' 
    + QUOTENAME(
        CASE 
            WHEN fk.ReferencedTableName IN ('DIM_AFILIADOS', 'DIM_APORTANTE_NOAFILIADO', 'DIM_BENEFICIARIOS', 'DIM_CAPACIDAD_FISICA', 'DIM_CATEGORIA', 'DIM_CUENTA_CONTABLE', 'DIM_EMPRESAS', 'DIM_INFRAESTRUCTURA_CCF', 'DIM_PERSONAL', 'DIM_SEDES', 'DIM_TARIFAS_SERVICIOS', 'DIM_UNIDAD', 'DIM_UNIDADES_ORGANIZATIVAS')
            THEN 'Transversal'
            WHEN fk.ReferencedTableName = 'DIM_TIEMPO' 
            THEN 'Dwh' 
            ELSE 'Cedesarrollo'
        END
    ) + '.' + QUOTENAME(fk.ReferencedTableName) 
    + '(' + QUOTENAME(fk.ReferencedColumnName) + '); '
FROM dbo.ForeignKeys_Cedesarrollo;

-- Ejecutamos el SQL para restaurar las llaves foráneas del esquema Cedesarrollo
EXEC sp_executesql @restoreSql_Cedesarrollo;

-- Limpiar la tabla persistente
DROP TABLE dbo.ForeignKeys_Cedesarrollo;
