    00. WEB SCRAPING

    El contenido proporcionado documenta una amplia variedad de componentes y tareas en una solución SSIS. Aquí se integran scripts de Python como tareas externas para realizar operaciones específicas. A continuación, se detalla la actualización del modelo con base en este contenido.

    Introducción¶

    El modelo de integración documentado combina tareas de procesamiento de datos en SSIS con la ejecución de scripts de Python externos. Estos scripts permiten realizar validaciones avanzadas, procesamiento dinámico y conexiones con sistemas externos como SharePoint. La solución garantiza una ejecución eficiente, flexible y configurable gracias al uso de expresiones del proyecto para adaptar rutas de trabajo y ejecutables a diferentes entornos.

    Resumen General¶

    Objetivos:

    1. Automatización: Integrar tareas repetitivas mediante scripts de Python.
    2. Estandarización: Establecer configuraciones dinámicas para adaptabilidad en entornos variados.
    3. Escalabilidad: Diseñar un modelo modular para incorporar nuevos flujos.

    Componentes Clave:

    • Scripts Python:
      • Conexión y extracción desde SharePoint.
      • Procesamiento de datos educativos, administrativos y financieros.
    • Tareas SSIS:
      • Ejecución de scripts externos con configuraciones dinámicas.
      • Validación y carga en el Data Warehouse.

    Fuentes y Destinos:

    • Fuentes:
      • Archivos planos, Excel y bases de datos relacionales.
      • Sistemas externos a través de scripts Python.
    • Destinos:
      • Tablas de hechos y dimensiones en el Data Warehouse DWH_COMFENALCO.

    Diagrama de Secuencia¶

    sequenceDiagram
    participant SSIS as Tarea SSIS
    participant Python as Python Executable
    participant Script as Python Script
    participant DWH as Data Warehouse

    SSIS ->> Python: Ejecuta script con configuraciones dinámicas
    Python ->> Script: Procesa datos o conecta con sistemas externos
    Script ->> Python: Devuelve resultado
    Python ->> SSIS: Código de retorno
    SSIS ->> DWH: Carga datos procesados

    Buenas Prácticas¶

    1. Configuración Dinámica:

      • Definir ejecutables y directorios mediante variables de proyecto.
      • Adaptar las rutas según los entornos de desarrollo, pruebas y producción.

    2. Registro de Logs:

      • Capturar la salida de los scripts Python para auditoría y resolución de errores.
      • Configurar registros detallados en SSIS.

    3. Compatibilidad:

      • Validar la versión de Python y las dependencias necesarias para cada script.
      • Probar la ejecución en entornos similares al de producción.

    4. Ejecución Segura:

      • Establecer manejadores para códigos de retorno inesperados.
      • Asegurar la idempotencia de los scripts para evitar duplicidades.

    Detalles Técnicos¶

    Componentes Principales:¶

    1. Ejecución de Scripts Python:

    • Propiedades Clave:
      • Ejecutable: Ruta del ejecutable de Python.
      • Argumentos: Nombre del script y parámetros adicionales.
      • Directorio de Trabajo: Ruta base configurada dinámicamente.
    • Código de Retorno:
      • Configurado para no marcar fallos automáticamente.

    2. Conexiones con Sistemas Externos:

    • Scripts dedicados para conectar y extraer datos desde SharePoint.
    • Generación de registros detallados durante el proceso.

    3. Tareas de Procesamiento:

    • Scripts que procesan datos educativos (docentes, matrículas, egresados).
    • Scripts para consolidar datos operativos (ausencias, ingresos).

    Componentes¶

    Secuencia de Operaciones¶

    1. Lee el valor de las expresiones del proyecto para determinar el ejecutable y el directorio de trabajo.
    2. Construye la instrucción de ejecución del script.
    3. Ejecuta el script en el contexto del directorio especificado.
    4. Verifica el código de retorno para determinar el estado de la tarea, pero no marca como fallo si no es un valor de éxito.

    Diagrama de Secuencia¶

    sequenceDiagram
    participant SSIS as SSIS Package
    participant Python as Python Executable
    participant Script as Python Script

    SSIS ->> Python: Ejecutar `python.exe archivo_python.py`
    Python ->> Script: Inicia ejecución
    Script ->> Python: Devuelve resultado
    Python ->> SSIS: Código de retorno (no se fuerza fallo)

    Consideraciones¶

    1. Versiones de Python: El ejecutable especificado debe ser compatible con el script, asegurando que las bibliotecas y dependencias estén instaladas.
    2. Errores en el Script: La tarea no fallará automáticamente si el código de retorno no es exitoso. Si se requiere manejar este comportamiento, se debe incluir lógica adicional en el flujo de trabajo.
    3. Configuración Dinámica: Las expresiones del proyecto permiten adaptar el ejecutable y el directorio de trabajo según las configuraciones del entorno, facilitando portabilidad y mantenimiento.
    4. Control de Log: Configurar un registro adecuado para capturar la salida del script, especialmente si se omiten errores de retorno.

    Este componente es ideal para integrar scripts externos de Python en el flujo de ETL de SSIS, proporcionando flexibilidad para realizar tareas avanzadas o específicas fuera del entorno nativo de SSIS.
