    ETLS CUBO¶

    El paquete SSIS "ETLS CUBO" está diseñado para gestionar procesos ETL (Extracción, Transformación y Carga) enfocados en datos transversales y financieros. Este paquete asegura la correcta preparación de tablas, la consolidación de datos y la integración eficiente en el Data Warehouse DWH_COMFENALCO, proporcionando una base sólida para análisis y generación de reportes.

    Propósito del Paquete¶

    El objetivo principal del paquete es centralizar datos clave de actividades empresariales, afiliaciones, aportes, personal y evaluaciones, asegurando su calidad y consistencia. Además, prepara las tablas destino para la carga eficiente y garantiza la integridad referencial de los datos.

    Descripción del Paquete¶

    1. Preparación de Tablas:

    • Limpieza y creación de tablas clave como:
      • FACT_ACTIVIDADES
      • FACT_PERSONAL
      • FACT_EVALUACION_DOCENTE
      • FACT_FINANCIERA
      • DIM_TIEMPO_MENSUAL
    • Acciones realizadas:
      • Si las tablas existen: TRUNCATE TABLE.
      • Si no existen: CREATE TABLE.

    2. Extracción de Datos:

    • Fuentes principales:
      • Archivos Excel y CSV.
      • Bases de Datos:
      • DIM_EMPRESAS
      • DIM_AFILIADOS
      • FACT_APORTES_SHR_DET
      • FACT_PERSONAL
      • FACT_FINANCIERA
    • Herramientas utilizadas:
      • Conexiones ADO.NET para consultas optimizadas.

    3. Transformación de Datos:

    • Validaciones (Lookup):
      • Comprobaciones en tablas maestras como DIM_TIEMPO_MENSUAL y DIM_POBLACION.
    • Conversión de Tipos (Data Conversion):
      • Garantiza compatibilidad con tablas destino.
    • Columnas Derivadas (Derived Column):
      • Generación de valores calculados como ID_FECHA y agregados mensuales.

    4. Carga de Datos en el Data Warehouse:

    • Tablas procesadas:
      • FACT_ACTIVIDADES
      • FACT_PERSONAL
      • FACT_EVALUACION_DOCENTE
      • FACT_FINANCIERA
      • DIM_TIEMPO_MENSUAL
    • Configuración:
      • Inserciones masivas habilitadas para maximizar el rendimiento.

    Tablas y Columnas Clave¶

    1. FACT_ACTIVIDADES:

    • PARTNER, ID_EMPRESA, ID_AFILIADO, FECHA_AFILIACION, TIPO_AFILIADO, ACTIVIDAD.

    2. FACT_PERSONAL:

    • ID_PERSONAL, NOMBRE, CONCEPTO, DESCRIPCION, FECHA_FIN.

    3. FACT_EVALUACION_DOCENTE:

    • PERIODO_ACADEMICO, ID_UNIDAD, ID_PERSONAL, NOMBRE_DOCENTE, CALIFICACION_DEFINITIVA.

    4. FACT_FINANCIERA:

    • ID_FECHA, ID_CEBE, DESCRIPCION_CEBE, DEPARTAMENTO, TOTAL, COSTOS.

    5. DIM_TIEMPO_MENSUAL:

    • ID_FECHA, FECHA, ID_MES, ID_ANIO, DESC_MES.

    Componente Limpiar y Crear Tablas¶

    Descripción General¶

    El componente Limpiar y Crear Tablas es una tarea de ejecución SQL que garantiza la preparación de las tablas del Data Warehouse (DWH_COMFENALCO) antes de la carga de datos. Este proceso incluye la verificación de la existencia de tablas, el borrado de datos en caso de que existan, y la creación de tablas si aún no han sido definidas.

    Componentes del Flujo de Datos¶

    Componente DIM_TIEMPO_MENSUAL¶

    Descripción General¶

    El componente DIM_TIEMPO_MENSUAL es un flujo de datos diseñado para la extracción, transformación y carga (ETL) de información temporal en la tabla DIM_TIEMPO_MENSUAL. Este flujo asegura la integración y consolidación de datos relacionados con fechas y periodos temporales clave, organizados desde días hasta semestres, con detalles descriptivos y métricas relevantes.

    Componentes del Flujo de Datos¶

    1. Origen: Origen de ADO NET

      • Propósito: Extraer datos de la tabla [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO] que correspondan al primer día de cada mes en un rango de años específico.
      • Consulta SQL Utilizada: 
        WITH FechasPrimerDiaMes AS (
    SELECT 
        [ID_FECHA],
        [FECHA],
        [DESC_FECHA],
        [ID_SEMANA],
        [DESC_SEMANA],
        [ID_NO_MES],
        [DESC_NO_MES],
        [ID_MES],
        [DESC_MES],
        [DESC_MES_CORTA],
        [ID_BIMESTRE],
        [DESC_BIMESTRE],
        [ID_TRIMESTRE],
        [DESC_TRIMESTRE],
        [ID_CUATRIMESTRE],
        [DESC_CUATRIMESTRE],
        [ID_SEMESTRE],
        [DESC_SEMESTRE],
        [ID_ANIO],
        [ID_ANIO_ANT],
        [NUM_DIA_SEMANA],
        [FESTIVO],
        [FECHA_CORTA]
    FROM 
        [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
    WHERE 
        DATEPART(DAY, FECHA) = 1
        AND ID_ANIO BETWEEN DATEPART(YEAR, GETDATE()) - 2 AND DATEPART(YEAR, GETDATE())
)
SELECT 
    *
FROM 
    FechasPrimerDiaMes
WHERE 
    [FECHA] < '2024-06-01'
ORDER BY 
    [FECHA];
      • Propiedades:
      • Timeout de Comando: 30 segundos.
      • Conexión Utilizada: DWH_COMFENALCO.
      • Acceso: Modo SQL directo para optimizar la extracción de datos.

    2. Destino: Destino de ADO NET

      • Propósito: Cargar los datos transformados en la tabla "Transversal"."DIM_TIEMPO_MENSUAL".

      • Propiedades:

        • Nombre de la Tabla Destino: "Transversal"."DIM_TIEMPO_MENSUAL".
        • Inserción Masiva: Activada (UseBulkInsertWhenPossible = true).
        • Timeout de Comando: 30 segundos.
        • Conexión Utilizada: DWH_COMFENALCO.

      • Columnas Procesadas:

        • ID_FECHA, FECHA, DESC_FECHA
        • ID_SEMANA, DESC_SEMANA

        • ID_MES, DESC_MES, DESC_MES_CORTA

        • ID_TRIMESTRE, DESC_TRIMESTRE
        • ID_CUATRIMESTRE, DESC_CUATRIMESTRE
        • ID_SEMESTRE, DESC_SEMESTRE
        • ID_ANIO, ID_ANIO_ANT
        • NUM_DIA_SEMANA, FESTIVO, FECHA_CORTA.

    3. Ruta de Datos

      • Conexión: Los datos se transfieren directamente desde Origen de ADO NET hacia Destino de ADO NET.
      • Gestión de Errores: Los errores en filas se manejan a través de las salidas de error de cada componente.

    Componente Afiliación Mensual Empresas¶

    Descripción General¶

    El componente Afiliación Mensual Empresas pertenece al flujo de datos del paquete FACT_ACTIVIDADES y tiene como propósito la extracción, transformación y carga (ETL) de información relacionada con las afiliaciones mensuales de empresas. Este flujo extrae registros desde la dimensión DIM_EMPRESAS y genera datos agregados en la tabla destino FACT_ACTIVIDADES con granularidad mensual.

    Componentes del Flujo de Datos¶

    1. Origen: Empresas

    • Propósito: Extraer datos de la tabla DIM_EMPRESAS para generar un conjunto de resultados enfocado en empresas activas.
    • Consulta SQL Utilizada: 
      WITH MaxFecha AS (
    SELECT MAX(FECHA) AS MaxFecha
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
NumberSeries AS (
    SELECT ROW_NUMBER() OVER (ORDER BY (SELECT NULL)) - 1 AS Number
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
DateSeries AS (
    SELECT 
        DATEADD(MONTH, Number, DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1)) AS MonthDate,
        CONVERT(INT, FORMAT(DATEADD(MONTH, Number, DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1)), 'yyyyMMdd')) AS ID_FECHA
    FROM 
        NumberSeries
    CROSS JOIN 
        MaxFecha
    WHERE 
        Number BETWEEN 0 AND DATEDIFF(MONTH, DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1), MaxFecha)
),
Empresas AS (
    SELECT TOP (1000)
        [ID_EMPRESA],
        [PARTNER],
        [FECHA_AFILIACION],
        [FECHA_FUNDACION],
        [FECHA_RETIRO],
        [ESTADO],
        [ESTADOREGISTRO],
        [COD_CIUDAD]
    FROM 
        [DWH_COMFENALCO].[Aportes].[DIM_EMPRESAS]
    WHERE 
        [ESTADO] = 'VIGEN' AND [ESTADOREGISTRO] = 'CURRENT'
),
EmpresasPorMes AS (
    SELECT 
        e.[ID_EMPRESA],
        e.[PARTNER],
        e.[FECHA_AFILIACION],
        e.[FECHA_FUNDACION],
        e.[FECHA_RETIRO],
        e.[ESTADOREGISTRO],
        e.[ESTADO],
        e.[COD_CIUDAD],
        ds.MonthDate AS FECHA_MENSUAL,
        ds.ID_FECHA,
        5 AS ID_UNIDAD
    FROM 
        Empresas e
    INNER JOIN 
        DateSeries ds ON ds.MonthDate BETWEEN e.[FECHA_AFILIACION] AND COALESCE(e.[FECHA_RETIRO], GETDATE())
)
SELECT 
    em.[PARTNER],
    em.[ID_EMPRESA],
    -1 AS ID_AFILIADO,
    -1 AS ID_TIPO_AFILIADO,
    NULL AS TIPO_AFILIADO,
    5 AS ID_CATEGORIA,
    em.[FECHA_AFILIACION],
    CAST(STUFF(STUFF(em.[FECHA_RETIRO], 5, 0, '-'), 8, 0, '-') AS DATE) AS FECHA_RETIRO,
    -1 AS ID_GENERO,
    -1 AS ID_ESTADO_CIVIL,
    -1 AS ID_PERTENENCIA_ETNICA,
    -1 AS ID_FACTOR_VULNERABILIDAD,
    NULL AS ESTRATO,
    em.[COD_CIUDAD] AS ID_CIUDAD,
    NULL AS SALARIO_BASICO,
    em.[FECHA_MENSUAL],
    em.[ID_FECHA],
    em.[ID_UNIDAD],
    em.[ESTADOREGISTRO],
    -1 AS PARTNER_AFILIADO,
    em.[PARTNER] AS PARTNER_EMPRESA,
    'EMPRESA' AS TIPO_POBLACION,
    'AFILIACION' AS ACTIVIDAD
FROM 
    EmpresasPorMes em
INNER JOIN 
    [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] dtm ON em.ID_FECHA = dtm.ID_FECHA
ORDER BY 
    em.[ID_EMPRESA], em.[FECHA_MENSUAL];
    • Propiedades:
      • Timeout de Comando: 30 segundos.
      • Conexión Utilizada: DWH_COMFENALCO.

    2. Destino: Destino de ADO NET

    • Propósito: Cargar los datos transformados en la tabla "Transversal"."FACT_ACTIVIDADES".
    • Propiedades:
      • Nombre de la Tabla Destino: "Transversal"."FACT_ACTIVIDADES".
      • Inserción Masiva: Activada (UseBulkInsertWhenPossible = true).
      • Timeout de Comando: 30 segundos.
      • Conexión Utilizada: DWH_COMFENALCO.

    3. Ruta de Datos

    • Conexión: Los datos se transfieren directamente desde Empresas hacia Destino de ADO NET.
    • Gestión de Errores: Los errores en filas se manejan a través de las salidas de error de cada componente.

    Diagrama de Secuencia¶

    participant Destino as FACT_ACTIVIDADES

    Componente Afiliación Mensual Afiliados¶

    Descripción General¶

    El componente Afiliación Mensual Afiliados forma parte del flujo de datos del paquete FACT_ACTIVIDADES y está diseñado para manejar el proceso de extracción, transformación y carga (ETL) de información relacionada con los afiliados mensuales. Este flujo toma datos de la dimensión DIM_AFILIADOS, realiza transformaciones para organizar la información en granularidad mensual, y la almacena en la tabla destino FACT_ACTIVIDADES.

    Componentes del Flujo de Datos¶

    1. Origen: Afiliados

    • Propósito: Extraer datos de la tabla DIM_AFILIADOS para obtener información sobre afiliados actuales.
    • Consulta SQL Utilizada: 
      WITH MaxFecha AS (
    SELECT MAX(CAST(FECHA AS DATE)) AS MaxFecha
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
NumberSeries AS (
    SELECT ROW_NUMBER() OVER (ORDER BY (SELECT NULL)) - 1 AS Number
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
DateSeries AS (
    SELECT 
        DATEADD(MONTH, Number, CAST(DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1) AS DATE)) AS MonthDate,
        CONVERT(INT, FORMAT(DATEADD(MONTH, Number, CAST(DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1) AS DATE)), 'yyyyMMdd')) AS ID_FECHA
    FROM 
        NumberSeries
    CROSS JOIN 
        MaxFecha
    WHERE 
        Number BETWEEN 0 AND DATEDIFF(MONTH, CAST(DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1) AS DATE), MaxFecha)
),
Afiliados AS (
    SELECT TOP (1000)
        [ID_AFILIADO],
        [PARTNER],
        [ID_TIPO_AFILIADO],
        [TIPO_AFILIADO],
        [ID_CATEGORIA],
        CAST([FECHA_AFILIACION] AS DATE) AS FECHA_AFILIACION,
        CAST([FECHA_RETIRO] AS DATE) AS FECHA_RETIRO,
        [ID_GENERO],
        [ID_ESTADO_CIVIL],
        [ID_PERTENENCIA_ETNICA],
        [ID_FACTOR_VULNERABILIDAD],
        [ESTRATO],
        [ID_CIUDAD],
        [SALARIO_BASICO],
        [APORTANTE],
        [ESTADOREGISTRO]
    FROM 
        [DWH_COMFENALCO].[Aportes].[DIM_AFILIADOS]
    WHERE 
        [ESTADOREGISTRO] = 'CURRENT' AND ISDATE([FECHA_AFILIACION]) = 1 AND (ISDATE([FECHA_RETIRO]) = 1 OR [FECHA_RETIRO] IS NULL)
),
AfiliadosPorMes AS (
    SELECT 
        a.[ID_AFILIADO],
        a.[PARTNER],
        a.[ID_TIPO_AFILIADO],
        a.[TIPO_AFILIADO],
        a.[ID_CATEGORIA],
        a.[FECHA_AFILIACION],
        a.[FECHA_RETIRO],
        a.[ID_GENERO],
        a.[ID_ESTADO_CIVIL],
        a.[ID_PERTENENCIA_ETNICA],
        a.[ID_FACTOR_VULNERABILIDAD],
        a.[ESTRATO],
        a.[ID_CIUDAD],
        a.[SALARIO_BASICO],
        a.[APORTANTE],
        a.[ESTADOREGISTRO],
        ds.MonthDate AS FECHA_MENSUAL,
        ds.ID_FECHA,
        5 AS ID_UNIDAD,
        ROW_NUMBER() OVER (PARTITION BY a.[PARTNER], ds.ID_FECHA ORDER BY a.[SALARIO_BASICO] DESC) AS RN
    FROM

    INNER JOIN 
        DateSeries ds ON ds.MonthDate BETWEEN a.[FECHA_AFILIACION] AND COALESCE(a.[FECHA_RETIRO], GETDATE())
)

    Componente Afiliación Mensual Beneficiarios¶

    Descripción General¶

    El componente Afiliación Mensual Beneficiarios forma parte del flujo de datos del paquete FACT_ACTIVIDADES, diseñado para procesar información mensual relacionada con beneficiarios. Este flujo extrae datos de la tabla DIM_BENEFICIARIOS, organiza la información por fechas mensuales, realiza transformaciones específicas y almacena los datos en la tabla destino FACT_ACTIVIDADES.

    Componentes del Flujo de Datos¶

    1. Origen: Beneficiarios

    • Propósito: Extraer datos de la tabla DIM_BENEFICIARIOS para obtener registros actuales relacionados con los beneficiarios, aplicando transformaciones iniciales.
    • Consulta SQL Utilizada: 
      WITH MaxFecha AS (
    SELECT MAX(CAST(FECHA AS DATE)) AS MaxFecha
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
NumberSeries AS (
    SELECT ROW_NUMBER() OVER (ORDER BY (SELECT NULL)) - 1 AS Number
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
DateSeries AS (
    SELECT 
        DATEADD(MONTH, Number, CAST(DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1) AS DATE)) AS MonthDate,
        CONVERT(INT, FORMAT(DATEADD(MONTH, Number, CAST(DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1) AS DATE)), 'yyyyMMdd')) AS ID_FECHA
    FROM 
        NumberSeries
    CROSS JOIN 
        MaxFecha
    WHERE 
        Number BETWEEN 0 AND DATEDIFF(MONTH, CAST(DATEFROMPARTS(YEAR(MaxFecha) - 2, 1, 1) AS DATE), MaxFecha)
),
Beneficiarios AS (
    SELECT TOP (1000)
        [ID_BENEFICIARIO],
        [PARTNER],
        [FECHA_AFILIACION],
        [FECHA_RETIRO],
        [ID_GENERO],
        [ID_ESTADO_CIVIL],
        [ESTRATO],
        [ID_CIUDAD],
        [ID_PARENT_SAP],
        [PARENT_SAP],
        [APORTANTE],
        [TIPO_AFILIADO],
        [TIPO_APORTANTE],
        [ESTADOREGISTRO],
        ROW_NUMBER() OVER (PARTITION BY PARTNER ORDER BY CASE WHEN FECHA_INGRESO_EMPRESA = '' THEN 99991231 END) AS ROWNUM
    FROM 
        [DWH_COMFENALCO].[Aportes].[DIM_BENEFICIARIOS]
    WHERE 
        [ESTADOREGISTRO] = 'CURRENT' AND [TIPO_APORTANTE] = 'X'
),
FilteredBeneficiarios AS (
    SELECT *
    FROM Beneficiarios
    WHERE ROWNUM = 1
),
BeneficiariosPorMes AS (
    SELECT 
        b.[ID_BENEFICIARIO],
        b.[PARTNER],
        b.[FECHA_AFILIACION],
        b.[FECHA_RETIRO],
        b.[ID_GENERO],
        b.[ID_ESTADO_CIVIL],
        b.[ESTRATO],
        b.[ID_CIUDAD],
        b.[ID_PARENT_SAP],
        b.[PARENT_SAP],
        b.[APORTANTE],
        b.[TIPO_AFILIADO],
        b.[TIPO_APORTANTE],
        b.[ESTADOREGISTRO],
        ds.MonthDate AS FECHA_MENSUAL,
        ds.ID_FECHA,
        ROW_NUMBER() OVER (PARTITION BY b.[ID_BENEFICIARIO], ds.ID_FECHA ORDER BY ds.MonthDate DESC) AS RN
    FROM 
        FilteredBeneficiarios b
    INNER JOIN 
        DateSeries ds ON ds.MonthDate BETWEEN b.[FECHA_AFILIACION] AND COALESCE(b.[FECHA_RETIRO], GETDATE())
)
SELECT 
    bm.[PARTNER],
    -1 AS ID_EMPRESA,
    -1 AS ID_AFILIADO,
    -1 AS ID_TIPO_AFILIADO,
    NULL AS TIPO_AFILIADO,
    -1 AS ID_CATEGORIA,
    CAST(STUFF(STUFF(bm.[FECHA_AFILIACION], 5, 0, '-'), 8, 0, '-') AS DATE) AS [FECHA_AFILIACION],
    CAST(STUFF(STUFF(bm.[FECHA_RETIRO], 5, 0, '-'), 8, 0, '-') AS DATE) AS FECHA_RETIRO,
    bm.[ID_GENERO],
    bm.[ID_ESTADO_CIVIL],
    -1 AS ID_PERTENENCIA_ETNICA,
    -1 AS ID_FACTOR_VULNERABILIDAD,
    bm.[ESTRATO],
    bm.[ID_CIUDAD],
    NULL AS SALARIO_BASICO,
    bm.[FECHA_MENSUAL],
    bm.[ID_FECHA],
    5 AS ID_UNIDAD,
    bm.[ESTADOREGISTRO],
    bm.[ID_PARENT_SAP] AS PARTNER_AFILIADO,
    bm.[PARENT_SAP] AS PARTNER_EMPRESA,
    'BENEFICIARIO' as TIPO_POBLACION,
    'AFILIACION' AS ACTIVIDAD
FROM 
    BeneficiariosPorMes bm
INNER JOIN 
    [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] dtm ON bm.ID_FECHA = dtm.ID_FECHA
WHERE 
    bm.RN = 1
ORDER BY 
    bm.[ID_BENEFICIARIO], bm.[FECHA_MENSUAL];

      • Conexión Utilizada: DWH_COMFENALCO.

    Componente FACT_FINANCIERA_CUBO 1¶

           Tp.[ID_ANIO],
           Tp.[ID_MES],
           Ft.[ID_CEBE],

    LEFT JOIN [DWH_COMFENALCO].[Financiera].[DIM_UNIDADES_ORGANIZATIVAS] Uo ON Ft.ID_CEBE = Uo.ID_CEBE
    INNER JOIN [DWH_COMFENALCO].[Financiera].[DIM_CUENTA_CONTABLE] Ct ON Ft.ID_CUENTA = Ct.ID_CUENTA
    WHERE Ct.TIPO_CUENTA NOT IN ('INGRESOS', 'COSTOS', 'GASTOS')

    GROUP BY Tp.[ID_ANIO], Tp.[ID_MES], Ft.[ID_CEBE], ...
) uni
INNER JOIN (
    SELECT ID_FECHA AS ID_FECHA2
    FROM [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]
