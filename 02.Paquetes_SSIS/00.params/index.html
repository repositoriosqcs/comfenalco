    Introduccion

    La presente solución SSIS constituye un conjunto integral de paquetes ETL cuidadosamente diseñados para consolidar, transformar y cargar datos provenientes de fuentes heterogéneas hacia el Data Warehouse DWH_COMFENALCO. Cada paquete aborda dominios específicos, como educación, protección social, finanzas y operaciones transversales, maximizando la calidad, consistencia y accesibilidad de los datos con el propósito de sustentar decisiones estratégicas.

    Resumen General¶

    Propósitos Estratégicos:

    1. Centralizar y estandarizar datos multisectoriales.
    2. Proveer información verificada y estructurada para análisis operativos y de inteligencia empresarial.
    3. Implementar transformaciones avanzadas que optimicen la confiabilidad y el valor de los datos.

    Componentes Fundamentales:

    • Dimensiones:
      • Educativas: DIM_PLAN_CURRICULAR, DIM_LIBROS, DIM_POBLACION_MATRICULA.
      • Administrativas: DIM_CUENTA_CONTABLE, DIM_PERSONAL, DIM_INFRAESTRUCTURA_CCF.
      • Temporales: DIM_TIEMPO.
    • Hechos:
      • Operativos: FACT_TRANSPORTE, FACT_PERMISO_ESTUDIANTE, FACT_DIAGNOSTICOS_EE_JEC.
      • Financieros: FACT_DETALLE_CONTABLE, FACT_CONVENIOS, FACT_PRESUPUESTO.
    • Automatización:
      • Scripts Python para descargas y validaciones dinámicas.

    Paquetes Principales y Tablas Asociadas:

    1. 01-TRANSVERSAL_DIMENSIONES:

      • Tablas clave: DIM_CUENTA_CONTABLE, DIM_PERSONAL, DIM_INFRAESTRUCTURA_CCF.
      • Fuentes: Archivos Excel y bases de datos SQL Server.

    2. 02-TRANSVERSAL_FACT:

      • Tablas clave: FACT_DETALLE_CONTABLE, FACT_ENCUESTAS_PSR, FACT_INICIATIVAS.
      • Fuentes: Archivos CSV, bases de datos SAP.

    3. 03-COLEGIO_DIMENSIONES y 03-COLEGIO_DIMENSIONES_AUXILIAR:

      • Tablas clave: DIM_CURSO, DIM_GRADO, DIM_PLAN_CURRICULAR.
      • Fuentes: Bases remotas y archivos Excel.

    4. 04-COLEGIO_FACT:

      • Tablas clave: FACT_TRANSPORTE, FACT_CUPOS_NEGADOS, FACT_BIBLIOTECA.
      • Fuentes: Bases SAP y datos operativos.

    5. 05-CEDESARROLLO_DIMENSIONES:

      • Tablas clave: DIM_PERIODO_ACADEMICO, DIM_JORNADA.
      • Fuentes: Archivos Excel y bases SQL Server.

    6. 06-CEDESARROLLO_FACT:

      • Tablas clave: FACT_NOTAS, FACT_TRANSPORTE, FACT_PERMISO_ESTUDIANTE.
      • Fuentes: Bases de datos académicas.

    7. 07-PROTECCION_DIMENSIONES y 08-PROTECCION_FACT:

      • Tablas clave: DIM_PREGUNTAS_EE_JEC, FACT_ENTREGA_MATERIAL, FACT_DESERCION.
      • Fuentes: Archivos planos y bases SQL Server.

    Detalles Técnicos Generales¶

    Extracción de Datos¶

    • Fuentes Primarias:
      • Bases SAP y SQL Server.
      • Archivos CSV y Excel.
      • Conexiones SharePoint para datos remotos.
    • Instrumentos:
      • ADO.NET para bases estructuradas.
      • Scripts Python para descargas programáticas.

    Transformaciones¶

    • Validaciones:
      • Uso extensivo de Lookup para garantizar integridad referencial.
    • Transformaciones Especializadas:
      • Columnas derivadas para claves auxiliares y valores predeterminados.
      • Clasificación con Conditional Split.
      • Conversión de tipos para alineación de esquemas.

    Carga de Datos¶

    • Tablas de Dimensiones:
      • DIM_CUENTA_CONTABLE, DIM_PROGRAMA, DIM_ESTUDIANTES.
    • Tablas de Hechos:
      • FACT_TRANSPORTE, FACT_CONVENIOS, FACT_DIAGNOSTICOS_EE_JEC.
    • Optimización:
      • Inserciones masivas (Bulk Insert) configuradas para alto rendimiento.

    Automatización¶

    • Scripts Python:
      • Validaciones dinámicas en tiempo de ejecución.
      • Descarga de archivos con integración SharePoint.
    • Procedimientos Almacenados:
      • Restauración automatizada de reglas de integridad referencial.

    Herramientas y Tecnologías Utilizadas¶

    • SSIS (SQL Server Integration Services):
      • Para diseñar y ejecutar flujos de datos y procesos de control.
    • Python:
      • Automatización y validación de datos mediante scripts personalizados.
    • ADO.NET:
      • Conexiones robustas a bases de datos relacionales.
    • OLE DB:
      • Lectura y transformación de datos desde archivos Excel y CSV.
    • SQL Server:
      • Plataforma de destino para almacenar datos procesados.
    • Visual Studio:
      • Desarrollo y configuración de paquetes SSIS.

    Buenas Prácticas¶

    1. Estandarización:

      • Nombrado uniforme para conexiones y componentes.
      • Plantillas comunes en transformaciones y cargas.

    2. Optimización:

      • Configuración de caché para Lookup.
      • Paralelización en flujos intensivos de datos.

    3. Automatización:

      • Scripts Python para minimizar intervenciones manuales.
      • Uso de variables dinámicas para parametrización.

    4. Mantenimiento:

      • Validaciones exhaustivas previas a la carga.
      • Auditorías periódicas de consistencia.

    5. Seguridad:

      • Eliminación temporal de restricciones durante cargas masivas.
      • Restauración automática tras completarse las operaciones.

    Conclusión¶

    La solución SSIS documentada constituye un pilar estratégico para la gestión y explotación de datos organizacionales. Su diseño modular y altamente automatizado no solo garantiza eficiencia operativa, sino que también habilita a la organización para responder ágilmente a desafíos futuros. La implementación de buenas prácticas y recomendaciones adicionales potenciará su valor y sostenibilidad en el tiempo.

    Parámetros Clave en la Solución SSIS¶

    Documentación del Archivo project.params¶

    El archivo project.params es un componente esencial en las soluciones de SQL Server Integration Services (SSIS), diseñado para centralizar y administrar los parámetros utilizados globalmente en los paquetes de la solución. Su propósito principal es facilitar la configuración y gestión de variables críticas, proporcionando un punto único de control para ajustar valores clave sin necesidad de modificar cada paquete individualmente.

    Los parámetros definidos en este archivo son utilizados para:

    1. Cadenas de conexión:

      • Administrar el acceso a bases de datos y servicios externos.
      • Establecer conexiones consistentes y seguras mediante cadenas configuradas que pueden ser reutilizadas en múltiples paquetes.

    2. Rutas de trabajo:

      • Definir directorios y ubicaciones clave para archivos de entrada, salida o temporales.
      • Estandarizar las rutas utilizadas en scripts y procesos, reduciendo la posibilidad de errores debido a inconsistencias.

    3. Configuraciones de tiempo de espera:

      • Establecer valores predeterminados para manejar operaciones que requieren límites temporales, como conexiones a bases de datos o transferencias de datos.

    4. Parámetros sensibles:

      • Incluir contraseñas, claves de acceso y otras credenciales protegidas mediante la propiedad Sensitive que oculta estos valores en registros y exportaciones.

    5. Compatibilidad multi-entorno:

      • Facilitar la portabilidad y configuración de los paquetes SSIS en diferentes entornos (desarrollo, prueba, producción) mediante valores fácilmente ajustables en un único archivo.

    Ventajas del Uso de project.params¶

    1. Centralización: Permite un punto único de configuración para parámetros globales, simplificando la administración.
    2. Flexibilidad: Cambiar valores en el archivo afecta automáticamente todos los paquetes que dependen de estos parámetros, eliminando la necesidad de ediciones manuales en múltiples archivos.
    3. Estandarización: Garantiza consistencia en la configuración entre paquetes, reduciendo errores y aumentando la mantenibilidad.
    4. Seguridad: Los parámetros sensibles pueden ser protegidos para evitar la exposición de información confidencial.

    Estructura del Archivo¶

    Cada parámetro en el archivo incluye: - Nombre: Identificador único del parámetro. - Descripción: Información opcional que detalla el propósito del parámetro. - Propiedades clave: - Sensitive: Indica si el parámetro contiene información sensible que debe ser protegida. - DataType: Especifica el tipo de dato (e.g., cadena, número). - Valor: El valor asignado al parámetro, que puede ser una cadena de conexión, ruta, número, entre otros.

    El diseño del archivo project.params permite a los equipos de desarrollo y operaciones trabajar de manera eficiente y segura, garantizando que los procesos de integración de datos sean robustos y escalables.

    1. DWH_COMFENALCO_ConnectionString¶

    • Propósito: Cadena de conexión para la base de datos principal DWH_COMFENALCO, utilizada para operaciones de carga y extracción de datos.
    • Ejemplo de Valor: 
      Data Source=10.5.21.29\bi;User ID=prov_quality1;Initial Catalog=DWH_COMFENALCO;Persist Security Info=True;...
    • Propiedades Clave:
    • Sensitive: 0 (No contiene información sensible).
    • DataType: 18 (Texto).

    2. DWH_COMFENALCO_Destino_ConnectionString¶

    • Propósito: Cadena de conexión para la base de datos de destino DWH_COMFENALCO.
    • Ejemplo de Valor: 
      Data Source=QCSCONS19;Initial Catalog=DWH_COMFENALCO;Integrated Security=True;Encrypt=False;...
    • Propiedades Clave:
    • Sensitive: 0.
    • DataType: 18.

    3. DWH_COMFENALCO_Destino_OLEDB_ConnectionString¶

    • Propósito: Cadena de conexión OLEDB para operaciones específicas en la base de datos DWH_COMFENALCO.
    • Ejemplo de Valor: 
      Data Source=QCSCONS19;Initial Catalog=DWH_COMFENALCO;Provider=MSOLEDBSQL.1;Integrated Security=SSPI;...
    • Propiedades Clave:
    • Sensitive: 0.
    • DataType: 18.

    4. DWH_COMFENALCO_OLEDB_ConnectionString¶

    • Propósito: Cadena de conexión OLEDB para la base de datos principal DWH_COMFENALCO, diseñada para compatibilidad con aplicaciones que utilizan este proveedor.
    • Ejemplo de Valor: 
      Data Source=10.5.21.29\bi;User ID=prov_quality1;Provider=MSOLEDBSQL.1;Persist Security Info=True;...
    • Propiedades Clave:
    • Sensitive: 0.
    • DataType: 18.

    5. DWH_COMFENALCO_OLEDB_UserName¶

    • Propósito: Nombre de usuario utilizado para la autenticación en la conexión OLEDB de DWH_COMFENALCO.
    • Ejemplo de Valor: prov_quality1.
    • Propiedades Clave:
    • Sensitive: 0.
    • DataType: 18.

    6. Python_Executable¶

    • Propósito: Ruta al ejecutable de Python que se utiliza en los scripts del proyecto.
    • Ejemplo de Valor: 
      \COMFENALCO_EDUCACION\01.Scripts\Utils\env\Scripts\python.exe
    • Propiedades Clave:
    • Sensitive: 0.
    • DataType: 18.

    7. SAP_ERP_ConnectionString¶

    • Propósito: Cadena de conexión para acceder al sistema SAP ERP.
    • Ejemplo de Valor:

    • Sensitive: 0.
    • DataType: 18.

    8. STAGE_AREA_ConnectionString¶

    • Sensitive: 0.

    • Propósito: Configuración del tiempo de espera (en segundos) para operaciones de conexión.

    • Propiedades Clave:
    • DataType: 9 (Entero).

    12. Working_Directory¶

    • Propósito: Directorio de trabajo utilizado para guardar archivos temporales y otros datos procesados por los paquetes de SSIS.
    • Ejemplo de Valor:

    • Sensitive: 0.
