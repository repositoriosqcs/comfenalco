    Instalación y Configuración del Entorno para el Proyecto Comfenalco¶

    Este apartado detalla los pasos necesarios para configurar un entorno funcional que permita la implementación y ejecución de los procesos de carga, transformación y análisis de datos utilizados en el proyecto de Comfenalco. El objetivo principal es garantizar una instalación uniforme de herramientas y dependencias, configurar correctamente las conexiones a las bases de datos y estructurar el entorno para el óptimo funcionamiento del Data Warehouse (DWH).

    Pasos para la Configuración del Entorno¶

    1. Instalación de Herramientas Necesarias¶

    Versionamiento de Herramientas¶

    Los paquetes ETL del proyecto Comfenalco se desarrollaron originalmente utilizando Visual Studio Community 2022, según las definiciones establecidas en la fase 1 del proyecto. Posteriormente, en la entrega del proyecto, Comfenalco degrada a la versión Visual Studio Professional 2019 para garantizar compatibilidad con los estándares de la organización.

    Entre el 14 y el 17 de enero de 2025, se llevaron a cabo mesas de trabajo conjuntas con el equipo de Comfenalco. Durante estas sesiones, se validaron los paquetes y se realizó la migración al servidor de pruebas, asegurando el correcto funcionamiento en el entorno de la organización.

    1.1 SQL Server Management Studio (SSMS)¶

    Requisitos:

    • Versión 20.1.10.0 o posterior.
    • Disponible en el siguiente enlace: Descargar SSMS.

    1.2 Visual Studio¶

    Versión:

    • Comfenalco utiliza Visual Studio Profesional 2019 (paga).
    • Para desarrollo, se utiliza Visual Studio Community 2022 (gratuita), versión 17.11.5.
    • Disponible en el siguiente enlace: Descargar Visual Studio Community.

    1.3 Integration Services (SSIS)¶

    Pasos de Instalación:

    1. Crear un proyecto en blanco en Visual Studio.
    2. Ir a la pestaña Extensiones → Administrar Extensiones.
    3. Buscar Integration Services e instalar la versión 1.3.2 (compatibilidad con Visual Studio 17.11.5).

    1.4 Analysis Services (SSAS)¶

    Pasos de Instalación:

    1. Crear un proyecto en blanco en Visual Studio.
    2. Ir a la pestaña Extensiones → Administrar Extensiones.
    3. Buscar Analysis Services e instalar la versión más reciente.
    4. Validar que no presente problemas de compatibilidad.

    Recomendaciones:¶

    Se recomienda dejar las mismas estructuras. La actualización de datos de consultas y web scraping es automática. Y la de archivos corresponde a los propietarios de la información.

    2. Creación del Entorno Virtual¶

    Un entorno virtual permite organizar y aislar las dependencias necesarias para este proyecto. Siga estos pasos:

    1. Cree un entorno virtual ejecutando el siguiente comando:

    python -m venv comfenalco_env

    2. Active el entorno virtual:

    En Windows:

    comfenalco_env\Scripts\activate

    En Linux/macOS:

    source comfenalco_env/bin/activate

    3. Instalación de Dependencias¶

    Una vez activado el entorno virtual, instale las librerías necesarias:

    pip install sqlalchemy pandas beautifulsoup4 numpy logging python-dateutil concurrent.futures

    Configuración Inicial del Proyecto¶

    1. Creación de Conexiones¶

    1. Configuración en SSIS:

    • Use el administrador de conexiones ADO.NET.
    • Cree las siguientes conexiones:
      • DWH_COMFENALCO
      • STAGE_AREA
      • SAP_ERP
    • Es importante tener activa la VPN al configurar las conexiones.

    2. Configuración Local:

    • Cree una base de datos local en SQL Server Management Studio llamada DWH_COMFENALCO_local para realizar pruebas.
    • Cree una conexión a esta base local.

    3. Parametrización de Conexiones:

    • Configure las variables ConnectionString y Password en el proyecto de SSIS para cada conexión.
    • Consulte el siguiente video (minuto 11 en adelante).

    4. Referencia de Proyecto:

    • Abra el proyecto de referencia disponible en: Proyecto SSIS.

    Funciones Principales del Proyecto¶

    1. Configuración del Logger¶

    Para garantizar un registro de eventos y errores, utilice la función setup_logger:

    from Funciones import setup_logger
import logging

logger = setup_logger(log_filename='etl.log', log_level=logging.INFO)
logger.info("Proceso de configuración iniciado.")

    2. Conexión a la Base de Datos¶

    Configure la conexión a las bases de datos con la función obtener_conexion y el uso de SQLAlchemy:

    from Funciones import obtener_conexion
from sqlalchemy import create_engine

cadena_conexion = obtener_conexion('nombre_base_datos')
motor = create_engine(cadena_conexion)

    3. Funciones de Procesamiento¶

    3.1 guardar_en_dwh¶

    • Propósito: Guarda un DataFrame en una tabla específica del DWH.
    • Uso:
    from Funciones import guardar_en_dwh

guardar_en_dwh(df, 'nombre_tabla', logger, multiple=False, if_exists='replace')

    3.2 limpiar_html¶

    • Propósito: Limpia el contenido de texto con etiquetas HTML.
    • Uso:
    from Funciones import limpiar_html

texto_limpio = limpiar_html("<p>Ejemplo</p>")
print(texto_limpio)  # Salida: Ejemplo

    3.3 StoreDuplicated¶

    • Propósito: Guarda registros duplicados para análisis posterior.
    • Uso:
    from Funciones import StoreDuplicated

StoreDuplicated("duplicados.xlsx", ['columna1', 'columna2'], df, './output')

    Creación del Cubo¶

    1. Configuración del Cubo:

    • Use Analysis Services para crear un cubo que contenga las medidas y KPIs necesarios.
    • Incluya esquemas de datos de Transversal, Colegio, Cedesarrollo y Protección.

    2. Población de Tablas:

    • Llene primero las tablas Dim y posteriormente las tablas Fact.
    • Asegúrese de que las tablas de DWH estén vacías antes de iniciar el proceso de carga.

    3. Consumo en Power BI:

    • Todas las medidas deben estar definidas en el cubo, ya que Power BI no permite la creación de medidas en tiempo de consulta.

    Ejemplo de Flujo ETL¶

    from Funciones import obtener_conexion, guardar_en_dwh, setup_logger
import pandas as pd
from sqlalchemy import create_engine

# Configurar logger
logger = setup_logger(log_filename='etl_proceso.log', log_level=logging.INFO)

# Conectar a la base de datos
cadena_conexion = obtener_conexion('dwh_comfenalco')
motor = create_engine(cadena_conexion)

# Leer datos de origen
df_origen = pd.read_sql_query("SELECT * FROM tabla_origen", motor)

# Transformaciones necesarias
df_origen['nueva_columna'] = df_origen['columna_existente'] * 2

# Cargar datos transformados al DWH
guardar_en_dwh(df_origen, 'tabla_destino', logger, multiple=False, if_exists='append')

    Conclusión¶

    Con estos pasos, el entorno estará listo para manejar los procesos ETL del proyecto Comfenalco. Al seguir este flujo, se asegura una configuración consistente, se minimizan errores y se facilita la integración de nuevos módulos o expansiones del DWH.
