    03. ETL

    El paquete SSIS "FACT_MINERIA" está diseñado para gestionar procesos ETL (Extracción, Transformación y Carga) enfocados en datos de actividades empresariales, afiliaciones, aportes, personal y evaluaciones. Este paquete asegura la correcta preparación de tablas, la consolidación de datos y la integración eficiente en el Data Warehouse DWH_COMFENALCO, proporcionando una base sólida para análisis y generación de reportes.

    El paquete SSIS "FACT_MINERIA" centraliza datos clave de actividades empresariales, afiliaciones, aportes, personal y evaluaciones, asegurando su calidad y consistencia. Además, prepara las tablas destino para la carga eficiente y garantiza la integridad referencial de los datos.

    Este paquete se encarga de:

    1. Extracción de Datos: Recopila información de diversas fuentes, incluyendo archivos Excel, CSV y bases de datos, para asegurar una cobertura completa de los datos necesarios.
    2. Transformación de Datos: Realiza conversiones de tipos, validaciones y derivaciones de columnas para asegurar que los datos sean consistentes y estén listos para su análisis.
    3. Carga de Datos: Inserta los datos transformados en el Data Warehouse DWH_COMFENALCO, utilizando técnicas de inserción masiva para optimizar el rendimiento.
    4. Preparación de Tablas: Limpia y crea tablas clave, asegurando que estén listas para recibir nuevos datos sin duplicaciones ni inconsistencias.
    5. Validaciones: Verifica la integridad referencial y la consistencia de los datos, asegurando que todas las relaciones entre tablas sean correctas y que no haya datos faltantes o incorrectos.

    Descripción del Paquete¶

    1. Preparación de Tablas:

    • Limpieza y creación de tablas clave como:
      • FACT_UNIDADES
      • FACT_ACTIVIDADES
      • FACT_EVALUACION_DOCENTE
      • FACT_FINANCIERA
      • FACT_PERSONAL
      • FACT_MINERIA
      • DIM_TIEMPO_MENSUAL
    • Acciones realizadas:
      • Si las tablas existen: TRUNCATE TABLE.
      • Si no existen: CREATE TABLE.

    2. Extracción de Datos:

    • Fuentes principales:
      • Archivos Excel y CSV.
      • Bases de Datos:
      • DIM_EMPRESAS
      • DIM_AFILIADOS
      • FACT_APORTES_SHR_DET
      • FACT_PERSONAL
      • FACT_FINANCIERA
    • Herramientas utilizadas:
      • Conexiones ADO.NET para consultas optimizadas.
      • Scripts Python para automatización de validaciones y restauraciones.
      • Paquetes SSIS para la integración y transformación de datos.

    3. Transformación de Datos:

    • Comprobaciones en tablas maestras como DIM_TIEMPO_MENSUAL y DIM_POBLACION.
    • Conversión de Tipos (Data Conversion):
      • Asegura la compatibilidad de los tipos de datos con las tablas destino.
    • Columnas Derivadas (Derived Column):
      • Generación de valores calculados como ID_FECHA y agregados mensuales.
    • Validaciones (Lookup):
      • Verificación de integridad referencial y consistencia de datos.

    4. Carga de Datos en el Data Warehouse:

    • Tablas procesadas:
      • FACT_ACTIVIDADES
      • FACT_PERSONAL
      • FACT_EVALUACION_DOCENTE
      • FACT_FINANCIERA
      • FACT_MINERIA
      • DIM_TIEMPO_MENSUAL
    • Configuración:
      • Inserciones masivas habilitadas para maximizar el rendimiento.
      • Uso de conexiones ADO.NET y scripts Python para optimizar la carga y validación de datos.

    Componentes¶

    Componente Truncar Tablas Cubo¶

    Descripción General¶

    La tarea Truncar Tablas Cubo es un componente de ejecución SQL en un paquete SSIS diseñado para preparar el entorno del Data Warehouse (DWH_COMFENALCO) antes de la carga de datos. Esta tarea asegura que las tablas destino estén vacías eliminando todo su contenido mediante comandos TRUNCATE TABLE.

    Propósito¶

    1. Preparar el Data Warehouse:
      • Garantizar que las tablas destino no contengan datos previos que puedan causar duplicación o inconsistencias.
    2. Optimizar el rendimiento:
      • Al usar TRUNCATE TABLE, se eliminan rápidamente todos los registros de las tablas sin registrar las operaciones de borrado individualmente.

    Tablas Afectadas¶

    El componente ejecuta los comandos de truncado en las siguientes tablas:

    1. [DWH_COMFENALCO].[Transversal].[FACT_UNIDADES]

      • Tabla de hechos que contiene información relacionada con unidades operativas.

    2. [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES]

      • Almacena datos consolidados de actividades relacionadas con afiliados, empresas y aportes.

    3. [DWH_COMFENALCO].[Transversal].[FACT_EVALUACION_DOCENTE]

      • Contiene evaluaciones de desempeño docente.

    4. [DWH_COMFENALCO].[Transversal].[FACT_FINANCIERA]

      • Tabla de hechos para datos financieros.

    5. [DWH_COMFENALCO].[Transversal].[FACT_PERSONAL]

      • Registra datos relacionados con personal, como ausentismo y contrataciones.

    6. [DWH_COMFENALCO].[Transversal].[FACT_MINERIA]

      • Incluye información relevante a procesos de minería de datos en el análisis educativo.

    7. [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]

      • Dimensión que almacena información temporal organizada por meses.

    Implementación del SQL¶

    Consulta SQL Utilizada:

    TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_UNIDADES];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_EVALUACION_DOCENTE];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_FINANCIERA];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_PERSONAL];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_MINERIA];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL];

    Características del Comando:

    • Rápido: Elimina todas las filas de las tablas especificadas sin afectar la estructura.
    • Seguro: Restringe la eliminación si hay claves foráneas activas que referencian las tablas.

    Componente DIM_TIEMPO_MENSUAL¶

    Descripción General¶

    El componente DIM_TIEMPO_MENSUAL es una tarea de flujo de datos (Data Flow Task) en un paquete SSIS diseñada para cargar y mantener la tabla de dimensión temporal mensual en el Data Warehouse. Extrae datos de la tabla Dwh.DIM_TIEMPO, filtra los registros correspondientes al primer día de cada mes de los últimos tres años, y los carga en la tabla Transversal.DIM_TIEMPO_MENSUAL. Este proceso garantiza que la dimensión temporal esté actualizada para soportar análisis mensuales.

    Propósito¶

    1. Construir la dimensión temporal mensual:
      • Proporcionar una estructura de datos consolidada para análisis basados en meses, trimestres, semestres y años.
    2. Optimizar consultas analíticas:
      • Facilitar agregaciones rápidas y filtros temporales en cubos OLAP o informes.
    3. Garantizar integridad temporal:
      • Asegurar que solo se incluyan fechas válidas y relevantes (hasta junio de 2024 para la construcción inicial).

    Tablas Afectadas¶

    1. [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]:
      • Tabla de origen que contiene datos detallados de tiempo (días, semanas, meses, etc.).
    2. [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]:
      • Tabla destino que almacena datos temporales agregados por mes, con jerarquías precalculadas (trimestres, semestres, años).

    Consulta SQL Utilizada en el Origen (ADO NET):

    SELECT 
        [ID_FECHA],
        [FECHA],
        [DESC_FECHA],
        [ID_SEMANA],
        [DESC_SEMANA],
        [ID_NO_MES],
        [DESC_NO_MES],
        [ID_MES],
        [DESC_MES],
        [DESC_MES_CORTA],
        [ID_BIMESTRE],
        [DESC_BIMESTRE],
        [ID_TRIMESTRE],
        [DESC_TRIMESTRE],
        [ID_CUATRIMESTRE],
        [DESC_CUATRIMESTRE],
        [ID_SEMESTRE],
        [DESC_SEMESTRE],
        [ID_ANIO],
        [ID_ANIO_ANT],
        [NUM_DIA_SEMANA],
        [FESTIVO],
        [FECHA_CORTA]
    FROM 
        [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
    WHERE 
        DATEPART(DAY, FECHA) = 1
        AND ID_ANIO BETWEEN DATEPART(YEAR, GETDATE()) - 3 AND DATEPART(YEAR, GETDATE())
)
SELECT 
    [ID_FECHA],
    [FECHA],
    [DESC_FECHA],
    [ID_SEMANA],
    [DESC_SEMANA],
    [ID_NO_MES],
    [DESC_NO_MES],
    [ID_MES],

    Componente Afiliación Mensual Empresas (Paquete FACT_ACTIVIDADES)¶

    Descripción General¶

    Este componente corresponde a un Data Flow Task que realiza el proceso de extracción, transformación y carga (ETL) para registrar información de afiliación mensual de empresas en la tabla Transversal.FACT_ACTIVIDADES. El flujo está compuesto por dos componentes principales: Origen ADO NET y Destino ADO NET.

    Componentes del Flujo de Datos¶

    1. Empresas (Origen de ADO NET):

      • Descripción: Este componente extrae datos desde la base de datos SQL Server utilizando una instrucción SQL optimizada con múltiples CTEs.
      • Propiedades:
        • Consulta SQL: 
          WITH MinFecha AS (
    SELECT MIN(FECHA) AS MinFecha
    FROM [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]
),
MaxFecha AS (
    SELECT MAX(FECHA) AS MaxFecha
    FROM [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]
),
Empresas AS (
    SELECT
        [ID_EMPRESA],
        [PARTNER],
        CAST([FECHA_AFILIACION] AS DATE) AS [FECHA_AFILIACION],
        [FECHA_FUNDACION],
        CAST([FECHA_RETIRO] AS DATE) AS [FECHA_RETIRO],
        [ESTADO],
        [ESTADOREGISTRO],
        [COD_CIUDAD]
    FROM 
        [DWH_COMFENALCO].[Aportes].[DIM_EMPRESAS]
    WHERE [ESTADO] = 'VIGEN' AND [ESTADOREGISTRO] = 'CURRENT'
),
EmpresasOptimizado AS (
    SELECT 
        [ID_EMPRESA],
        [PARTNER],
        [FECHA_FUNDACION],
        CASE 
            WHEN [FECHA_AFILIACION] <= (SELECT MinFecha FROM MinFecha) THEN COALESCE(CAST((SELECT MinFecha FROM MinFecha) AS DATE), CAST(GETDATE() AS DATE))
            ELSE [FECHA_AFILIACION]
        END AS [FECHA_AFILIACION],
        CASE 
            WHEN [FECHA_RETIRO] >= (SELECT MaxFecha FROM MaxFecha) THEN COALESCE(CAST((SELECT MaxFecha FROM MaxFecha) AS DATE), CAST(GETDATE() AS DATE))
            ELSE [FECHA_RETIRO]
        END AS [FECHA_RETIRO],
        [ESTADOREGISTRO],
        [ESTADO],
        [COD_CIUDAD]
    FROM Empresas
),
EmpresasPorMes AS (
    SELECT 
        e.[ID_EMPRESA],
        e.[PARTNER],
        e.[FECHA_AFILIACION],
        e.[FECHA_FUNDACION],
        e.[FECHA_RETIRO],
        e.[ESTADOREGISTRO],
        e.[ESTADO],
        e.[COD_CIUDAD],
        ds.FECHA_CORTA AS FECHA_MENSUAL,
        ds.ID_FECHA,
        5 AS ID_UNIDAD
    FROM 
        EmpresasOptimizado e
    INNER JOIN 
        [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] ds ON ds.FECHA_CORTA BETWEEN e.[FECHA_AFILIACION] AND COALESCE(e.[FECHA_RETIRO], GETDATE())
)
SELECT 
    em.[PARTNER],
    em.[ID_EMPRESA],
    -1 AS ID_AFILIADO,
    -1 AS ID_TIPO_AFILIADO,
    NULL AS TIPO_AFILIADO,

    Componente Aportes Totales (Paquete FACT_ACTIVIDADES)¶

    Descripción General¶

    El componente Aportes Totales está diseñado para procesar datos relacionados con aportes mensuales de empresas y afiliados.       • Propiedades clave:
        • Conexión: DWH_COMFENALCO.
        • Consulta SQL:

        SELECT ISNULL(a.[BP_EMPRESA], '0000000000') as 'PARTNER',
            a.[ID_EMPRESA],
            a.[ID_AFILIADO],
            CONVERT(
                DATE,
                LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01'
            ) AS FECHA_MENSUAL,
            LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01' AS ID_FECHA,
            SUM(CONVERT(DECIMAL(18, 2), a.[APORTE])) AS TOTAL_APORTES,
            COUNT(a.[APORTE]) AS NUMERO_APORTES,
            ISNULL(a.[BP_EMPRESA], '0000000000') AS BP_EMPRESA,
            a.[BP_AFILIADO],
            'APORTES' AS ACTIVIDAD

            INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] t ON LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01' = CONVERT(VARCHAR(8), t.[ID_FECHA], 112)
        GROUP BY a.[ID_EMPRESA],
            a.[ID_AFILIADO],
            LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01',
            a.[BP_EMPRESA],
            a.[BP_AFILIADO]
    ),
    Poblacion_Educacion_Mes AS (
        SELECT DISTINCT [ID_FECHA],
            [PARTNER]
        FROM [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES]
        WHERE ACTIVIDAD != 'AFILIACION'
    ),
    AfiliadosMes AS (
        SELECT DISTINCT [ID_FECHA],
            [PARTNER],
            [PARTNER_AFILIADO]
        FROM [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES]

            AND PARTNER_AFILIADO IS NOT NULL

        SELECT am.[ID_FECHA],
            am.[PARTNER_AFILIADO]

    SELECT ap.* --,am.PARTNER_AFILIADO

            ELSE 'SI'

    FROM Aportes_Total ap
        LEFT JOIN AfiliadosEducacionMes aem ON ap.ID_FECHA = aem.ID_FECHA
        AND ap.BP_AFILIADO = aem.PARTNER_AFILIADO

    Componente FACT_EVALUACION_DOCENTE_CUBO¶

    Descripción General¶

    El componente FACT_EVALUACION_DOCENTE_CUBO es una tarea de flujo de datos (Data Flow Task) diseñada para consolidar y cargar datos de evaluaciones de desempeño docente desde múltiples fuentes en el Data Warehouse. Combina información de tres tablas principales (Colegio, Cedesarrollo) y la integra en la tabla de hechos Transversal.FACT_EVALUACION_DOCENTE, asegurando una visión unificada para análisis académicos.

    Propósito¶

    1. Consolidar evaluaciones docentes:
      • Unificar datos de desempeño docente de diferentes sistemas (Colegio y Cedesarrollo).
    2. Facilitar análisis temporal:
      • Vincular las evaluaciones con la dimensión temporal mensual (DIM_TIEMPO_MENSUAL) para análisis históricos.
    3. Optimizar consultas analíticas:
      • Proporcionar una estructura de datos estandarizada para reportes de calidad educativa.

    Tablas Afectadas¶

    1. Origen:
      • [Colegio].[FACT_DESEMPENHO_DOCENTE]: Evaluaciones de docentes en colegios.
      • [Cedesarrollo].[FACT_DESEMPENHO_DOCENTE_CE]: Evaluaciones de docentes en programas de educación continua.
      • [Cedesarrollo].[FACT_DESEMPENHO_DOCENTE_DE]: Evaluaciones de docentes en educación para el desarrollo.
      • [Cedesarrollo].[DIM_PERIODO_ACADEMICO]: Dimensiones de períodos académicos.
    2. Destino:
      • [Transversal].[FACT_EVALUACION_DOCENTE]: Tabla de hechos consolidada para análisis de desempeño docente.

    Implementación del SQL¶

    Consulta SQL Utilizada (Origen ADO NET):

    SELECT PERIODO_ACADEMICO, ID_UNIDAD, ID_PERSONAL, NOMBRE_DOCENTE, 
       CALIFICACION_DEFINITIVA, uni.ID_FECHA 
FROM (
    -- Datos de Colegio
    SELECT 
        [ANIO_ACADEMICO] as PERIODO_ACADEMICO,
        NULL as ID_UNIDAD,
        [ID_PERSONAL],
        [NOMBRE_DOCENTE],
        [TOTAL_GENERAL] as CALIFICACION_DEFINITIVA,
        [ID_FECHA]
    FROM [Colegio].[FACT_DESEMPENHO_DOCENTE]

    UNION

    -- Datos de Cedesarrollo (Educación Continua)
    SELECT 
        c2.[PERIODO_ACADEMICO],
        c1.[ID_UNIDAD],
        c1.[ID_PERSONAL],
        c1.[NOMBRE_DOCENTE],
        c1.[CALIFICACION_DEFINITIVA],
        NULL as ID_FECHA  -- Campo pendiente de implementación
    FROM [Cedesarrollo].[FACT_DESEMPENHO_DOCENTE_CE] c1
    LEFT JOIN [Cedesarrollo].[DIM_PERIODO_ACADEMICO] c2 
        ON c1.ID_PERIODO = c2.ID_PERIODO

    UNION

    -- Datos de Cedesarrollo (Educación para el Desarrollo)
    SELECT 
        c2.[PERIODO_ACADEMICO],
        c1.[ID_UNIDAD],
        c1.[ID_PERSONAL],
        c1.[NOMBRE_DOCENTE],
        c1.[CALIFICACION] as CALIFICACION_DEFINITIVA,
        c1.[ID_FECHA]
    FROM [Cedesarrollo].[FACT_DESEMPENHO_DOCENTE_DE] c1
    LEFT JOIN [Cedesarrollo].[DIM_PERIODO_ACADEMICO] c2 
        ON c1.ID_PERIODO = c2.ID_PERIODO
) uni
INNER JOIN [Transversal].[DIM_TIEMPO_MENSUAL] dimM 
    ON uni.ID_FECHA = dimM.ID_FECHA;

    Descripción General¶

    El componente FACT_FINANCIERA_CUBO es un Data Flow Task diseñado para consolidar datos financieros de ejecución contable y presupuestos en el Data Warehouse DWH_COMFENALCO. Combina información de múltiples fuentes (detalles contables y presupuestarios) y realiza transformaciones críticas para cargar los datos en la tabla Transversal.FACT_FINANCIERA. El flujo incluye conversiones de datos y manejo de errores para garantizar la integridad de la información.

    Propósito¶

    1. Consolidación Financiera:

      • Integra datos de ejecución contable (FACT_DETALLE_CONTABLE) y presupuestos (FACT_PRESUPUESTO).
      • Genera métricas clave como ingresos, gastos, costos, activos y pasivos.

    2. Transformación de Datos:

      • Normaliza tipos de datos para compatibilidad con el esquema destino (ej: conversión de numéricos a cadenas).
      • Aplica lógica de agregación (SUM) y filtros para excluir categorías redundantes.

    3. Optimización para Análisis:

      • Estructura los datos para facilitar análisis multidimensionales en cubos OLAP.

    Tablas Afectadas¶

    1. [Transversal].[FACT_FINANCIERA]:

      • Tabla de hechos destino que almacena métricas financieras mensualizadas.
      • Columnas clave: ID_FECHA, ID_CEBE, IMPORTE, INGRESOS, GASTOS, ACTIVO, PASIVO.

    2. [Financiera].[DIM_CUENTA_CONTABLE]:

      • Proporciona descripciones y clasificaciones de cuentas contables.

    3. [Dwh].[DIM_TIEMPO]:

      • Dimensiona fechas para asociar registros a períodos mensuales.

    1. Origen ADO NET: FACT_FINANCIERA

      • Descripción: Extrae datos de las tablas FACT_DETALLE_CONTABLE y FACT_PRESUPUESTO usando la consulta SQL anterior.
      • Propiedades:
        • Conexión: DWH_COMFENALCO.
        • Tiempo de espera: 30 segundos.
        • Columnas clave: ID_FECHA, ID_CEBE, IMPORTE, INGRESOS, GASTOS.

    2. Conversión de Datos

      • Descripción: Convierte tipos de datos para evitar truncamientos y errores de inserción:

      • Columnas Afectadas:
        • SEGMENTO, SEGMENT, CUENTA_DESCRIPCION_SSF, ID_CEBE.

      • Descripción: Carga los datos procesados en Transversal.FACT_FINANCIERA.
      • Propiedades:
        • Inserción masiva: Habilitada (UseBulkInsertWhenPossible=true).

    Componente FACT_UNIDADES¶

    Descripción General¶

    El componente FACT_UNIDADES es una tarea de flujo de datos (Data Flow Task) en un paquete SSIS diseñado para consolidar y cargar datos relacionados con unidades operativas en la tabla Transversal.FACT_UNIDADES. Este flujo combina información de múltiples fuentes, incluyendo presupuestos, planes de cobertura, evaluaciones académicas y datos de matrículas, para generar un registro unificado de actividades educativas y operativas.

    Propósito¶

    1. Consolidación de Datos:
      • Integrar información de diversas tablas del Data Warehouse (DWH_COMFENALCO) para formar una vista unificada de las unidades operativas y educativas.
    2. Carga Eficiente:
      • Utilizar inserción masiva para optimizar el rendimiento al cargar datos en la tabla destino.
    3. Registro de Actividades:
      • Documentar actividades como cobertura proyectada, evaluaciones de diseño curricular, deserción estudiantil y resultados de pruebas académicas.

    Tablas Afectadas¶

    1. [Transversal].[FACT_UNIDADES]:
      • Tabla destino que almacena los datos consolidados de unidades operativas, incluyendo información de cursos, grados, programas y métricas de población estudiantil.

    Componente PQRs¶

    Descripción General¶

    El componente PQRs es una tarea de flujo de datos (Data Flow Task) dentro del paquete SSIS FACT_MINERIA. Su objetivo principal es procesar datos relacionados con PQR (Preguntas, Quejas y Reclamos) y cargarlos en la tabla Transversal.FACT_MINERIA. Este flujo incluye la extracción de datos desde una fuente, la aplicación de transformaciones y la carga en la tabla destino.

    Propósito¶

    1. Extracción de Datos:
      • Recuperar datos de PQR desde la tabla Transversal.FACT_PQRS y otras tablas relacionadas.
    2. Transformación de Datos:
      • Aplicar lógica de negocio para calcular campos como TIEMPO_SEGUNDOS (diferencia en segundos entre la creación y resolución de PQR) y ACTIVIDAD (concatenación de "PQR" con la causa).
    3. Carga de Datos:
      • Insertar los datos procesados en la tabla Transversal.FACT_MINERIA para su posterior análisis.

    Tablas Afectadas¶

    1. Origen:

      • [DWH_COMFENALCO].[Transversal].[FACT_PQRS]: Tabla que almacena los registros de PQR.
      • [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]: Tabla de dimensiones para el tiempo mensual.
      • [DWH_COMFENALCO].[Transversal].[DIM_AFILIADOS]: Tabla de dimensiones para afiliados.
      • [DWH_COMFENALCO].[Transversal].[DIM_BENEFICIARIOS]: Tabla de dimensiones para beneficiarios.

    2. Destino:

      • [Transversal].[FACT_MINERIA]: Tabla de hechos que almacena los datos procesados de PQR.

    Implementación del SQL¶

    Consulta SQL Utilizada en el Origen ADO NET:

    SELECT 
    a.[PARTNER] as BP,
    CONVERT(INT, LEFT(CAST(f.ID_FECHA AS VARCHAR), 6) + '01') AS ID_FECHA_MENSUAL,
    CONVERT(INT, LEFT(CAST(f.ID_FECHA AS VARCHAR), 4)) AS ID_ANIO_ACADEMICO,
    [ID_UNIDAD],
    [FECHA_CREACION] AS FECHA_INICIAL,
    [FECHA_RESOLUCION] AS FECHA_FINAL,
    CONCAT('PQR ', LOWER([CAUSA])) AS ACTIVIDAD,
    DATEDIFF(SECOND, [FECHA_CREACION], [FECHA_RESOLUCION]) AS TIEMPO_SEGUNDOS,
    0 AS VALOR_PAGADO,
    [ID_CATEGORIA],
    -1 AS ID_CURSO,
    -1 AS ID_TIPO_ESTUDIANTE
FROM [DWH_COMFENALCO].[Transversal].[FACT_PQRS] f
INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] tm 
    ON tm.[ID_FECHA] = CONVERT(INT, LEFT(CAST(f.ID_FECHA AS VARCHAR), 6) + '01')
INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_AFILIADOS] a 
    ON f.[ID_AFILIADO] = a.[ID_AFILIADO]
WHERE f.ID_AFILIADO != -1 AND ID_UNIDAD IN (2, 3)

UNION ALL

SELECT 
    b.[PARTNER] as BP,
    CONVERT(INT, LEFT(CAST(f.ID_FECHA AS VARCHAR), 6) + '01') AS ID_FECHA_MENSUAL,
    CONVERT(INT, LEFT(CAST(f.ID_FECHA AS VARCHAR), 4)) AS ID_ANIO_ACADEMICO,
    [ID_UNIDAD],
    [FECHA_CREACION] AS FECHA_INICIAL,
    [FECHA_RESOLUCION] AS FECHA_FINAL,
    CONCAT('PQR ', LOWER([CAUSA])) AS ACTIVIDAD,
    DATEDIFF(SECOND, [FECHA_CREACION], [FECHA_RESOLUCION]) AS TIEMPO_SEGUNDOS,
    0 AS VALOR_PAGADO,
    ISNULL(a.[ID_CATEGORIA], 4) AS ID_CATEGORIA,
    -1 AS ID_CURSO,
    -1 AS ID_TIPO_ESTUDIANTE
FROM [DWH_COMFENALCO].[Transversal].[FACT_PQRS] f
INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] tm 
    ON tm.[ID_FECHA] = CONVERT(INT, LEFT(CAST(f.ID_FECHA AS VARCHAR), 6) + '01')
INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_BENEFICIARIOS] b 
    ON f.[ID_BENEFICIARIO] = b.[ID_BENEFICIARIO]
LEFT JOIN [DWH_COMFENALCO].[Transversal].[DIM_AFILIADOS] a 
    ON b.[ID_AFILIADO] = a.[ID_AFILIADO]
WHERE f.ID_BENEFICIARIO != -1 AND ID_UNIDAD IN (2, 3)

      • ACTIVIDAD: Concatena "PQR" con la causa.
      • TIEMPO_SEGUNDOS: Calcula la diferencia en segundos entre la creación y resolución de la PQR.
    • Filtros:
