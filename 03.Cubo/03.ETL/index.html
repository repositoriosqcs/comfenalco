    ETLS CUBO¶

    El paquete SSIS "ETLS CUBO" está diseñado para gestionar procesos ETL (Extracción, Transformación y Carga) enfocados en datos transversales y financieros. Este paquete asegura la correcta preparación de tablas, la consolidación de datos y la integración eficiente en el Data Warehouse DWH_COMFENALCO, proporcionando una base sólida para análisis y generación de reportes.

    Propósito del Paquete¶

    El objetivo principal del paquete es centralizar datos clave de actividades empresariales, afiliaciones, aportes, personal y evaluaciones, asegurando su calidad y consistencia. Además, prepara las tablas destino para la carga eficiente y garantiza la integridad referencial de los datos.

    Descripción del Paquete¶

    1. Preparación de Tablas:

    • Limpieza y creación de tablas clave como:
      • FACT_ACTIVIDADES
      • FACT_PERSONAL
      • FACT_EVALUACION_DOCENTE
      • FACT_FINANCIERA
      • DIM_TIEMPO_MENSUAL
    • Acciones realizadas:
      • Si las tablas existen: TRUNCATE TABLE.
      • Si no existen: CREATE TABLE.

    2. Extracción de Datos:

    • Fuentes principales:
      • Archivos Excel y CSV.
      • Bases de Datos:
      • DIM_EMPRESAS
      • DIM_AFILIADOS
      • FACT_APORTES_SHR_DET
      • FACT_PERSONAL
      • FACT_FINANCIERA
    • Herramientas utilizadas:
      • Conexiones ADO.NET para consultas optimizadas.

    3. Transformación de Datos:

    • Validaciones (Lookup):
      • Comprobaciones en tablas maestras como DIM_TIEMPO_MENSUAL y DIM_POBLACION.
    • Conversión de Tipos (Data Conversion):
      • Garantiza compatibilidad con tablas destino.
    • Columnas Derivadas (Derived Column):
      • Generación de valores calculados como ID_FECHA y agregados mensuales.

    4. Carga de Datos en el Data Warehouse:

    • Tablas procesadas:
      • FACT_ACTIVIDADES
      • FACT_PERSONAL
      • FACT_EVALUACION_DOCENTE
      • FACT_FINANCIERA
      • DIM_TIEMPO_MENSUAL
    • Configuración:
      • Inserciones masivas habilitadas para maximizar el rendimiento.

    Tablas y Columnas Clave¶

    1. FACT_ACTIVIDADES:

    • PARTNER, ID_EMPRESA, ID_AFILIADO, FECHA_AFILIACION, TIPO_AFILIADO, ACTIVIDAD.

    2. FACT_PERSONAL:

    • ID_PERSONAL, NOMBRE, CONCEPTO, DESCRIPCION, FECHA_FIN.

    3. FACT_EVALUACION_DOCENTE:

    • PERIODO_ACADEMICO, ID_UNIDAD, ID_PERSONAL, NOMBRE_DOCENTE, CALIFICACION_DEFINITIVA.

    4. FACT_FINANCIERA:

    • ID_FECHA, ID_CEBE, DESCRIPCION_CEBE, DEPARTAMENTO, TOTAL, COSTOS.

    5. DIM_TIEMPO_MENSUAL:

    • ID_FECHA, FECHA, ID_MES, ID_ANIO, DESC_MES.

    Componentes¶

    Componente Truncar Tablas Cubo¶

    Descripción General¶

    La tarea Truncar Tablas Cubo es un componente de ejecución SQL en un paquete SSIS diseñado para preparar el entorno del Data Warehouse (DWH_COMFENALCO) antes de la carga de datos. Esta tarea asegura que las tablas destino estén vacías eliminando todo su contenido mediante comandos TRUNCATE TABLE.

    Propósito¶

    1. Preparar el Data Warehouse:
      • Garantizar que las tablas destino no contengan datos previos que puedan causar duplicación o inconsistencias.
    2. Optimizar el rendimiento:
      • Al usar TRUNCATE TABLE, se eliminan rápidamente todos los registros de las tablas sin registrar las operaciones de borrado individualmente.

    Tablas Afectadas¶

    El componente ejecuta los comandos de truncado en las siguientes tablas:

    1. [DWH_COMFENALCO].[Transversal].[FACT_UNIDADES]

      • Tabla de hechos que contiene información relacionada con unidades operativas.

    2. [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES]

      • Almacena datos consolidados de actividades relacionadas con afiliados, empresas y aportes.

    3. [DWH_COMFENALCO].[Transversal].[FACT_EVALUACION_DOCENTE]

      • Contiene evaluaciones de desempeño docente.

    4. [DWH_COMFENALCO].[Transversal].[FACT_FINANCIERA]

      • Tabla de hechos para datos financieros.

    5. [DWH_COMFENALCO].[Transversal].[FACT_PERSONAL]

      • Registra datos relacionados con personal, como ausentismo y contrataciones.

    6. [DWH_COMFENALCO].[Transversal].[FACT_MINERIA]

      • Incluye información relevante a procesos de minería de datos en el análisis educativo.

    7. [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]

      • Dimensión que almacena información temporal organizada por meses.

    Implementación del SQL¶

    Consulta SQL Utilizada:

    TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_UNIDADES];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_EVALUACION_DOCENTE];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_FINANCIERA];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_PERSONAL];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[FACT_MINERIA];
TRUNCATE TABLE [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL];

    Características del Comando:

    • Rápido: Elimina todas las filas de las tablas especificadas sin afectar la estructura.
    • Seguro: Restringe la eliminación si hay claves foráneas activas que referencian las tablas.

    Consideraciones Técnicas¶

    • Conexión: La tarea utiliza un administrador de conexiones definido en el proyecto (DWH_COMFENALCO).
    • Dependencias: Asegurarse de que no haya procesos concurrentes que dependan de los datos de las tablas durante la ejecución de esta tarea.
    • Restricciones de Claves Foráneas: Si las tablas tienen relaciones referenciales, es necesario deshabilitarlas temporalmente antes de ejecutar TRUNCATE TABLE.

    Componente Afiliación Mensual Empresas¶

    Descripción General¶

    Este componente corresponde a un Data Flow Task que realiza el proceso de extracción, transformación y carga (ETL) para registrar información de afiliación mensual de empresas en la tabla Transversal.FACT_ACTIVIDADES. El flujo está compuesto por dos componentes principales: Origen ADO NET y Destino ADO NET.

    Componentes del Flujo de Datos¶

    1. Empresas (Origen de ADO NET):

      • Descripción: Este componente extrae datos desde la base de datos SQL Server utilizando una instrucción SQL optimizada con múltiples CTEs.
      • Propiedades:
        • Consulta SQL: 
          WITH MinFecha AS (
    SELECT MIN(FECHA) AS MinFecha
    FROM [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]
),
MaxFecha AS (
    SELECT MAX(FECHA) AS MaxFecha
    FROM [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]
),
Empresas AS (
    SELECT
        [ID_EMPRESA],
        [PARTNER],
        CAST([FECHA_AFILIACION] AS DATE) AS [FECHA_AFILIACION],
        [FECHA_FUNDACION],
        CAST([FECHA_RETIRO] AS DATE) AS [FECHA_RETIRO],
        [ESTADO],
        [ESTADOREGISTRO],
        [COD_CIUDAD]
    FROM 
        [DWH_COMFENALCO].[Aportes].[DIM_EMPRESAS]
    WHERE [ESTADO] = 'VIGEN' AND [ESTADOREGISTRO] = 'CURRENT'
),
EmpresasOptimizado AS (
    SELECT 
        [ID_EMPRESA],
        [PARTNER],
        [FECHA_FUNDACION],
        CASE 
            WHEN [FECHA_AFILIACION] <= (SELECT MinFecha FROM MinFecha) THEN COALESCE(CAST((SELECT MinFecha FROM MinFecha) AS DATE), CAST(GETDATE() AS DATE))
            ELSE [FECHA_AFILIACION]
        END AS [FECHA_AFILIACION],
        CASE 
            WHEN [FECHA_RETIRO] >= (SELECT MaxFecha FROM MaxFecha) THEN COALESCE(CAST((SELECT MaxFecha FROM MaxFecha) AS DATE), CAST(GETDATE() AS DATE))
            ELSE [FECHA_RETIRO]
        END AS [FECHA_RETIRO],
        [ESTADOREGISTRO],
        [ESTADO],
        [COD_CIUDAD]
    FROM Empresas
),
EmpresasPorMes AS (
    SELECT 
        e.[ID_EMPRESA],
        e.[PARTNER],
        e.[FECHA_AFILIACION],
        e.[FECHA_FUNDACION],
        e.[FECHA_RETIRO],
        e.[ESTADOREGISTRO],
        e.[ESTADO],
        e.[COD_CIUDAD],
        ds.FECHA_CORTA AS FECHA_MENSUAL,
        ds.ID_FECHA,
        5 AS ID_UNIDAD
    FROM 
        EmpresasOptimizado e
    INNER JOIN 
        [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] ds ON ds.FECHA_CORTA BETWEEN e.[FECHA_AFILIACION] AND COALESCE(e.[FECHA_RETIRO], GETDATE())
)
SELECT 
    em.[PARTNER],
    em.[ID_EMPRESA],
    -1 AS ID_AFILIADO,
    -1 AS ID_TIPO_AFILIADO,
    NULL AS TIPO_AFILIADO,
    5 AS ID_CATEGORIA, 
    em.[FECHA_AFILIACION],
    em.[FECHA_RETIRO],
    -1 AS ID_GENERO,
    -1 AS ID_ESTADO_CIVIL,
    -1 AS ID_PERTENENCIA_ETNICA,
    -1 AS ID_FACTOR_VULNERABILIDAD,
    NULL AS ESTRATO,
    em.[COD_CIUDAD] AS ID_CIUDAD,
    NULL AS SALARIO_BASICO,
    em.[FECHA_MENSUAL],
    em.[ID_FECHA],
    em.[ID_UNIDAD],
    em.[ESTADOREGISTRO],
    -1 AS PARTNER_AFILIADO,
    em.[PARTNER] AS PARTNER_EMPRESA,
    'EMPRESA' AS TIPO_POBLACION,
    'AFILIACION' AS ACTIVIDAD
FROM 
    EmpresasPorMes em
ORDER BY 
    em.[ID_EMPRESA], em.[FECHA_MENSUAL];
        • Tiempo de espera del comando (CommandTimeout): 30 segundos.
        • Conexión: [DWH_COMFENALCO], manejado por un administrador de conexiones.

    2. Destino de ADO NET:

      • Descripción: Este componente carga los datos procesados en la tabla Transversal.FACT_ACTIVIDADES en la base de datos destino.
      • Propiedades:
        • Nombre de la tabla o vista: "Transversal"."FACT_ACTIVIDADES".
        • Tamaño del lote (BatchSize): 0 (utiliza el tamaño predeterminado del búfer).
        • Uso de inserción masiva (UseBulkInsertWhenPossible): Habilitado para mejorar el rendimiento.
        • Tiempo de espera del comando (CommandTimeout): 30 segundos.
        • Conexión: [DWH_COMFENALCO], manejado por un administrador de conexiones.

    Componente Afiliación mensual Afiliados¶

    Descripción General¶

    El componente Afiliación mensual Afiliados extrae, transforma y carga datos relacionados con afiliados en el sistema DWH_COMFENALCO. Este proceso genera datos mensualizados para afiliados activos y los carga en la tabla FACT_ACTIVIDADES.

    Componentes del Flujo de Datos¶

    1. Componente: Afiliados

      Descripción:
      Este componente realiza la extracción de datos de la tabla [DWH_COMFENALCO].[Aportes].[DIM_AFILIADOS], mensualiza la información y filtra afiliados activos.

      Propiedades:

      • SQLCommand: 

        WITH MaxFecha AS (
    SELECT MAX(FECHA) AS MaxFecha
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
NumberSeries AS (
    SELECT ROW_NUMBER() OVER (ORDER BY (SELECT NULL)) - 1 AS Number
    FROM [DWH_COMFENALCO].[Dwh].[DIM_TIEMPO]
),
DateSeries AS (
    SELECT 
        DATEADD(MONTH, Number, DATEFROMPARTS(YEAR(MaxFecha) - 3, 1, 1)) AS MonthDate,
        CONVERT(INT, FORMAT(DATEADD(MONTH, Number, DATEFROMPARTS(YEAR(MaxFecha) - 3, 1, 1)), 'yyyyMMdd')) AS ID_FECHA
    FROM 
        NumberSeries
    CROSS JOIN 
        MaxFecha
    INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] dtm 
        ON dtm.ID_FECHA = CONVERT(INT, FORMAT(DATEADD(MONTH, Number, DATEFROMPARTS(YEAR(MaxFecha) - 3, 1, 1)), 'yyyyMMdd'))
    WHERE 
        Number BETWEEN 0 AND DATEDIFF(MONTH, DATEFROMPARTS(YEAR(MaxFecha) - 3, 1, 1), MaxFecha)
),
AfiliadosPorMes AS (
    SELECT 
        a.[ID_AFILIADO], a.[PARTNER], a.[ID_TIPO_AFILIADO], a.[TIPO_AFILIADO],
        a.[ID_CATEGORIA], a.[FECHA_AFILIACION], a.[FECHA_RETIRO],
        a.[ID_GENERO], a.[ID_ESTADO_CIVIL], a.[ID_PERTENENCIA_ETNICA],
        a.[ID_FACTOR_VULNERABILIDAD], a.[ESTRATO], a.[ID_CIUDAD],
        a.[SALARIO_BASICO], a.[APORTANTE], a.[ESTADOREGISTRO],
        ds.MonthDate AS FECHA_MENSUAL, ds.ID_FECHA, 5 AS ID_UNIDAD
    FROM 
        [DWH_COMFENALCO].[Aportes].[DIM_AFILIADOS] a
    INNER JOIN 
        DateSeries ds 
        ON ds.MonthDate BETWEEN a.[FECHA_AFILIACION] AND COALESCE(a.[FECHA_RETIRO], GETDATE())
)
SELECT 
    am.[PARTNER], -1 AS ID_EMPRESA, am.[ID_AFILIADO], am.[ID_TIPO_AFILIADO],
    am.[TIPO_AFILIADO], am.[ID_CATEGORIA], am.[FECHA_AFILIACION],
    am.[FECHA_RETIRO], am.[ID_GENERO], am.[ID_ESTADO_CIVIL],
    am.[ID_PERTENENCIA_ETNICA], am.[ID_FACTOR_VULNERABILIDAD],
    am.[ESTRATO], am.[ID_CIUDAD], am.[SALARIO_BASICO],
    am.[FECHA_MENSUAL], am.[ID_FECHA], am.[ID_UNIDAD], am.[ESTADOREGISTRO],
    am.[PARTNER] AS PARTNER_AFILIADO, NULL AS PARTNER_EMPRESA,
    'AFILIADO' AS TIPO_POBLACION, 'AFILIACION' AS ACTIVIDAD
FROM 
    AfiliadosPorMes am;

      • CommandTimeout: 600 segundos

      • Columnas de salida principales:
        • PARTNER
        • ID_AFILIADO
        • FECHA_MENSUAL
        • TIPO_AFILIADO
        • ESTADOREGISTRO

    Componente Afiliación Mensual Beneficiarios¶

    Descripción General¶

    El componente Afiliación Mensual Beneficiarios es un flujo de datos que procesa y carga información relacionada con los beneficiarios de afiliaciones mensuales. Utiliza consultas SQL avanzadas para extraer, transformar y cargar datos desde la base de datos fuente hacia el destino en la tabla Transversal.FACT_ACTIVIDADES.

    SQL Utilizado¶

    El flujo de datos incluye la siguiente instrucción SQL para procesar la información:

    WITH MinFecha AS (
    SELECT MIN(FECHA) AS MinFecha
    FROM [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]
),
MaxFecha AS (
    SELECT MAX(FECHA) AS MaxFecha
    FROM [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL]
),
Beneficiarios AS (
    SELECT 
        [ID_BENEFICIARIO],
        [PARTNER],
        COALESCE(TRY_CAST([FECHA_AFILIACION] AS DATE), CAST(GETDATE() AS DATE)) as [FECHA_AFILIACION],
        CASE 
            WHEN COALESCE([FECHA_RETIRO], CAST(GETDATE() AS DATE)) = '1900-01-01' THEN '9999-12-31'
            ELSE COALESCE([FECHA_RETIRO], CAST(GETDATE() AS DATE))
        END AS [FECHA_RETIRO],
        [ID_GENERO],
        [ID_ESTADO_CIVIL],
        [ESTRATO],
        [ID_CIUDAD],
        [ID_PARENT_SAP],
        [PARENT_SAP],
        [APORTANTE],
        [TIPO_AFILIADO],
        [TIPO_APORTANTE],
        [ESTADOREGISTRO],
        FECHA_INGRESO_EMPRESA
    FROM 
        [DWH_COMFENALCO].[Aportes].[DIM_BENEFICIARIOS]
    WHERE 
        [ESTADOREGISTRO] = 'CURRENT' AND [TIPO_APORTANTE] = 'X' 
),
BeneficiariosPosibles AS (
    SELECT *,
        ROW_NUMBER() OVER (PARTITION BY PARTNER ORDER BY CASE WHEN FECHA_INGRESO_EMPRESA = '' THEN 99991231 END) AS ROWNUM
    FROM Beneficiarios
    WHERE FECHA_RETIRO >= (SELECT MinFecha FROM MinFecha)
),
FilteredBeneficiarios AS (
    SELECT *
    FROM BeneficiariosPosibles
    WHERE ROWNUM = 1
),
BeneficiariosOptimizado AS (
    SELECT 
        [ID_BENEFICIARIO],
        [PARTNER],
        CASE 
            WHEN [FECHA_AFILIACION] <= (SELECT MinFecha FROM MinFecha) THEN COALESCE(CAST((SELECT MinFecha FROM MinFecha) AS DATE), CAST(GETDATE() AS DATE))
            ELSE [FECHA_AFILIACION]
        END AS [FECHA_AFILIACION],
        CASE 
            WHEN [FECHA_RETIRO] >= (SELECT MaxFecha FROM MaxFecha) THEN COALESCE(CAST((SELECT MaxFecha FROM MaxFecha) AS DATE), CAST(GETDATE() AS DATE))
            ELSE [FECHA_RETIRO]
        END AS [FECHA_RETIRO],
        [ID_GENERO],
        [ID_ESTADO_CIVIL],
        [ESTRATO],
        [ID_CIUDAD],
        [ID_PARENT_SAP],
        [PARENT_SAP],
        [APORTANTE],
        [TIPO_AFILIADO],
        [TIPO_APORTANTE],
        [ESTADOREGISTRO]
    FROM FilteredBeneficiarios
),
BeneficiariosPorMes AS (
    SELECT 
        b.[ID_BENEFICIARIO],
        b.[PARTNER],
        b.[FECHA_AFILIACION],
        b.[FECHA_RETIRO],
        b.[ID_GENERO],
        b.[ID_ESTADO_CIVIL],
        b.[ESTRATO],
        b.[ID_CIUDAD],
        b.[ID_PARENT_SAP],
        b.[PARENT_SAP],
        b.[APORTANTE],
        b.[TIPO_AFILIADO],
        b.[TIPO_APORTANTE],
        b.[ESTADOREGISTRO],
        ds.FECHA_CORTA AS FECHA_MENSUAL,
        ds.ID_FECHA,
        ROW_NUMBER() OVER (PARTITION BY b.[ID_BENEFICIARIO], ds.ID_FECHA ORDER BY ds.FECHA_CORTA DESC) AS RN
    FROM 
        BeneficiariosOptimizado b
    INNER JOIN 
        [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] ds ON ds.FECHA_CORTA BETWEEN b.[FECHA_AFILIACION] AND COALESCE(b.[FECHA_RETIRO], GETDATE())
)
SELECT 
    bm.[PARTNER],
    -1 AS ID_EMPRESA,
    -1 AS ID_AFILIADO,
    bm.[ID_BENEFICIARIO],
    -1 AS ID_TIPO_AFILIADO,
    NULL AS TIPO_AFILIADO,
    -1 AS ID_CATEGORIA,
    bm.[FECHA_AFILIACION],
    bm.[FECHA_RETIRO],
    bm.[ID_GENERO],
    bm.[ID_ESTADO_CIVIL],
    -1 AS ID_PERTENENCIA_ETNICA,
    -1 AS ID_FACTOR_VULNERABILIDAD,
    bm.[ESTRATO],
    bm.[ID_CIUDAD],
    NULL AS SALARIO_BASICO,
    bm.[FECHA_MENSUAL],
    bm.[ID_FECHA],
    5 AS ID_UNIDAD,
    bm.[ESTADOREGISTRO],
    bm.[ID_PARENT_SAP] AS PARTNER_AFILIADO,
    bm.[PARENT_SAP] AS PARTNER_EMPRESA,
    'BENEFICIARIO' AS TIPO_POBLACION,
    'AFILIACION' AS ACTIVIDAD
FROM 
    BeneficiariosPorMes bm
WHERE bm.RN = 1;

      • Descripción: Extrae datos desde la tabla de aportes utilizando una consulta SQL con múltiples CTEs (Common Table Expressions) para calcular aportes totales, número de aportes, y la población educativa asociada.
      • Propiedades clave:
        • Conexión: DWH_COMFENALCO.
        • Consulta SQL: 
              WITH Aportes_Total AS (
        SELECT ISNULL(a.[BP_EMPRESA], '0000000000') as 'PARTNER',
            a.[ID_EMPRESA],
            a.[ID_AFILIADO],
            CONVERT(
                DATE,
                LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01'
            ) AS FECHA_MENSUAL,
            LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01' AS ID_FECHA,
            SUM(CONVERT(DECIMAL(18, 2), a.[APORTE])) AS TOTAL_APORTES,
            COUNT(a.[APORTE]) AS NUMERO_APORTES,
            ISNULL(a.[BP_EMPRESA], '0000000000') AS BP_EMPRESA,
            a.[BP_AFILIADO],
            'APORTES' AS ACTIVIDAD
        FROM [DWH_COMFENALCO].[Aportes].[FACT_APORTES_SHR_DET] a
            INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] t ON LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01' = CONVERT(VARCHAR(8), t.[ID_FECHA], 112)
        GROUP BY a.[ID_EMPRESA],
            a.[ID_AFILIADO],
            LEFT(CONVERT(VARCHAR(8), a.[FECHA_CONTABLE], 112), 6) + '01',
            a.[BP_EMPRESA],
            a.[BP_AFILIADO]
    ),
    Poblacion_Educacion_Mes AS (
        SELECT DISTINCT [ID_FECHA],
            [PARTNER]
        FROM [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES]
        WHERE ACTIVIDAD != 'AFILIACION'
    ),
    AfiliadosMes AS (
        SELECT DISTINCT [ID_FECHA],
            [PARTNER],
            [PARTNER_AFILIADO]
        FROM [DWH_COMFENALCO].[Transversal].[FACT_ACTIVIDADES]
        WHERE ACTIVIDAD = 'AFILIACION'
            AND PARTNER_AFILIADO IS NOT NULL
    ),
    AfiliadosEducacionMes AS (
        SELECT am.[ID_FECHA],
            am.[PARTNER_AFILIADO]
        FROM AfiliadosMes am
            INNER JOIN Poblacion_Educacion_Mes pem ON am.[ID_FECHA] = pem.[ID_FECHA]
            and am.PARTNER = pem.PARTNER
    )
    SELECT ap.* --,am.PARTNER_AFILIADO
    ,
    CASE
            WHEN aem.PARTNER_AFILIADO IS NULL THEN 'NO'
            ELSE 'SI'
        END AS POBLACION_EDUCACION
    FROM Aportes_Total ap
        LEFT JOIN AfiliadosEducacionMes aem ON ap.ID_FECHA = aem.ID_FECHA
        AND ap.BP_AFILIADO = aem.PARTNER_AFILIADO
          • Aportes_Total: Agrupa y calcula los aportes totales y el número de aportes por empresa, afiliado y mes.
          • Poblacion_Educacion_Mes: Identifica registros relacionados con actividades educativas en el mes.
          • AfiliadosMes: Extrae afiliados registrados en el mes correspondiente.
          • AfiliadosEducacionMes: Combina afiliados y población educativa para identificar asociaciones.
        • Tiempo de espera: 600 segundos.
        • Columnas principales:
          • PARTNER
          • ID_EMPRESA
          • ID_AFILIADO
          • FECHA_MENSUAL
          • ID_FECHA
          • TOTAL_APORTES
          • NUMERO_APORTES
          • ACTIVIDAD (valor constante: 'APORTES')
          • POBLACION_EDUCACION (indica 'SI' o 'NO' según la asociación con educación).

    Componente FACT_PERSONAL_CUBO¶

    Descripción General¶

    El componente FACT_PERSONAL_CUBO tiene como objetivo consolidar datos relacionados con el personal docente, sus contrataciones, ausencias y reemplazos, para luego cargarlos en la tabla Transversal.FACT_PERSONAL. Este flujo combina datos de múltiples fuentes, procesándolos para crear un registro unificado con detalles específicos sobre fechas, conceptos y horas asociadas.

    Componentes del Flujo de Datos¶

    1. Origen ADO NET: FACT_PERSONAL

      • Descripción: Extrae datos desde múltiples tablas de origen relacionadas con el personal docente, sus contrataciones, ausencias y reemplazos.
      • Propiedades clave:
        • Conexión: DWH_COMFENALCO.
        • Consulta SQL: 
              SELECT uni.ID_FECHA,
        uni.ID_PERSONAL,
        uni.NOMBRE,
        uni.CONCEPTO,
        uni.DESCRIPCION,
        uni.FECHA_FIN,
        uni.HORAS_CONTRATADAS_MENSUAL,
        pers.ID_UNIDAD,
        AUSENCIA_HORAS,
        TIPO_AUSENCIA
    FROM(
            SELECT LEFT([ID_FECHA], 6) * 100 + 1 as ID_FECHA,
                [ID_PERSONAL],
                [NOMBRE_DOCENTE] as NOMBRE,
                'AUSENTISMO' as CONCEPTO,
                [MOTIVO_AUSENCIA] as DESCRIPCION,
                [FECHA_FIN],
                NULL as HORAS_CONTRATADAS_MENSUAL,
                AUSENCIA_HORAS,
                TIPO_AUSENCIA
            FROM [DWH_COMFENALCO].[Cedesarrollo].[FACT_AUSENTISMO_DOCENTE]
            UNION
            SELECT LEFT([ID_FECHA], 6) * 100 + 1 as ID_FECHA,
                [ID_PERSONAL_REEMPLAZA] as ID_PERSONAL,
                [DOCENTE_REEMPLAZA] as NOMBRE,
                'REEMPLAZO' as CONCEPTO,
                NULL as DESCRIPCION,
                NULL as FECHA_FIN,
                NULL as HORAS_CONTRATADAS_MENSUAL,
                NULL as AUSENCIA_HORAS,
                NULL as TIPO_AUSENCIA
            FROM [DWH_COMFENALCO].[Colegio].[FACT_REEMPLAZO_DOCENTE]
            UNION
            SELECT LEFT([ID_FECHA], 6) * 100 + 1 as ID_FECHA,
                [ID_PERSONAL],
                [NOMBRE_DOCENTE] as NOMBRE,
                'AUSENTISMO' as CONCEPTO,
                [MOTIVO_AUSENCIA] as DESCRIPCION,
                [FECHA_FIN],
                NULL as HORAS_CONTRATADAS_MENSUAL,
                AUSENCIA_HORAS,
                TIPO_AUSENCIA
            FROM [DWH_COMFENALCO].[Colegio].[FACT_AUSENTISMO_DOCENTE]
            UNION
            SELECT LEFT(
                    CONVERT(
                        INT,
                        FORMAT([FECHA_INICIO_CONTRATACION], 'yyyyMMdd')
                    ),
                    6
                ) * 100 + 1 AS ID_FECHA,
                [ID_PERSONAL],
                [NOMBRE],
                'CONTRATACION' as CONCEPTO,
                NULL as DESCRIPCION,
                [FECHA_FIN_CONTRATACION] AS FECHA_FIN,
                [HORAS_CONTRATADAS_MENSUAL],
                NULL as AUSENCIA_HORAS,
                NULL as TIPO_AUSENCIA
            FROM [DWH_COMFENALCO].[Transversal].[DIM_PERSONAL]
            UNION
            SELECT LEFT(
                    CONVERT(
                        INT,
                        FORMAT([FECHA_FIN_CONTRATACION], 'yyyyMMdd')
                    ),
                    6
                ) * 100 + 1 AS ID_FECHA,
                [ID_PERSONAL],
                [NOMBRE],
                'FIN_CONTRATACION' as CONCEPTO,
                [CAUSA_TERMINACION_CONTRATO] as DESCRIPCION,
                NULL as FECHA_FIN,
                NULL as HORAS_CONTRATADAS_MENSUAL,
                NULL as AUSENCIA_HORAS,
                NULL as TIPO_AUSENCIA
            FROM [DWH_COMFENALCO].[Transversal].[DIM_PERSONAL]
        ) as uni --   WHERE [ID_FECHA] IS NOT NULL
        INNER JOIN [DWH_COMFENALCO].[Transversal].[DIM_TIEMPO_MENSUAL] dimM ON uni.ID_FECHA = dimM.ID_FECHA
        LEFT JOIN [DWH_COMFENALCO].[Transversal].[DIM_PERSONAL] pers ON uni.ID_PERSONAL = pers.ID_PERSONAL
          • FACT_AUSENTISMO_DOCENTE y FACT_REEMPLAZO_DOCENTE: Proveen información sobre ausencias y reemplazos.
          • DIM_PERSONAL: Agrega datos sobre contrataciones y terminaciones de contrato.
          • Join con DIM_TIEMPO_MENSUAL: Relaciona las fechas con el tiempo mensual.
          • Columnas seleccionadas:
            • ID_FECHA: Fecha procesada en formato mensual.
            • ID_PERSONAL: Identificador único del personal.
            • NOMBRE: Nombre del personal.
            • CONCEPTO: Identifica la categoría del registro (AUSENTISMO, REEMPLAZO, CONTRATACION, FIN_CONTRATACION).
            • DESCRIPCION: Información adicional (motivo de ausencia, causa de terminación, etc.).
            • FECHA_FIN: Fecha de finalización de contrato o ausencia.
            • HORAS_CONTRATADAS_MENSUAL: Horas mensuales contratadas.
            • AUSENCIA_HORAS y TIPO_AUSENCIA: Información específica de ausencias.
        • Tiempo de espera: 30 segundos.
        • Columnas de salida:
          • ID_FECHA, ID_PERSONAL, NOMBRE, CONCEPTO, DESCRIPCION, FECHA_FIN, HORAS_CONTRATADAS_MENSUAL, ID_UNIDAD, AUSENCIA_HORAS, TIPO_AUSENCIA.
