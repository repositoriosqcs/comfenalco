    01. ORIGEN Y MEDIDAS

    Origen de Datos del Cubo¶

    Descripción General¶

    El origen de datos es un componente estructurado que conecta el cubo de SSIS con el Data Warehouse principal. Esta conexión permite la extracción, transformación y carga (ETL) de datos necesarios para las operaciones analíticas.

    Detalles de Configuración¶

    1. Tipo de Origen:

    • Estructurado (structured).

    2. Nombre del Origen:

    • SQL/10 5 21 29\\bi;DWH_COMFENALCO.

    3. Detalles de Conexión:

    • Protocolo: tds (Tabular Data Stream, utilizado para bases de datos SQL Server).
    • Dirección:
      • Servidor: 10.5.21.29\bi.
      • Base de Datos: DWH_COMFENALCO.

    4. Autenticación:

    • Tipo: UsernamePassword.
    • Usuario: prov_quality2.
    • Conexión encriptada: false.

    5. Credenciales:

    • Ruta: 10.5.21.29\\bi;DWH_COMFENALCO.
    • Cifrado: No aplica (EncryptConnection: false).

    Propósito¶

    El origen de datos garantiza la conectividad y el acceso a las tablas y medidas necesarias para el funcionamiento del cubo. Este origen permite a los paquetes de SSIS consumir datos directamente desde el DWH para su análisis y explotación.

    Medidas¶

    Descripción General¶

    La tabla Medidas contiene columnas calculadas y una serie de medidas configuradas para realizar análisis y cálculos dentro del cubo. Estas medidas se utilizan para analizar datos clave y generar métricas significativas para la toma de decisiones.

    Estructura¶

    Cada medida calculada se describe brevemente, indicando su objetivo, la lógica DAX utilizada y (donde aplica) el formato de salida.

    1. # Unidades

      • Objetivo: Retorna el conteo distinto de unidades (ID_UNIDAD) de la dimensión “Transversal DIM_UNIDAD”, excluyendo la unidad 5.
      • Expresión DAX:
        var dev = CALCULATE(
    DISTINCTCOUNT('Transversal DIM_UNIDAD'[ID_UNIDAD]),
    'Transversal DIM_UNIDAD'[ID_UNIDAD] <> 5
)
return IF( ISBLANK(dev), -1, dev)
      • Formato: Valor numérico (sin formato adicional).

    2. # Empresas Afiliadas Caja

      • Objetivo: Calcula el número de empresas afiliadas (con TIPO_POBLACION = "EMPRESA" y ACTIVIDAD = "AFILIACION") según la unidad seleccionada; devuelve 0 para algunas unidades y aplica filtros para otras.
      • Expresión DAX (resumida):
        VAR EmpresasCalculo = CALCULATE(
    DISTINCTCOUNT('Transversal FACT_ACTIVIDADES'[PARTNER]),
    'Transversal FACT_ACTIVIDADES'[TIPO_POBLACION] = "EMPRESA",
    'Transversal FACT_ACTIVIDADES'[ACTIVIDAD] = "AFILIACION"
)
VAR EmpresasCalculoSinFiltros = CALCULATE(
    DISTINCTCOUNT('Transversal FACT_ACTIVIDADES'[PARTNER]),
    'Transversal FACT_ACTIVIDADES'[TIPO_POBLACION] = "EMPRESA",
    'Transversal FACT_ACTIVIDADES'[ACTIVIDAD] = "AFILIACION",
    REMOVEFILTERS('Transversal DIM_UNIDAD')
)
var dev = SWITCH(
    TRUE(),
    SELECTEDVALUE('Transversal DIM_UNIDAD'[ID_UNIDAD]) IN {1,2,4}, 0,
    SELECTEDVALUE('Transversal DIM_UNIDAD'[ID_UNIDAD]) = 3, EmpresasCalculoSinFiltros,
    SELECTEDVALUE('Transversal DIM_UNIDAD'[ID_UNIDAD]) = 5, EmpresasCalculo,
    EmpresasCalculo
)
return IF( ISBLANK(dev), -1, dev)
      • Formato: Numérico.

    3. # Afiliados Caja

      • Objetivo: Retorna el conteo de afiliados (TIPO_POBLACION = "AFILIADO") con actividad "AFILIACION", usando variantes con y sin filtros según la unidad seleccionada.
      • Expresión DAX: Similar a la medida anterior, usando SWITCH y dos variables (AfiliadosCalculo y AfiliadosCalculoSinFiltros).
      • Formato: Numérico.

    4. # Beneficiarios Caja

      • Objetivo: Calcula el número de beneficiarios (TIPO_POBLACION = "BENEFICIARIO") en actividad "AFILIACION", aplicando lógica similar a las medidas de afiliados y empresas.
      • Expresión DAX: Utiliza dos variables para el cálculo con o sin filtros y SWITCH para determinar el valor final.
      • Formato: Numérico.

    5. # Aportes Totales

      • Objetivo: Suma el campo NUMERO_APORTES para registros cuya actividad es "APORTES", considerando dos variantes (con y sin filtros de unidad).
      • Expresión DAX:
        VAR AportesCalculo = CALCULATE(
    SUM('Transversal FACT_ACTIVIDADES'[NUMERO_APORTES]),
    'Transversal FACT_ACTIVIDADES'[ACTIVIDAD] = "APORTES"
)
VAR AportesCalculoSinFiltros = CALCULATE(
    SUM('Transversal FACT_ACTIVIDADES'[NUMERO_APORTES]),
    'Transversal FACT_ACTIVIDADES'[ACTIVIDAD] = "APORTES",
    REMOVEFILTERS('Transversal DIM_UNIDAD')
)
var dev = SWITCH(
    TRUE(),
    SELECTEDVALUE('Transversal DIM_UNIDAD'[ID_UNIDAD]) IN {1,2,3,4}, AportesCalculoSinFiltros,
    SELECTEDVALUE('Transversal DIM_UNIDAD'[ID_UNIDAD]) = 5, AportesCalculo,
    AportesCalculo
)
return IF( ISBLANK(dev), -1, dev)
      • Formato: Numérico (con formato de moneda: \$ #,0.00).

    6. Valor Aportes Totales

      • Objetivo: Suma el campo TOTAL_APORTES para actividad "APORTES", aplicando la misma lógica condicional que en # Aportes Totales.
      • Expresión DAX: Similar a la medida anterior pero usando SUM('...'[TOTAL_APORTES]).
      • Formato: Moneda (formateada).

    7. #Empresas_Atendidas

      • Objetivo: Cuenta las empresas atendidas, es decir, el número de registros únicos de ID_EMPRESA en FACT_ACTIVIDADES (excluyendo -1 y unidad 5).
      • Expresión DAX:
        var dev = CALCULATE(
    DISTINCTCOUNTNOBLANK('Transversal FACT_ACTIVIDADES'[ID_EMPRESA]),
    'Transversal FACT_ACTIVIDADES'[ID_EMPRESA] <> -1,
    'Transversal FACT_ACTIVIDADES'[ID_UNIDAD] <> 5,
    'Transversal FACT_ACTIVIDADES'[ID_UNIDAD] <> BLANK()
)
return IF( ISBLANK(dev), -1, dev)
      • Formato: Numérico.

    8. #Afiliados_Atendidos

      • Objetivo: Similar a la medida anterior, pero para ID_AFILIADO.
      • Expresión DAX: Usa DISTINCTCOUNTNOBLANK sobre ID_AFILIADO con condiciones similares.
      • Formato: Numérico.

    9. #Beneficiarios_Atendidos

      • Objetivo: Calcula la cantidad de beneficiarios atendidos mediante DISTINCTCOUNTNOBLANK de ID_BENEFICIARIO.
