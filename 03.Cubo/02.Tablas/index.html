    02. TABLAS

    Tablas¶

    Tabla: Transversal DIM_UNIDAD¶

    Descripción General¶

    La tabla Transversal DIM_UNIDAD es una dimensión clave del modelo semántico que define las unidades organizacionales o funcionales del sistema. Esta tabla permite clasificar y segmentar la información de los procesos y actividades, sirviendo como eje de referencia para relacionar datos en otras tablas del cubo (por ejemplo, para análisis de actividades o métricas específicas por unidad).

    Estructura de la Tabla¶

    La tabla está compuesta por las siguientes columnas:

    Columna Tipo de Datos Columna Fuente Descripción
    ID_UNIDAD int64 ID_UNIDAD Identificador único de cada unidad organizacional.
    UNIDAD string UNIDAD Nombre descriptivo de la unidad (por ejemplo, "Ventas", "Operaciones", etc.).

    Particiones¶

    • Nombre de la Partición: Partition
    • Vista de Datos: full (carga completa de los datos de la tabla).
    • Fuente:
      La tabla se obtiene a través de una consulta en M con la siguiente expresión:
    let
    Source = #"SQL/10 5 21 29\bi;DWH_COMFENALCO",
    Transversal_DIM_UNIDAD = Source{[Schema="Transversal",Item="DIM_UNIDAD"]}[Data]
in
    Transversal_DIM_UNIDAD

    Propósito¶

    La dimensión Transversal DIM_UNIDAD se utiliza para:

    • Identificación y Clasificación: Proveer una llave para identificar cada unidad operativa o funcional en el modelo.
    • Segmentación Analítica: Permitir el filtrado y segmentación de datos en análisis y reportes, facilitando comparaciones y relaciones entre distintas unidades.
    • Integración del Modelo: Servir como referencia en las relaciones entre las tablas de hechos y otras dimensiones, asegurando consistencia en el análisis de la información empresarial.

    Tabla: Transversal DIM_TIEMPO_MENSUAL¶

    Descripción General¶

    La tabla Transversal DIM_TIEMPO_MENSUAL es una dimensión temporal diseñada para segmentar la información en diferentes niveles cronológicos. Esta tabla consolida datos de fechas y sus descripciones, permitiendo analizar la información en intervalos específicos como días, semanas, meses, bimestres, trimestres, cuatrimestres, semestres y años. Su estructura detallada facilita la integración de datos temporales con los hechos del cubo, permitiendo análisis históricos y tendencias a lo largo del tiempo.

    Estructura de la Tabla¶

    La tabla está compuesta por las siguientes columnas:

    • ID_FECHA
    • Tipo de Datos: int64
    • Fuente: ID_FECHA

    • Descripción: Identificador único de la fecha.

    • FECHA

    • Tipo de Datos: dateTime
    • Fuente: FECHA

    • Descripción: Representa la fecha completa.

    • DESC_FECHA

    • Tipo de Datos: string
    • Fuente: DESC_FECHA

    • Descripción: Descripción textual de la fecha (por ejemplo, "Lunes, 01 de Enero de 2024").

    • ID_SEMANA

    • Tipo de Datos: int64
    • Fuente: ID_SEMANA

    • Descripción: Número identificador de la semana del año.

    • DESC_SEMANA

    • Tipo de Datos: string
    • Fuente: DESC_SEMANA

    • Descripción: Descripción o nombre de la semana.

    • ID_NO_MES

    • Tipo de Datos: int64
    • Fuente: ID_NO_MES

    • Descripción: Número del mes en formato numérico (por ejemplo, 1 para enero).

    • DESC_NO_MES

    • Tipo de Datos: string
    • Fuente: DESC_NO_MES

    • Descripción: Descripción numérica del mes.

    • ID_MES

    • Tipo de Datos: int64
    • Fuente: ID_MES

    • Descripción: Identificador del mes.

    • DESC_MES

    • Tipo de Datos: string
    • Fuente: DESC_MES

    • Descripción: Nombre completo del mes (por ejemplo, "Enero").

    • DESC_MES_CORTA

    • Tipo de Datos: string
    • Fuente: DESC_MES_CORTA

    • Descripción: Versión abreviada del nombre del mes (por ejemplo, "Ene").

    • ID_BIMESTRE

    • Tipo de Datos: int64
    • Fuente: ID_BIMESTRE

    • Descripción: Identificador del bimestre.

    • DESC_BIMESTRE

    • Tipo de Datos: string
    • Fuente: DESC_BIMESTRE

    • Descripción: Descripción del bimestre.

    • ID_TRIMESTRE

    • Tipo de Datos: int64
    • Fuente: ID_TRIMESTRE

    • Descripción: Identificador del trimestre.

    • DESC_TRIMESTRE

    • Tipo de Datos: string
    • Fuente: DESC_TRIMESTRE

    • Descripción: Descripción del trimestre.

    • ID_CUATRIMESTRE

    • Tipo de Datos: int64
    • Fuente: ID_CUATRIMESTRE

    • Descripción: Identificador del cuatrimestre.

    • DESC_CUATRIMESTRE

    • Tipo de Datos: string
    • Fuente: DESC_CUATRIMESTRE

    • Descripción: Descripción del cuatrimestre.

    • ID_SEMESTRE

    • Tipo de Datos: int64
    • Fuente: ID_SEMESTRE

    • Descripción: Identificador del semestre.

    • DESC_SEMESTRE

    • Tipo de Datos: string
    • Fuente: DESC_SEMESTRE

    • Descripción: Descripción del semestre.

    • ID_ANIO

    • Tipo de Datos: int64
    • Fuente: ID_ANIO

    • Descripción: Año correspondiente a la fecha.

    • ID_ANIO_ANT

    • Tipo de Datos: int64
    • Fuente: ID_ANIO_ANT

    • Descripción: Año anterior al actual, utilizado para comparaciones temporales.

    • NUM_DIA_SEMANA

    • Tipo de Datos: int64
    • Fuente: NUM_DIA_SEMANA

    • Descripción: Número del día de la semana (por ejemplo, 1 para lunes).

    • FESTIVO

    • Tipo de Datos: int64
    • Fuente: FESTIVO

    • Descripción: Indicador de festivo (1 si la fecha es festiva, 0 en caso contrario).

    • FECHA_CORTA

    • Tipo de Datos: dateTime
    • Fuente: FECHA_CORTA
    • Descripción: Representación abreviada o simplificada de la fecha, útil para visualizaciones y cálculos rápidos.

    Particiones¶

    La tabla cuenta con una partición configurada de la siguiente forma:

    • Nombre de la Partición: Partition
    • Vista de Datos: full (se carga la totalidad de los datos de la tabla).
    • Fuente:
      Se obtiene mediante la siguiente expresión en M:
    let
    Source = #"SQL/10 5 21 29\bi;DWH_COMFENALCO",
    Transversal_DIM_TIEMPO_MENSUAL = Source{[Schema="Transversal",Item="DIM_TIEMPO_MENSUAL"]}[Data]
in
    Transversal_DIM_TIEMPO_MENSUAL

    Propósito¶

    La dimensión Transversal DIM_TIEMPO_MENSUAL se utiliza para:

    • Segmentación Temporal: Dividir los datos en intervalos específicos (día, semana, mes, bimestre, trimestre, cuatrimestre, semestre y año) para facilitar el análisis cronológico.
    • Jerarquías Temporales: Permitir análisis jerárquicos que vinculan datos detallados (como días y semanas) con niveles más agregados (meses, trimestres, años), lo cual es fundamental para la elaboración de informes y dashboards.
    • Integración de Datos: Servir como llave para relacionar información de hechos (por ejemplo, en las tablas de actividades, personal o financiera) con el tiempo, asegurando la consistencia y calidad en los análisis.

    DESCRIPCION string DESCRIPCION

    • Tipo: m.

       Source = #"SQL/10 5 21 29\bi;DWH_COMFENALCO",

    • Relacionar datos categorizados con otras dimensiones y hechos para análisis más detallados.
    • Facilitar la segmentación de datos en reportes y consultas.

    Tabla: Transversal FACT_ACTIVIDADES¶

    Descripción General¶

    La tabla Transversal FACT_ACTIVIDADES es el hecho central del modelo semántico y consolida información de diversas actividades realizadas en el entorno empresarial. Esta tabla integra datos provenientes de múltiples procesos ETL – tales como afiliación, pagos, PQR, evaluaciones, entre otros – y contiene información demográfica, económica y operativa. Es fundamental para el análisis multidimensional y el cálculo de métricas clave en el cubo.

    ID_CATEGORIA int64 ID_CATEGORIA Identificador de la categoría a la que pertenece el registro.
